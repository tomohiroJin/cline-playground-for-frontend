# 原始進化録（PRIMAL PATH）ビルド多様性ブラッシュアップ設計

## 概要

PRIMAL PATH の「ビルドの多様性・リプレイ性」を高める。現状、進化カード32種・文明ツリー全60ノードのほぼ全てが「+数値」の加算効果であり、ビルドの違いが「数字の積み方の違い」に留まっている。これに対し、**質的にプレイの仕方を変えるキーストーン進化**と、**ラン開始時に戦い方の軸を決める始祖トーテム**を導入し、「毎ランの手触りが違う」体験を実現する。

加えて、ビルドの多様性を**横軸（アーキタイプ：何で戦うか）と縦軸（パワーカーブ：いつ強いか）の2次元**で捉える。即効型（序盤強い／後半失速）と晩成型（序盤控えめ／後半圧倒）といった戦略的なテンポの違いを設計に組み込み、さらにキーストーン×トーテム×シナジーの乗算的な噛み合わせで**コンボ・予想外の化け方（emergent build）**が生まれるようにする。

## 背景：なぜビルドが似通うのか（診断）

コード精査により、リプレイ性が頭打ちになっている構造的原因を3点特定した。

1. **ビルドを定義するキーストーンが無い**
   進化カード（`constants/evolution.ts`）もツリー（`constants/tree.ts`）も全て加算ステータス（ATK+5, HP+30 等）。「攻撃が全体化」「会心で回復」「火傷がキルで伝播」のような*ルールを変える*効果が存在せず、全ビルドが「殴って勝つ」に収束する。シナジー tier2 の一部（業火＝火傷2倍等）だけが質的変化に踏み込むが、タグ数依存の受動ドリフトに留まる。
2. **ドラフトの主体性が無い**
   `rollE`（`domain/evolution/evolution-service.ts`）は文明タイプごとに1枚ずつ自動バランス提示。狙ったビルドを能動的に追う手段が無い。
3. **ラン開始時の個性が無い**
   始祖・遺物・トーテムのような「このランはこう戦う」を決める起点がなく、バイオームの微ボーナスと難易度スケールのみ。

本設計は主因 1・3 に直接対処する（2 は将来拡張）。

## 採用アプローチ

**A: キーストーン進化＋始祖トーテム。** 主訴「毎回同じようなビルドになる」の根本原因＝*質的変化の不在*に最も直接効く。既存の synergy/tag 基盤（8タグ）に乗せられるため破壊的変更は少ない。

検討した代替案：
- **B: ドラフト主体性（リロール/バニッシュ/ロック）** — 「狙えるようにする」改善だが、狙う先のカードが加算ステータスのままでは効果が薄い。将来拡張候補。
- **C: ツリーを分岐型に再設計** — 周回間の多様性には効くが、*毎ランの手触り*という主訴には A より弱い。

## システム設計

### 1. 始祖トーテム（ラン開始時の軸選択）

ラン開始時（難易度選択の後、バイオーム選択の前）に専用画面でトーテムを1つ選択。そのランの**戦い方の軸（横軸：アーキタイプ）とテンポ（縦軸：パワーカーブ）**が決まる。**基本3種は最初から、上位3種はクリア回数で解放**（既存 `TIER_UNLOCK` と同方式で `save.clears` から導出）。基本3種で3カーブ（⚡即効／🔗コンボ／🌱晩成）すべてを体験できるよう割り当てる。

| トーテム | 解放条件 | カーブ | 効果 | 狙うビルド |
|---|---|---|---|---|
| 🩸 血の祖 | 最初から | ⚡即効 | 最大HP-20%＋ATK+20%＋会心+5% | 高リスク高火力（序盤バースト） |
| 🔥 炎の祖 | 最初から | 🔗コンボ | 開始時に火傷カード保証＋火傷ダメージ+25% | 火傷伝播コンボ |
| 🏕️ 群れの祖 | 最初から | 🌱晩成 | 仲間枠+1＋開始仲間1体＋仲間ATK+10% | 部族・仲間スケール |
| 🛡️ 岩の祖 | 2クリア | 🔗コンボ | DEF+4＋環境ダメージ-30% | 反射タンク |
| 👻 霊の祖 | 5クリア | 🌱晩成 | 覚醒要求-1（早期覚醒）＋覚醒効果+25% | 覚醒スケール |
| 🌰 種火の祖 | 10クリア | 🌱晩成（極） | 開始ATK-30%だがバイオーム踏破ごとに全ステ+12% | 極・晩成（後半逆転） |

- 各トーテムは対応アーキタイプ／カーブのキーストーンが**節目で優先提示**されるよう重み付けする。
- トーテム効果は `RunState` 初期化時（`startRunState`）に適用する。効果は「ステータス補正」「開始カード/仲間付与」「要求値緩和」「キーストーン提示重み」「カーブ補正（踏破ごとスケール等）」の組み合わせで表現する。

#### パワーカーブ（縦の多様性）

「いつ強いか」を明示的な設計軸とし、トーテム・キーストーンに `curve` メタ属性を付与してUI表示する。同じアーキタイプでもカーブ違いで別物の体験になる（例：火傷の即効型 vs 火傷の晩成型）。

- **⚡ 即効型**：序盤から強いが、敵スケールに対し伸び悩む。リスクで尖らせる。
- **🌱 晩成型**：序盤控えめ、キル数／ウェーブ／バイオーム踏破数／仲間数／獲得骨に応じて雪だるま式に伸びる。スケール元は `RunState` の既存カウンタ（`kills`, `cW`, `bc`, `al`, `bE`）を流用。
- **🔗 コンボ型**：単体は地味だが、他要素と噛み合うと乗算的に化ける（emergent build の起点）。

### 2. キーストーン進化（質的効果カード・新レアリティ `r: 2`）

「数を積む」ではなく「ルールを変える」10種。アーキタイプ（タグ）×パワーカーブの2軸で構成し、各カーブを網羅する。`curve` メタ属性をUI表示し、ビルドのテンポを可視化する。

| キーストーン | タグ | カーブ | 効果（ルール変化） |
|---|---|---|---|
| 狂血の覚醒 | 🐾 wild | ⚡即効 | HP30%以下の間 ATK×2 |
| 原始の咆哮 | 🏹 hunt | ⚡即効 | 各戦闘開始時ATK+50%、ウェーブ毎に-10%ずつ減衰 |
| 狩人の蓄積 | 🏹 hunt | 🌱晩成 | キルごとにATK+3（ラン中恒久スタック） |
| 群狼の戦術 | 🏕️ tribe | 🌱晩成 | 生存仲間1体ごとにプレイヤーATK+10% |
| 骨喰らい | 👻 spirit | 🌱晩成 | 獲得骨10ごとにATK+1（戦闘中も累積反映） |
| 連鎖の業火 | 🔥 fire | 🔗コンボ | 火傷状態の敵を倒すと、残り火傷の50%を次の敵へ継承 |
| 棘の守護 | 🛡️ shield | 🔗コンボ | 被ダメージの30%を敵に反射 |
| 永久凍結 | 🧊 ice | 🔗コンボ | 4ターンごとに敵の次の攻撃を無効化 |
| 不滅の祈り | ♻️ regen | 🃏ワイルド | 戦闘ごとに1回、致死ダメージをHP1で耐える（低空コンボの土台） |
| 諸刃の進化 | 🐾 wild | 🃏ワイルド | DEFを0にし、失ったDEF×3をATKに変換（防御型を攻撃型へ反転） |

> 霊魂解放（覚醒時に全体ダメージ）は霊軸のシナジー tier1/tier2 と機能が重なるため本ロスターからは外し、霊軸は「骨喰らい（晩成）」＋既存覚醒シナジーで賄う。

**技術設計：**
- 既存の加算 `EvoEffect` とは別に `RunState.keystones: KeystoneId[]` を持たせる。スタック値（狩人の蓄積の累積ATK等）は `RunState` の専用フィールド（例：`ksStacks: Record<KeystoneId, number>`）で管理し、戦闘間も保持する。
- `KEYSTONES` レジストリ（`constants/keystone.ts`）に各キーストーンのメタ情報（id, 名前, 説明, タグ, **curve**, アイコン）を定義。
- 効果ロジックは `domain/keystone/` に分離し、**tick-phases の各フック**で発火する：
  - 攻撃前/ATK算出フック（狂血の覚醒・原始の咆哮・群狼の戦術・骨喰らい・狩人の蓄積の累積反映・諸刃の変換）
  - キル時フック（連鎖の業火の継承・狩人の蓄積のスタック加算）
  - 被ダメージフック（棘の守護の反射）
  - 死亡判定フック（不滅の祈りの致死耐え）
  - 敵攻撃フック（永久凍結の周期無効化）
  - 戦闘開始フック（原始の咆哮の初期バフ）
- キーストーン効果を加算系（`applyStatFx`）と分離することで、既存329テストへの影響を最小化しテスト容易性を保つ。

### 3. 入手メカニズム（両建て）

- **節目で確定**：各バイオーム踏破後に専用 `keystone` フェーズで3択提示、必ず1つ選択。トーテム軸に応じて提示候補を重み付け。提示は同一キーストーンの重複取得を避ける。
- **ドラフトに低確率**：`rollE` にキーストーン低確率混入（既存レア判定の上にさらに低い `keystoneRate`）。既に取得済みのキーストーンは除外。

### 4. シグネチャーコンボ（設計意図・バランス指針）

「加算の足し算」を「乗算の掛け算」に変える噛み合わせを設計意図として明文化する。バランス調整時はこれらが**成立はするが必勝ではない**（リスク・条件を伴う）よう調整する。

1. **低空飛行**（🃏予想外）：不滅の祈り＋狂血の覚醒＋🩸血の祖 → わざとHP低空を維持して常時ATK×2、致死は祈りで耐える。事故死リスクと隣り合わせ。
2. **反射タンク**（🔗予想外）：棘の守護＋🛡️岩の祖＋高DEF/HPカード → 殴られるほど反射で勝つ。防御ステが攻撃手段に反転。火力は遅いが安定。
3. **火傷伝播**（🌱晩成）：連鎖の業火＋業火シナジー（tier2）＋火傷カード群 → 火傷が波を溶かし続ける。火傷源を揃えるまで弱い。
4. **群狼**（🌱晩成）：群狼の戦術＋🏕️群れの祖＋部族シナジー＋仲間枠ツリー → 仲間数でプレイヤー火力が乗算。仲間全滅で瓦解。
5. **諸刃の逆転**（🃏予想外）：諸刃の進化＋盾シナジー/DEFカード（DEF源を高く積む）→ 高DEFを爆発的ATKへ変換する一発逆転。変換後は紙装甲。
6. **極・晩成**（🌱）：🌰種火の祖＋骨喰らい/狩人の蓄積 → 序盤の劣勢を耐え、終盤に圧倒的火力へ。序盤の難易度が上がる。

これにより、同じキーストーンでもトーテム・シナジー・カードの組み合わせ次第で**予想外の化け方**が生まれ、横軸（アーキタイプ）×縦軸（カーブ）×コンボの三次元でリプレイ性を確保する。

## フェーズ遷移への影響（既存ステートマシン）

現状：`diff → biome`、`challenge → biome`、`biome → evo`。

変更後：
- `diff → totem`、`challenge → totem`、`totem → biome` を追加。
- バイオーム踏破時に `battle → keystone` を許可し、`keystone → evo`（または次バイオームの `biome`）へ遷移。
- `PHASE_TRANSITIONS` テーブルと `phase-transitions.test.ts` を更新。

## 実装フェーズ（TDD: Red → Green → Refactor）

### Phase 1：始祖トーテム基盤＋基本3種＋カーブ枠組み
- 型（`TotemId`, `PowerCurve`, `RunState` 拡張）・`constants/totem.ts`・`domain/totem/` 作成。
- `startRunState` をトーテム適用対応に拡張（ステータス補正・開始カード/仲間・カーブ補正、TDD）。
- `totem` フェーズ追加・`PHASE_TRANSITIONS` 更新・`TotemSelectScreen.tsx`（カーブ表示付き）作成。
- 基本3種（⚡血／🔗炎／🌱群れ）の効果と選択フローのテスト。3カーブが体験できることを確認。

### Phase 2：キーストーン効果エンジン＋10種＋提示
- 型（`KeystoneId`, `RunState.keystones`, `ksStacks`）・`constants/keystone.ts`・`domain/keystone/` 作成。
- tick-phases へのフック追加（攻撃前/キル時/被ダメ/死亡判定/敵攻撃/戦闘開始、各効果を純粋関数で実装、TDD）。
- `keystone` フェーズ追加・`PHASE_TRANSITIONS` 更新・`KeystoneScreen.tsx`（節目3択・カーブ表示）作成。
- `rollE` へのキーストーン低確率混入（取得済み除外）。
- キーストーン10種それぞれの効果テスト。晩成スタックが戦闘間で保持されることを確認。

### Phase 3：上位トーテム解放＋コンボ／連携＋調整
- 上位トーテム3種（🔗岩／🌱霊／🌰種火）の効果実装と解放条件（`save.clears` 導出）。
- トーテム軸・カーブによるキーストーン提示重み付け。
- シグネチャーコンボ6種が成立することを統合テストで確認（必勝にならない範囲で）。
- `RunStats`／StatsScreen／実績との連携。
- バランス調整（パワークリープ抑制、即効/晩成のカーブ交差点が中盤に来るよう確認）。

## 非目標（YAGNI）

- ドラフトのリロール/バニッシュ/ロック（アプローチB）は本設計の対象外。Phase 完了後に効果を見て別途検討する。
- 文明ツリーの分岐型再設計（アプローチC）は対象外。

## リスクと留意点

- **パワークリープ**：トーテム＋キーストーンは総合的に強化方向。各 Phase でテスト＋手動プレイによりバランス確認し、必要なら難易度スケールを調整する。
- **既存テストへの影響**：フェーズ追加は `PHASE_TRANSITIONS` 依存テストに波及する。キーストーン効果は加算系と分離して既存ロジックへの侵襲を抑える。
- **セーブ後方互換**：`totemId`／`keystones` は新規ランから付与。既存セーブ（骨・ツリー・クリア回数）の構造は変更しない。
