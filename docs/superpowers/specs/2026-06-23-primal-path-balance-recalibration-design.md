# PRIMAL PATH バランス再調整（実プレイ較正）設計

- 日付: 2026-06-23
- 対象: `src/features/primal-path/`（シミュレータ・キーストーン・戦闘計算・難易度定数）
- ブランチ: `chore/primal-path-fun-verification`（PR #133 と同一ブランチで継続）
- 関連: [面白さ検証レポート](../reports/2026-06-21-primal-path-fun-verification-report.md) / [前回バランス設計](2026-06-21-primal-path-balance-meta-curve-design.md)

## 1. 背景と目的

前回のバランス調整 (a)(b)（PR #133）後、ユーザーが実プレイしてフィードバック:

- 戦略は増えたが同時に敵が弱くなり**楽勝**になった。
- **狂気（狂血 ATK×2 × 儀式 ATK×3 = 最大×6 の低HPバースト）が強すぎ**、ATK を上げれば勝ててしまう。
- **ピンチにならない**（緊張感がない）。
- **文明ツリーを取らなくても氷河期(di1)まで苦戦なしでクリア**できる。
- 大災厄(di2)まで進んで辛うじて負けた。

### 根本原因

バランスガードレールが `greedy-atk` ＋**キーストーンをランダム選択**（`run-simulator.ts:243`）で測っており、
実プレイヤーが狙う **狂血(madblood: HP30%以下で ATK×2, `keystone-service.ts:48`)** や
**儀式(rit: 低HPで ATK×3, `combat-calculator.ts:83`)** の低HPバースト一強を再現していなかった。
そのため「sim では di1=33%（苦戦）」なのに実プレイでは楽勝、というズレが生じ、di1/di2 を緩めすぎた。

加えて di2(hm1.8/am1.55)→di3(hm5.8/am5.4) の**断崖**も残っている。

## 2. 方針

**シミュレータを実プレイ（最適化バースト）に忠実化 → それを基準に較正ループを回す。**
ユーザー決定:
- 較正アプローチ: シミュレータ強化＋ループ。
- 難易度目標: **無強化（ツリーなし）は原始(di0)でも苦戦**（ツリー投資の価値を強める）。
- 狂気対処: **バースト倍率を下げてリスク化**＋難易度上昇で低HPを本当に危険にする。

## 3. 設計

### 3.1 シミュレータ忠実化（較正の土台）

最適化ビルドを再現する強化を `__sim__/run-simulator.ts` に入れる。

- **キーストーン選択の最適化**: 現状の一様ランダム（243行）を、ATK 系強キーストーン
  （`madblood` / `primal_roar` / `hunter_stack`）を優先する選択へ変更する。
- **「バースト」進化戦略の追加**: `EvoStrategy` に `'burst'` を追加。ATK 最大化に加え、
  HP 犠牲系進化（例: 血の契約＝HP半減で ATK×2）を許容して自然に低HP域へ入り、
  狂血/儀式の倍率が発動するガラスキャノンを再現する。
- **較正の前提確認**: まず**現状定数のまま**この強化 sim を回し、di1/di2 が楽勝（高勝率）になることを確認する。
  これがユーザーフィードバックの再現＝sim が実プレイに近づいた証拠であり、較正の出発点になる。

### 3.2 狂気バーストのリスク化（一強の緩和）

倍率は残し高リスク・高リターンを維持しつつ、合算上限を下げる:
- 狂血(madblood): ATK×2 → **×1.6**（`keystone-service.ts:48`）
- 儀式(rit)低HP: ATK×3 → **×2**（`combat-calculator.ts:83` の `pa *= 3`）
- 合算最大 ×6 → **×3.2**。難易度上昇と併せ、低HP維持を本当に危険にしてピンチを作る。

（最終倍率は較正ループで微調整しうる。上記は出発点。）

### 3.3 難易度の再較正（ループの本体）

`constants/difficulty.ts` の hm/am を再調整する:
- 過度に緩めた di0/di1/di2 を引き上げる（無強化・最適バーストで **di0 勝率 ~50%＝苦戦**）。
- di2→di3 の**断崖を是正**し、隣接難易度間の勝率低下が緩やかになるようにする
  （バースト弱体化で di3 の hm/am は現状 5.8/5.4 から下げられる見込み）。
- メタ進行（ツリー）の傾斜を強め、ツリー投資がより上の難易度の解放に効くようにする。

### 3.4 ガードレール書き換え（新基準）

`__sim__/balance-guardrail.test.ts` を新 sim 戦略（`'burst'`）＋新目標で更新する:
- **無強化・最適バースト di0** トーテム平均勝率 ∈ **[0.40, 0.65]**（原始でも苦戦）。
- **無強化 di1** は di0 より明確に低い（ツリーなしは厳しい。例: < 0.30）。
- **フル強化 di3** ∈ **[0.55, 0.80]**（必勝でない・維持）。
- **断崖なし**: 無強化での隣接難易度の勝率差が極端でない（例: di0→di1→di2 の各低下が滑らか）。崖検出アサーションを追加。
- (b) パワーカーブ終盤交差（bc0/bc1 blood>ember、bc2 ember>blood）は維持（フィードバック対象外・前回緑）。

## 4. 検証と反復

1. sim 忠実化（3.1）を実装し、現状定数で「楽勝再現」を確認（フィードバック再現）。
2. 新ガードレール（3.4）を新目標で記述（現状定数では FAIL するはず）。
3. 倍率ナーフ（3.2）＋難易度（3.3）を**1レバーずつ**調整し、軽量（`SIM_SEEDS=30`）で方向確認。
4. フル（`SIM_SEEDS=150`）でガードレール全件緑を確認。
5. 必要に応じてユーザーの実プレイで再検証（ハイブリッド）。

## 5. 影響と既存テスト

- `keystone-service.ts`・`combat-calculator.ts` の倍率変更で、該当のユニットテスト期待値が壊れる可能性（madblood/rit のダメージ計算をアサートするテスト）。計画作成時に精密列挙し更新する。
- `run-simulator.ts` の `EvoStrategy` 拡張は型変更を伴う。`balance-report.test.ts` の `STRATEGIES` 配列等への波及を確認。
- 前回の (a) ガードレール目標（中間バンド）は本設計で上書きされる（実プレイ較正が優先）。

## 6. リスクと限界

- sim 忠実化は近似。人間の判断は更に上手い可能性があり、最終確認はプレイテスト推奨。
- 儀式(rit) は文明コア機構。倍率変更はビルド全体の感触に影響するため、ナーフは控えめから始め較正で詰める。
- バランスは経験的反復で収束。§3 の数値は出発点、**ガードレール緑＋プレイテスト体感**が成功基準。
