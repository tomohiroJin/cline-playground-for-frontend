# PRIMAL PATH UX調整 (c) デッドスペース／可読性／タグ強調 設計

- 日付: 2026-06-21
- 対象: `src/features/primal-path/`（styles・shared コンポーネント・色定義）
- ブランチ: `chore/primal-path-fun-verification`
- 関連: [面白さ検証レポート](../reports/2026-06-21-primal-path-fun-verification-report.md) §5「UX レビュー」

## 1. 背景と目的

面白さ検証の UX レビュー（design-review）で挙がった指摘のうち、Must Fix 2件＋密接する推奨1件を是正する。
ビルド多様性の「個性」が UI の余白・可読性で視覚的に埋もれている状態を解消し、品質感とビルド軸の可読性を高める。

対象（スコープ）:
- **① デッドスペース解消**（Must Fix）: 720×960 固定シェルがデスクトップで黒い余白に囲まれ「モバイル流用・未完成」に見える。
- **② 可読性 WCAG AA**（Must Fix）: シナジータグ・覚醒進捗・数値が 7px かつ低コントラストで WCAG AA(4.5:1) 未満。
- **③ アーキタイプタグ強調**（推奨・②と密接）: ビルド軸を決める最重要情報なのに小さく発見しにくい。

スコープ外: ④ 進化カード情報密度、⑤ 節目KS演出（推奨・別途）。

## 2. 現状（調査済み）

- `styles.ts` `GameContainer`: 全画面・背景フラット `#0a0a12`。`GameShell`: デスクトップ(>1024px)で **720×960px 固定**、中央配置。モバイル(<600px)は 100vw/100dvh 全画面。
- 7px のマイクロラベル: `components/shared/AffinityBadge.tsx`、`components/shared/SynergyBadges.tsx`、`components/EvolutionScreen.tsx`、`components/battle/PlayerPanel.tsx`。
- 低コントラスト色（背景 #12121e/#0a0a12 に対し 4.5:1 未満）: 茶 #988070(3.1:1)、暗黄 #c0a040(2.9:1)、黄 #e0c060(3.8:1)、オレンジ #f08050(3.8:1)。色は `SYNERGY_TAG_INFO`・アーキタイプ色定義に集中。

## 3. 設計

### ① デッドスペース解消（装飾背景の額縁化）

`GameContainer` の背景を、中央シェルを引き立てるテーマ背景に置換する。レイアウトは無変更（リフローしない）。

- 多層グラデ＋ヴィネット: 放射状グラデで中心やや明るく外周を落とし、余白を「意図した額縁」に見せる。
- 原始モチーフ: CSS による極薄テクスチャ（彩度・明度を強く抑制し、シェルと競合させない）。
- デスクトップのみ効果が出る（モバイルはシェル全画面で余白なし＝現状維持）。
- 60-30-10 配色（紺黒 60% / カード 30% / 金アクセント 10%）を維持する。

### ② 可読性（WCAG AA 4.5:1）

- **極小フォント是正**: 7px のマイクロラベルを **最小 10px** へ引き上げる（アーキタイプタグは ③ で 12px）。
  カード内のはみ出しが出ないことを確認する（密度④はスコープ外だが破綻は回避）。
- **低コントラスト色の是正**: 4.5:1 未満の色を、同系統で明度を上げた AA 準拠の変種へ置換する。
  色は共有定義（`SYNERGY_TAG_INFO`・アーキタイプ色）で集中管理されているため一括で是正する。
- **コントラストガードレールテスト（新規）**: 採用色×背景の WCAG コントラスト比を計算し **≥ 4.5** をアサートする
  ユニットテストを追加する。「WCAG AA」を自動回帰ガードに落とし込む（バランスガードレールと同思想・CI で常時実行可能な軽量テスト）。

### ③ アーキタイプタグ強調

ビルド軸を決める `AffinityBadge`／アーキタイプタグを発見しやすくする:
- フォント 12px ＋ チップ状の背景/枠で視覚的な重みを付与。
- 色彩心理（即効=赤 / コンボ=青 / 晩成=緑 / ワイルド=白）は維持しつつ、AA(4.5:1) を満たす明度に調整。

## 4. 影響コンポーネントと検証

- 変更: `styles.ts`（GameContainer）、`components/shared/AffinityBadge.tsx`・`SynergyBadges.tsx`、
  色定義（`SYNERGY_TAG_INFO`・アーキタイプ色）、`components/EvolutionScreen.tsx`・`components/battle/PlayerPanel.tsx`。
- 検証:
  - 既存コンポーネントテストが緑のまま（構造・文言の非破壊）。
  - 新規コントラストガードレールで採用色が AA(≥4.5:1) を数値保証。
  - 視覚確認は Playwright スクリーンショット（可能な範囲）。E2E はローカル実行に制約あり。

## 5. リスク

- 装飾背景がシェルと競合して可読性を下げないよう、彩度・明度を強く抑制する。
- フォント拡大でカード内が窮屈／はみ出しにならないか確認する。
- 色の明度調整で色彩心理（アーキタイプの色の意味）が崩れないよう、色相は維持し明度のみ上げる。
