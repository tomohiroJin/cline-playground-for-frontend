# IPNE キャラクター・ブラッシュアップ 設計

> 作成日: 2026-06-17
> 対象: `src/features/ipne/`（プレゼンテーション層のみ）
> 目的: キャラクターのドット絵と動きの質を向上させ、IPNE の見栄え・手応えをブラッシュアップする。

## 背景・課題

ビジュアルコンパニオンで現行スプライトを実データ描画し、ユーザーと確認した結果、
以下すべてを改善対象とすることで合意した。

- ドットの精細さ・形の読み取りやすさ（シルエットが背景に溶ける）
- 色・陰影・立体感（のっぺりしている）
- 歩行の滑らかさ・躍動感（現状はパタパタ。後述の通り歩行フレームを2枚しか使っていない）
- 待機モーション（存在感が薄い）
- 攻撃の迫力・手応え
- 敵キャラの表現
- 向きごとの差・一貫性（左右が単純ミラーで個性が薄い）

### 現状実装で判明した事実

- スプライトは外部画像ではなく **2次元パレットインデックス配列のコードスプライト**
  （`presentation/sprites/spriteData.ts`）。改善＝配列・描画コードの編集で完結し、ビルド成果物に画像を持たない。
- 歩行シートは `[idle, walk1, mid, walk2]` の **4枚定義されているが、描画は
  `sprites[1 + (frame % 2)]`（= walk1 と mid の2枚）しか使っておらず walk2 が死蔵**
  （`presentation/screens/render/drawPlayer.ts`）。
- スプライト描画は `SpriteRenderer` がスケール別にキャッシュ（最近傍拡大）している。

## スコープ（3層構造）

| 層 | 内容 | 手段 | リスク |
|---|---|---|---|
| 動きの層 | 歩行4枚循環 / 上下バウンス / 接地シャドウ / 踏み込みスカッシュ / 攻撃の予備動作＋踏み込み＋オーバーシュート | 描画・タイミングのみ（描き直し無し） | 低 |
| ドット自動補正層 | 輪郭線＋陰影ランプを機械生成し**キャラクタースプライトのみ**に適用 | 純粋関数＋キャッシュ時処理 | 低 |
| ドット手描き層 | 戦士・盗賊の**全4方向**を手描きで作り込み（顔/装備のディテール、左右のミラー脱却で個性付与） | ピクセル配列の手編集 | 中（工数大） |

### スコープ外

- `domain/` `application/` `infrastructure/` 層への変更（ビジネスロジック非干渉）。
- 床/壁/アイテム/罠など**環境タイルへの自動補正適用**（タイルグリッドの可読性維持のため除外。ユーザー承認済み）。
- 移動速度・当たり判定などゲームバランスの変更（見た目・動きの質のみ）。

## アーキテクチャ / 配置方針

変更は `src/features/ipne/presentation/` 配下のみ。DDD の依存方向を維持する。

### 新規モジュール

- `presentation/sprites/motion.ts` — 動きの計算を担う**純粋関数**群。
  - `selectWalkFrameIndex(now, frameDuration)`: `[1,2,3,2]` 循環でフレーム番号を返す。
  - `computeWalkBob(now, frameDuration)`: 歩行サイクル同期の上下オフセット（±2px 相当）。
  - `computeSquash(now, frameDuration)`: 接地フレームの scaleY（最小 0.94、足元基準）。
  - `computeAttackTransform(now, attackStart, direction)`: 予備動作→踏み込み→オーバーシュートの
    位相に応じた `{ dx, dy, scale, frameIndex }` を返す。
- `presentation/sprites/dotEnhance.ts` — 自動補正の**純粋関数**群（ImageData 非依存、配列操作）。
  - `applyOutline(pixels, palette)`: 透明かつキャラ縁に隣接するピクセルを暗色化した新配列を返す。
  - `applyEdgeShading(pixels, palette)`: 縁の内側を暗く・上面を明るくした新配列を返す。
  - いずれも非破壊（入力を変更しない）。

### 既存モジュールの変更

- `presentation/screens/render/drawPlayer.ts`
  - 歩行・bob・スカッシュ・攻撃トランスフォームを `motion.ts` 呼び出しに置換。
  - 接地シャドウ描画を追加。
- `presentation/screens/render/drawEnemies.ts`
  - 接地シャドウ描画を追加。歩行する敵には軽い bob を付与（任意・軽量）。
- 共通ヘルパー `drawGroundShadow(ctx, screenX, screenY, tileSize, lift)` を描画層に追加
  （`drawPlayer` / `drawEnemies` から共用）。
- `presentation/sprites/spriteRenderer.ts`
  - キャッシュ生成時（`getCachedCanvas`）に、対象がキャラクタースプライトの場合のみ
    `applyOutline`（常時）と `applyEdgeShading`（`autoShade` が false でない場合のみ）を適用。
  - スケール別キャッシュのため**1スプライトあたり1回だけ**処理。実行時性能への影響なし。
- `presentation/sprites/spriteData.ts`
  - `SpriteDefinition` に `autoShade?: boolean` を追加（既定は自動陰影ON、手描き対象は false）。

## 各層の詳細仕様

### 動きの層

- **歩行**: フレーム選択を `[1,2,3,2]` 循環に変更し、死蔵中の walk2 を活用してパタパタ感を解消。
- **bob**: 歩行サイクルに同期した上下 ±2px。**攻撃中・被弾中は無効**。
- **接地シャドウ**: プレイヤー＋敵の足元に楕円シャドウ。bob で浮くと縮小・減光して接地感を出す。
- **スカッシュ**: 接地フレームで scaleY を一瞬 0.94（足元基準で拡縮）。
- **攻撃**: 予備動作（後方へ引き）→ 踏み込み（`direction` 方向へ前進＋scale 1.08）→
  オーバーシュート復帰。既存の武器光跡・衝撃波（RADIANT ティア）は維持し、踏み込みと併用する。

### ドット自動補正層

- **適用対象 = キャラクターのみ**（プレイヤー・敵・ボス）。環境タイルには適用しない。
- **輪郭線**: 透明かつキャラ縁に隣接するピクセルを暗色化。silhouette を背景から分離。
  **全キャラ常時ON**。
- **陰影ランプ**: 縁の内側を暗く・上面を明るく。
  **手描き対象（戦士・盗賊）は自前の陰影を持つため自動陰影を OFF**（`autoShade: false`）にし、
  輪郭線のみ適用する。これにより自動陰影と手描き陰影の二重処理を回避する。

### ドット手描き層

- 対象: **戦士・盗賊 × 4方向**の基本ポーズ（idle ＋ 歩行2枚）。
  攻撃/被弾フレームは既存ベース＋自動輪郭で対応する。
- **左右のミラー脱却**: 現状 `left = mirror(right)`。武器/装備の持ち手・髪の流れを左右で
  描き分け、個性を付与する。
- **上・下の作り込み**: 顔（目）・装備（剣/盾・ダガー/マント）の視認性を向上させる。

## テスト方針（TDD）

- `motion.ts`: 各純粋関数の境界値テスト（フレーム選択の循環、bob の周期と振幅、
  攻撃トランスフォームの各位相の dx/dy/scale/frameIndex）。
- `dotEnhance.ts`: 輪郭・陰影の入出力をピクセル配列で検証
  （縁検出の正しさ、範囲外座標の安全性、入力非破壊）。
- スプライト構造の不変条件テスト: 全フレーム 32×32、パレット参照が範囲内、
  手描き後の戦士・盗賊 left/right が「単純ミラーでない」ことの検証。
- 既存テスト（`movement.test.ts`、`drawPlayer` 関連、`spriteData.test.ts` 等）を壊さないこと。
- 新規コード カバレッジ 80%+、純粋関数は 90%+ を目標。

## フェーズ分割（PR単位）

リスクと即効性が逆相関のため、低リスク・即効の層から順にリリースする。

1. **PR-1 動きの層**: `motion.ts` ＋ `drawGroundShadow` ＋ `drawPlayer`/`drawEnemies` 配線。
   描き直し無しで体感効果が最も大きい。
2. **PR-2 自動補正層**: `dotEnhance.ts` ＋ `SpriteRenderer` 適用 ＋ `autoShade` フラグ追加。
3. **PR-3 手描き層**: 戦士・盗賊 4方向のドット作り込み（ミラー脱却、`autoShade: false` 設定）。

各 PR は `lint:ci → typecheck → test:coverage → build` を通し、`main` へは PR 経由でマージする。
