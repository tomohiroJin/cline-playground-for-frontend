# Agile Quiz Sugoroku — AI クイズ追加 設計書

- 作成日: 2026-06-25
- 対象 Feature: `src/features/agile-quiz-sugoroku/`
- 目的: 「今話題の AI」をテーマにしたクイズ問題を追加し、ゲームの題材を最新化する

## 1. 背景と目的

Agile Quiz Sugoroku は現在 445 問・16 ジャンルでアジャイル開発／ソフトウェア工学の知識を問うゲーム。
昨今関心の高い AI（生成 AI・AI 支援開発）を題材として加え、学習価値と話題性を高める。

AI は特定のスプリント工程に一対一で対応しないため、**「AI 活用」という横断ジャンル（タグ）を新設**し、
内容の文脈に合う既存カテゴリ（工程）へ問題を分散配置する。

## 2. 内容の方向性

AI 支援開発と生成 AI 基礎の **両方を混在** させる。

- **AI 支援開発・実務**: AI ペアプログラミング、AI コードレビュー、テスト生成、リファクタリング提案、AIOps など
- **生成 AI・LLM 基礎**: LLM、トークン、コンテキストウィンドウ、RAG、ファインチューニング、プロンプト技法、埋め込み、ハルシネーションなど
- **AI 倫理・ガバナンス**: AI 生成コードの著作権・バイアス・レビュー観点など（review/emergency 工程に少量）

## 3. タグ設計

新タグ `ai` を 1 つ追加（16 → 17 ジャンル）。AI 関連問題を 1 ジャンルに集約し、結果画面で独立集計する。

```typescript
// data/questions/tag-master.ts に追記
{ id: 'ai', name: 'AI活用', description: 'LLM・生成AI・AIコーディング・プロンプト・AI倫理', color: '#e879f9' }
```

- 色は既存 16 色と重複しない未使用のマゼンタ系 `#e879f9` を採用。
- `tag-master.ts` が源泉のため、追加するだけで以下に自動反映される:
  - 勉強会モードのジャンル選択（`StudySelectGenre.tsx` が `TAG_MASTER` を `.map`）
  - 結果画面のジャンル別正答率分析（`tag-stats.ts` が `TAG_MAP` を参照）
- 複合タグを許容する。例: AI コードレビュー問題は `["ai", "code-quality"]`。
  これにより AI 問題が既存ジャンルの正答率にも貢献し、新ジャンルが孤立しない。

## 4. 配置設計（工程＝出題タイミングを維持）

内容の文脈に最も合う既存カテゴリ（JSON ファイル）へ約 32 問を分散する。

| カテゴリ (JSON) | 問数 | AI 問題の切り口 |
|---|---|---|
| planning | 3 | AI による見積もり補助・要件整理、PBI 草案生成 |
| impl1 | 6 | 生成 AI 基礎（LLM・トークン・コンテキストウィンドウ）、AI コーディング基礎 |

問数の内訳は実装時に微調整可（合計 30 問以上を維持）。

## 5. 問題データ形式と品質契約

各問題は既存スキーマに完全準拠する。

```json
{
  "question": "LLMにおける「コンテキストウィンドウ」とは？",
  "options": ["一度に処理できるトークンの最大量", "学習データの総量", "生成速度の上限", "並列リクエスト数"],
  "answer": 0,
  "tags": ["ai"],
  "explanation": "コンテキストウィンドウは入力＋出力で扱えるトークン上限を指す。"
}
```

### テストが自動で守る契約（`questions.test.ts`）

- 4 択固定 / `answer` は 0-3 / 正解インデックスは選択肢範囲内
- 全問 `explanation`（解説）必須
- `tags` は全てタグマスタに存在し、最低 1 つ持つ
- 全タグが最低 1 問で使用される（→ `ai` 定義後は AI 問題が最低 1 問必須）

### 編集方針（テストでは守れない・人間レビューで担保）

- **概念ベースで書き、陳腐化を避ける**: 特定製品のバージョン番号・価格・最新モデル名などの揮発的事実は使わない。
  例: 「Copilot の最新モデル名」ではなく「AI ペアプログラミングの利点と注意点」。
- **正解位置を 0〜3 に散らす**（新規分は均等を意識）。
- **誤答選択肢（distractor）はもっともらしく作る**（明らかなダミーを避ける）。

## 8. スコープ外（YAGNI）

- AI 専用の新スプリント工程・新カテゴリ JSON の追加はしない（既存 8 カテゴリへ分散）
- 粒度別の複数 AI タグ（ai-dev / ai-foundations 等）への分割はしない（1 タグに集約）
- 既存問題の AI 視点での書き換えはしない（新規追加のみ）
