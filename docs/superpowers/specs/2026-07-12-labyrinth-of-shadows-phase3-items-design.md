# Labyrinth of Shadows Phase 3: アイテムの意味の再設計

日付: 2026-07-12
ステータス: 承認済み
前提: Phase 1（3D化 #162）・Phase 2（逃走再設計 #163）マージ済み

## 背景と目的

Phase 2 で「視線・音・追跡（patrol→chase→search）」というステルスの土台が完成したが、
アイテムは小石を除き 2D 時代の「触れた瞬間に即時発動する置き物」のまま。
Phase 3 では全アイテムをこの土台に接続し、**アイテム＝敵との駆け引きの道具**へ再定義する。

- アイテム種類は増やさない（既存6種の作り直し）
- 中心原則: **「音は敵を呼ぶ」という1つのルール**を、小石（味方の道具）と罠（敵の道具）の
  表裏として学べるようにする

## 各アイテムの新仕様

| アイテム | 新仕様 | 変更 |
|---|---|---|
| 🔑 鍵 | 変更なし（出口解錠＋コンボスコア） | なし |
| 🪨 小石 | 変更なし（クリック投擲・音で誘導） | なし |
| 💊 回復薬 | 変更なし（ライフ+1／満タン時ボーナス） | なし |
| ⚡ 加速 | **能動化**: 拾うとストック、任意タイミングで発動 | 即時発動 → ストック制 |
| 🗺️ 地図 | 即時発動のまま、周囲公開＋**全敵位置を数秒間表示** | 索敵ツール化 |
| 📦 罠 | **騒音罠**: 時間ペナルティ廃止、大音で敵を呼ぶ | 減点箱 → ステルス崩壊イベント |

### ⚡ 加速（能動ストック制）

- 拾うと `speedCharges` が +1（上限 **2**。上限時はフィールドに残す＝小石と同じ挙動）
- 発動: キーボード **E** ／ タッチ専用ボタン。発動で 10 秒間 1.3 倍加速（現行値を維持）
- **chase 中でも使用可**（追われた瞬間に切る切り札が主用途）
- 加速効果中の再発動は不可（無駄撃ち防止）。hiding 中も不可（小石と同基準）
- UI: 小石の所持表示・投擲ボタンのパターンを踏襲したストック表示＋使用ボタン

### 🗺️ 地図（索敵ツール化）

- 従来の周囲公開（半径 10 セル）は維持
- 追加: 取得から **5 秒間**、全敵の現在位置をミニマップにリアルタイム表示
  （`enemyRevealTimer` が有効な間、ミニマップ描画に敵アイコンを追加）
- 索敵 UI（!/? マーカー）とは独立に動作する

### 📦 罠（騒音罠）

- 時間 -12 秒ペナルティを**廃止**
- 踏むと大音が鳴り、**騒音半径内の敵が search 状態でその地点へ殺到**
  （小石着地時の search 遷移ロジックを流用）
- 騒音半径の初期値: **8 セル**（小石の NOISE_RADIUS=5 より広い＝罠の方が音が大きい）
- コンボリセットは維持（踏む=ミスの手応えを残す）
- chase 中の敵には影響しない（小石と同じ規則で一貫性を保つ）

## 実装の骨子

- **状態**: `GameState` に `speedCharges: number` と `enemyRevealTimer: number` を追加
- **入力**: `game-tick.ts` の `GameInput` に `useSpeed: boolean` を追加。
  発動処理は `advanceGame` 内（純粋関数のまま）
- **罠**: `game-logic.ts` の `updateItems` の trap 分岐から、
  `enemy-strategy` の騒音反応（小石着地と同じ経路）を呼ぶ。半径のみ定数で差し替え
- **ミニマップ**: `minimap-renderer.ts` で `enemyRevealTimer > 0` の間、敵位置を描画
- **UI**: プレゼン層に加速ボタン（タッチ）＋ E キー割り当て。石ボタンの実装パターンを踏襲
- **定数**: 騒音半径・表示秒数・ストック上限は `domain/constants.ts`（GAME_BALANCE）に定数化。
  注意: `domain/types.ts` の手書き SoundName 重複（Phase 2 教訓）— サウンド追加時は両方同期
- **コンテンツ**: `CONTENT.demo`（チュートリアル）とアイテム説明・イントロ文を新仕様に更新

## バランス初期値（実機プレイで較正）

| 定数 | 初期値 |
|---|---|
| 加速ストック上限 | 2 |
| 地図の敵表示時間 | 5 秒 |
| 罠の騒音半径 | 8 セル |
| 加速時間・倍率 | 10 秒・1.3 倍（現行維持） |

難易度別のアイテム個数（traps/heals/speeds/maps）は現行値を出発点とし、実機フィードバックで調整する。

## テスト方針（TDD）

`advanceGame` / `game-logic` は純粋関数のため、以下をユニットテストで先に固める:

- 罠踏み → 騒音半径内の敵が search 遷移・半径外は不変・chase 中は不変・コンボリセット
- 加速: 拾得でストック増（上限で残置）・`useSpeed` で発動・効果中/hiding 中の再発動不可
- 地図: 取得で `enemyRevealTimer` セット・時間経過で減衰
- ミニマップ描画のタイマー連動（canvas 描画は呼び出しレベルで検証）

R3F 表示部（ボタン・3D表現）は既存同様 jsdom テスト対象外。実機確認で補完する。
