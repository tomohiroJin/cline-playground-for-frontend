# Labyrinth of Shadows Phase 2: 逃走の駆け引き再設計 — 設計書

- 日付: 2026-07-10
- 対象: `src/features/labyrinth-of-shadows/`
- 前提: Phase 1（R3F による本物の 3D 化、PR #162）が main マージ済み
- 位置づけ: 4フェーズ改修の Phase 2（本丸）。元の不満「一定レベルを超えると逃走が成り立たない」への直接回答

## 背景と問題

現在の敵 AI は壁を無視した距離判定（`d < CHASE_RANGE`）でプレイヤーを検知し、BFS
パスファインディングで最短経路を回り込んでくる。「角を曲がって視線を切る」という逃走の
基本が構造的に成立せず、レベル上昇で敵速度が上がると逃走自体が破綻する。
隠れ場所（`isPlayerHiding`）は検知の完全無効化であり、駆け引きではなく安全スイッチに
なっている。

## 意思決定（ブレスト結果）

| 論点 | 決定 |
|------|------|
| 再設計の軸 | 見失いルール明確化（土台）＋能動の一手（1つ）の両方・段階導入 |
| 視界モデル | 視野角コーン（±60°）＋壁による視線遮蔽（レイキャスト） |
| 索敵 UI | 音（距離減衰＋パン）＋画面端の方向付き状態インジケータ |
| 能動の一手 | 石を投げる（音で敵を捜索状態に誘導） |
| 隠れ場所 | 視線切りと連動（視認中に隠れても無効、視線を切ってから隠れれば安全） |

## 1. 敵の状態機械

Chaser の「距離内なら常時追跡」を 4 状態の状態機械に置き換える。

```
巡回 patrol ──発見(視認)──▶ 追跡 chase ──視線喪失──▶ 捜索 search ──時間切れ──▶ 巡回
   ▲                            ▲                        │
   └────────────────────────────┴──────再発見────────────┘
```

- **発見条件（全成立）**: ①距離が `SIGHT_RANGE` 内 ②敵の向き ±60° の視野角コーン内
  ③壁レイキャストで遮蔽なし ④有効に隠れていない
- **追跡（chase）**: 現行の BFS 追跡を流用。`lastSeenX/Y` の更新は視認できている間のみ
- **捜索（search）**: 視線喪失後、猶予時間（`LOSE_SIGHT_GRACE` ≒ 2 秒）は最終目撃地点へ
  直行 → 到達後、周辺セルを `SEARCH_DURATION` 秒うろつく → 再発見できなければ巡回へ復帰
- **隠れ場所**: 視認されている最中に隠れても検知は継続（敵は最終目撃地点＝隠れ場所付近に
  来て捜索する）。視線が切れた後に隠れた場合のみ検知不可
- **Wanderer**: 変更なし。追跡しない徘徊型の性格を維持し、接触の脅威として残す
- **Teleporter**: テレポートのクールダウンは維持。追跡開始を視認ベースに統一し、
  テレポート後にプレイヤーが視界内にいなければ即追跡しない（ワープ即ホーミングの解消）

## 2. 視線判定（vision サービス）

`domain/services/vision.ts` を新設し、純粋関数として実装する。

- `hasLineOfSight(maze, x1, y1, x2, y2): boolean` — グリッド DDA で線分上の壁セルを検査
- `canSeePlayer(params): boolean` — 距離＋視野角＋遮蔽＋隠れ状態の総合判定
- `bfsPath` と同様に `domain/services` 配下の純粋関数とし、jsdom で完全にユニットテスト
  可能にする。TDD でテストを先に書く

## 3. 能動の一手: 石を投げる

- **所持**: 初期 3 個・上限 5 個。迷路内に小石が落ちており拾って補充できる
- **操作**: ポインタロック中の左クリックで照準（カメラ）方向へ投擲
  （現状左クリックに操作割り当てが無いため競合しない）
- **挙動**: 直線飛行し、壁衝突または最大飛距離で着地 → 着地点に音イベントを発生
- **敵の反応**: 音の半径内にいる巡回/捜索状態の敵が、着地点を目標とする捜索状態に遷移する。
  **追跡中（chase）の敵には効かない**。万能脱出ボタン化を防ぎ、「追われる前に仕込む/
  視線を切った直後に使う」という駆け引きに寄せるための制約
- **3D 表現**: 小球メッシュの飛翔＋着弾音（Tone.js）

## 4. 索敵 UI（音＋画面端インジケータ）

- **音**: 敵の足音を距離減衰＋ステレオパンで定位。発見時のスティング音、捜索中の唸り声
- **視覚**: 敵の状態変化時に画面端へ方向付きマーカーを表示 — 発見「!」（赤）、
  捜索/見失い「?」（黄）。プレイヤーとの相対角度から画面端の表示位置を算出する
- **HUD**: 石の残数表示を追加

## 5. 難易度バランスの再設計

- **敵速度に上限**: プレイヤー速度の約 0.9 倍でキャップ。「直線では走り勝てないが、
  視線を切れば撒ける」を全レベルで保証する
- **レベル上昇の軸を変更**: 速度ではなく「敵の数・視野の広さ（`SIGHT_RANGE`/視野角）・
  捜索の粘り強さ（`SEARCH_DURATION`）」を伸ばす
- 新定数は `GAME_BALANCE.enemy` に集約: `FOV_ANGLE`, `SIGHT_RANGE`,
  `LOSE_SIGHT_GRACE`, `SEARCH_DURATION`, `STONE_INITIAL_COUNT`, `STONE_MAX_COUNT`,
  `STONE_THROW_RANGE`, `STONE_NOISE_RADIUS` 等

## 7. テスト方針

- **ユニットテスト（TDD で先行）**: 視線判定（直線廊下で見える/壁越しに見えない/視野角外は
  見えない）、状態遷移（発見→追跡→喪失→捜索→復帰、隠れ場所の有効/無効）、石のロジック
  （所持数、着地、対象状態の敵のみ反応）
- **既存テスト**: 検知仕様の変更に伴うパリティ調整
- **E2E**: 現状の smoke テストを維持。R3F コンポーネントは従来どおり jsdom テスト対象外
