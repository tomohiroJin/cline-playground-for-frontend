# 迷宮の残響 — 面白さ追求 Phase 1 設計書（物語の背骨＋残響書庫）

- 作成日: 2026-06-26
- 対象ゲーム: `src/features/labyrinth-echo/`
- ステータス: 設計承認待ち → 実装計画（writing-plans）へ

## 1. 背景と全体ビジョン

### 1.1 解決したい課題（ユーザー実感）

1. **簡単すぎる** — 初回プレイから歯ごたえが薄い。
2. **ストーリーが単調** — 163件のイベントが個別完結で、縦軸の物語がない。
3. **ストーリーに重厚さが足りない** — 明示的なロア（世界設定）がほぼ無い。
4. **クリア後の周回でわくわくしない** — 周回しても世界・自分・迷宮が変化しない。

### 1.2 統合コンセプト「残響深度（Echo Depth）」

`MetaState` に **`echoDepth`** という単一のメタ変数を導入し、これを背骨として
以下4軸を同時に駆動する。テーマは **「過去の探索者たち」**。

| 不満 | 残響深度が駆動する解決軸 |
|---|---|
| 物語が単調・浅い | 生還ごとに「先人の残響」（手記・最期・証言の断片）が解禁。名前を持つ先人ごとの物語アークに束ねられ、新画面「残響書庫」で読み解く |
| 周回でわくわくしない | 書庫が周回で埋まり、深度の節目で世界の見え方が変わる（同じ血痕が「あの先人の最期」と分かる） |
| 簡単すぎる | ①ベース難易度の再調整（Phase 2）＋②深度連動 NG+ エスカレーション（Phase 3） |
| 自分が変わらない | 深度ごとに先人の「残響継承」を排他選択（Phase 4） |
| 迷宮が変質しない | 高深度で第6階層「残響の回廊」＋真エンディング解禁（Phase 5） |

### 1.3 フェーズ分割（各フェーズ独立で main マージ可能）

| Phase | 内容 | 解決する不満 |
|---|---|---|
| **1（本書）** | 物語の背骨＋残響書庫。`echoDepth` 導入／先人の物語アーク／書庫画面／既存イベントへの断片織り込み | 単調・重厚さ |
| 2 | ベース難易度の再調整＋バランス契約テスト | 簡単すぎ |
| 3 | NG+ 残響エスカレーション（侵蝕強化＋「残響の亡霊」敵性イベント） | わくわく・難易度 |
| 4 | 残響継承（先人レガシーの排他選択ビルド） | 自分が変わる |
| 5 | 第6階層＋真エンディング | 迷宮の変質 |

**本書は Phase 1 のみを対象とする。** Phase 2-5 は各自 spec→plan→実装サイクルを持つ。

## 2. Phase 1 スコープ

### 2.1 やること

- `MetaState` に `echoDepth: number` と `fragments: readonly string[]` を追加。
- 先人キャスト5名と残響断片 19件のデータを定義。
- 真相レイヤー4段のテキストを定義。

- 画像アセットの必須化（テキスト/絵文字フォールバックで完結。画像は任意・後日差し替え）。

## 3. 物語の背骨

### 3.1 世界の前提（重厚さの核）

迷宮は **「忘れたくない者」を呼び、その記憶を残響として永久保存する古い意思体**。
先人たちは死んだのではなく、最期の想い・後悔・発見が壁に焼き付いて**残響**になった。
主人公が何度も戻ってこられるのは、ローグライトの「死と再挑戦」が**設定上も正史**であり、
主人公自身が迷宮の残響記憶に支えられているから。
既存 `e049`（迷宮の核「帰りたいか」）の問いが、深度を上げると別の意味に見えてくる。

### 3.2 先人キャスト（残響の主＝物語アーク）

各先人は複数の断片を持ち、周回で集めると人物像と最期が浮かび上がる。
既存イベントの種に意味的に接続する。

| ID | 先人 | 出没階層 | 物語 | 真相レイヤー | 接続する既存の種 |
|---|---|---|---|---|---|
| `p_lian` | 写本師リアン | 1-2 | 全てを記録しようとし、情報に溺れて檻の中で餓死 | 1 | e003(白骨と檻)・e005(掲示板)・e035(手書き地図) |
| `p_twins` | 双子 カイとノア | 2-3 | 二人で挑み、片方がもう片方を背負い続けた別離 | 1-2 | e007(鏡)・e035(血痕) |
| `p_galen` | 地図屋ガレン | 3-4 | 歪んだ幾何を地図化しようとして精神崩壊 | 2-3 | 深淵の間・忘却の底（空間の歪み） |
| `p_elna` | 守人エルナ | 4-5 | 最深部に到達し、自ら「留まる」ことを選んだ | 3 | e049(迷宮の核の声) |
| `p_first` | 始まりの探索者 | 5 | 迷宮がなぜ生まれたか、主人公との繋がり | 4 | 真ENDへの布石（Phase 5） |

#### 断片数の内訳（合計19件・契約テストの基準値）

| 先人 | 断片数 | `depthGate` の目安 |
|---|---|---|
| `p_lian` | 4 | 0,1,1,2 |
| `p_twins` | 4 | 1,2,2,3 |
| `p_galen` | 4 | 2,3,3,4 |
| `p_elna` | 4 | 3,4,4,5 |
| `p_first` | 3 | 4,5,6 |

合計 **19件**。`depthGate` は出現可能になる最小 `echoDepth`。各先人の最初の断片は
比較的浅い深度で出会え、最後の断片ほど深い深度を要する（物語が段階的に深まる）。

### 3.3 真相が降りてくる段階（echoDepth 連動）

| 深度 | 解禁される真相レイヤー |
|---|---|
| 1–2 | 先人は実在した。断片で人となり・最期が分かる |
| 3–4 | 迷宮は「忘れたくない者」を呼ぶ。残響＝記憶保存の仕組みと判明 |
| 5–6 | 主人公自身がなぜ戻れるのか——主人公も残響の一部 / 始まりの探索者の継承者 |
| 7+ | （Phase 5 真END）迷宮を解放するか、残響として留まるかの最終選択 |

**ねらい**: 同じ血痕・遺体・鏡が、周回を重ねるごとに「あの先人の最期」として
意味を持ち直す。これが「単調さの解消」と「周回のわくわく」を同時に生む。

## 4. 断片の解禁メカニズム

### 4.1 echoDepth のライフサイクル

- **生還（escape）するたびに `echoDepth` が +1**。Phase 1 の上限は **6**（Phase 5 で 7+ へ拡張）。
- `echoDepth` が深い先人/真相レイヤーの解禁ゲートになる → 「物語を進める＝クリアを重ねる」が直結。
- 既存セーブ互換: `migrate-meta-state` に欠損補完（`echoDepth:0, fragments:[]`）。

### 4.2 断片データモデル

```ts
interface EchoFragment {
  readonly id: string;            // 例 "f_lian_1"（一意）
  readonly predecessorId: string; // 例 "p_lian"
  readonly order: number;         // アーク内順序（1始まり連番）
  readonly depthGate: number;     // この echoDepth 以上で出現可能（0-6）
  readonly floors: readonly number[]; // 探索中発見の対象フロア
  readonly title: string;         // 書庫の断片見出し
  readonly body: string;          // 断片本文（重厚な散文）
}

interface Predecessor {
  readonly id: string;
  readonly name: string;
  readonly icon: string;          // 絵文字（画像は任意で後日）
  readonly color: string;
  readonly floors: readonly number[];
  readonly summary: string;       // 全断片収集で解禁される人物総括
  readonly truthLayer: number;    // 1-4
}
```

### 4.3 解禁の2チャネル

**① 探索中の発見（残響イベント `tp:"echo"`）＝ 主軸**

- 先人の痕跡（壁の手記・遺品・残響の幻影）に遭遇する物語内在的な発見。
- 出現条件: `echoDepth >= fragment.depthGate` かつ 現在フロアが `floors` に合致。
  既存 `pickEvent` の `fl`／`metaCond` 機構を流用。
- 選択肢例: 「読み解く（精神を少し消費して断片獲得）」「触れずに進む（後周回で再出現）」。
- 獲得時に outcome の `frag` フラグで `meta.fragments` へ追加。
- **効果**: echoDepth が上がるとイベントプール自体が入れ替わり、同じ階層でも先人の残響が
  新たに混ざる＝単調さの直接解決。

**② 生還報酬（セーフティネット）＝ 取りこぼし救済**

- RNG で残響イベントに当たらなくても進めるよう、生還時に
  **現在の解禁済みティアで未読の断片を1つ**付与。
- 「クリアしたのに物語が進まない」事故を防止。
- 選択ロジック: `depthGate <= echoDepth` かつ未収集の断片を `order`→`predecessor` 順で最小1件。

### 4.4 既存メタイベントの統合

- 既存の自己残響イベント（e157-e163: 前回の自分の血痕等）は「自分自身の残響」として温存。
- 今回追加するのは「先人の残響」。両者の併存で「残響」テーマが厚みを増す。

## 5. 残響書庫（読み解き UI）

### 5.1 位置づけ

既存3コレクション画面（Unlocks / Titles / Records）と同じパターンの4枚目として追加。
タイトル/メニューから遷移（`phase: "archive"` を追加）。
`Particles`／`Section`／`setPhase`／`meta` の既存 props を流用。

### 5.2 画面構成

```
┌─ 残響書庫 ──────────────────────────┐
│ 真相の深度 ●●●○○○  (echoDepth 3/6)   │  ← 真相レイヤー進捗バー
│ 収集 12/19 断片                      │
├──────────────────────────────────┤
│ 先人カード一覧（縦スクロール）          │
│ ┌────────────┐ ┌────────────┐      │
│ │ 写本師リアン │ │ ？？？      │      │  ← 未発見はシルエット
│ │ ▣▣▣□ 3/4   │ │ □□□ 0/3    │      │
│ └────────────┘ └────────────┘      │
└──────────────────────────────────┘
```

### 5.3 インタラクション

- **先人カード**: 1枚目の断片を得るまで「？？？」シルエット。発見後に名前・アイコン・
  色・断片スロットを開示。
- **断片スロット**: 収集済み＝クリックで本文を全画面リーダー表示。未収集＝
  「第N層で出会う残響」等のヒント付きロック表示。
- **人物総括**: その先人の全断片を集めると `summary` が解禁され、カードに記章を付与。
- **真相レイヤーバー**: `echoDepth` で 1→4 のレイヤーが点灯。点灯時に「迷宮の真相」
  テキストを1段開示。

### 5.4 演出（既存資産で完結）

- 断片リーダーは既存 `use-text-reveal`（テキスト逐次表示）を流用し、手記が「滲み出る」演出。
- 新規画像アセットは Phase 1 では必須にしない（先人アイコンは絵文字/CSS、断片はテキスト主体）。

- echo イベントの重み微調整・読み解きコスト（mn消費量）の最終調整は Phase 2 の
  バランスパスで実施。
