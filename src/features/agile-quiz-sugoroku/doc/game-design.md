# ゲームデザイン

## ゲームフロー

### 通常ゲーム

```
タイトル画面（スプリント数・難易度選択）
  → ストーリー（ノベルゲーム風）
    → スプリント開始画面（キャラクターナラティブ）
      → クイズ画面（7イベント × N スプリント）
        → 振り返り画面（キャラクターナラティブ）
          → [次のスプリントへ / 最終スプリント後は結果画面へ]
    → エンディングストーリー（共通 + チームタイプ別エピローグ）
      → 結果画面（グレード発表演出 → チームタイプ → エンジニアタイプ → ジャンル分析 → 不正解レビュー）
```

### 画面遷移（GamePhase）

| フェーズ | 画面 | 説明 |
|----------|------|------|
| `title` | TitleScreen | スプリント数・難易度選択、各モードへの入口 |
| `story` | StoryScreen | ノベルゲーム風ストーリー |
| `sprint-start` | SprintStartScreen | スプリント開始演出 + キャラナラティブ |
| `game` | QuizScreen | クイズ出題（すごろくボード・キャラリアクション付き） |
| `retro` | RetrospectiveScreen | スプリント振り返り + キャラナラティブ |
| `ending` | StoryScreen | エンディングストーリー |
| `result` | ResultScreen | 最終結果（グレード・タイプ判定・分析） |
| `guide` | GuideScreen | 遊び方 & チーム紹介 |
| `study-select` | StudySelectScreen | 勉強会モード ジャンル選択 |
| `study` | StudyScreen | 勉強会モード 学習 |
| `achievements` | AchievementScreen | 実績一覧 |
| `history` | HistoryScreen | プレイ履歴・成長グラフ |
| `challenge` | QuizScreen | チャレンジモード（1ミス即終了） |
| `challenge-result` | ChallengeResultScreen | チャレンジ結果・ハイスコア |
| `daily-quiz` | DailyQuizScreen | デイリークイズ（5問） |

## 操作方法

- **マウスクリック**: 選択肢を選択
- **キーボード**: A/B/C/D または 1/2/3/4 で選択、Enter/Space で次へ
- **ストーリー画面**: Enter/Space で次の行へ、Escape でスキップ

## スプリント構成

### スプリント数の選択

タイトル画面でスプリント数を選択可能: **1 / 2 / 3 / 5 / 8**（フィボナッチ数列ベース）。

### 1スプリントの構成（7イベント）

各スプリントは以下の7つのイベント（工程）で構成:

| 順番 | イベント | アイコン | 不正解時の負債 |
|------|---------|---------|--------------|
| 1 | プランニング | 📋 | なし |
| 2 | 実装（1回目） | ⌨️ | +5 |
| 3 | テスト（1回目） | 🧪 | +3 |
| 4 | リファインメント | 🔧 | +4 |
| 5 | 実装（2回目） | ⌨️ | +5 |
| 6 | テスト（2回目） | ✅ | +3 |
| 7 | スプリントレビュー | 📊 | なし |

### 技術的負債メカニクス

- 実装・テスト・リファインメントの工程で不正解すると、イベントに応じた負債ポイントが加算
- プランニングとレビューでは負債は発生しない
- 負債が蓄積すると緊急対応イベントの発生確率が上昇

### 緊急対応イベント

- **発生条件**: 2スプリント目以降に確率で発生（緊急対応マスが通常イベントに差し替わる）
- **発生確率**: `基本確率(10%) + 負債 × 0.4%`（最大50%）
- **挿入位置**: 2番目〜5番目のイベントのいずれかにランダム挿入
- **出題ジャンル**: インシデント、SRE

## ゲームモード

| モード | 説明 |
|--------|------|
| 通常ゲーム | 1〜8スプリント（選択可能）× 7イベント、難易度別制限時間、技術的負債・緊急対応あり |
| 勉強会モード | ジャンル選択（キャラクター別絞り込み対応）→ タイマーなし学習 → 即時解説表示 → 結果分析 |
| チャレンジモード | 1ミス即終了のサバイバルモード。正解数がスコア、ハイスコア保存対応 |
| デイリークイズ | 毎日5問の日替わりクイズ。日付ベースのシード値で同じ日は同じ問題。連続参加ストリーク表示 |

### 難易度選択

| 難易度 | 制限時間 | 負債倍率 | 特記 |
|--------|----------|----------|------|
| Easy | 20秒 | x0.5 | 初心者向け |
| Normal | 15秒 | x1.0 | 標準設定 |
| Hard | 10秒 | x2.0 | 上級者向け |
| Extreme | 8秒 | x3.0 | 1ミスで負債+15 |

Hard 以上はグレード計算にボーナス係数（1.1）が適用。

### 勉強会モード — キャラクター別ジャンル絞り込み

キャラクターを選ぶと、そのキャラクターの専門分野に関連するジャンルが自動で選択されます。

| キャラ | 役職 | 関連ジャンル |
|--------|------|-------------|
| タカ | ビジネスオーナー | アジャイル、スクラム、チーム、リリース |
| イヌ | プロダクトオーナー | スクラム、バックログ、見積もり、アジャイル |
| ペンギン | スクラムマスター | スクラム、アジャイル、チーム、見積もり |
| ネコ | フルスタックエンジニア | 設計原則、デザインパターン、プログラミング、データ構造、リファクタリング |
| ウサギ | QAエンジニア | テスト、コード品質、CI/CD、SRE、インシデント |
