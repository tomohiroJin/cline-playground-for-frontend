# 機能詳細

## レスポンシブデザイン

- 内部解像度 600x1200 固定、CSS スケーリングで画面サイズに自動フィット
- アスペクト比 1:2 を維持し、小画面では幅いっぱい、大画面では max-width で制限

## カウントダウン演出

- ゲーム開始時に「3 → 2 → 1 → GO!」のカウントダウン表示
- カウントダウン中はパック・マレットの移動停止

## 画面シェイク

- ゴール時に強シェイク、強打ヒット時に弱シェイク（CSS transform で実装）

## パック速度ビジュアル

速度に応じてパックの色・トレイル長・グローが変化:
- 通常: 白色
- 高速: 黄色
- 超高速: 赤色＋スピードライン

## サウンド

- **BGM**: Web Audio API でゲーム中にループ BGM を動的生成（フィーバー時テンポアップ）
- **SE**: ヒット音が速度に応じて変化、ゴール時のファンファーレ/下降音

## 音量設定

- BGM / SE を個別にスライダーで調整（0〜100%）
- ミュートボタン、設定は localStorage に永続化

## ポーズ機能

- ゲーム中いつでも一時停止可能
- ポーズ中: 「Resume」「Restart」「Back to Menu」を選択

## リザルト画面

- 勝利時: 紙吹雪エフェクト
- 試合統計のカウントアップアニメーション表示（ヒット数、最高パック速度、アイテム取得数、セーブ数、試合時間）
- MVP スタッツのハイライト表示
- 「REPLAY」ボタンで同条件で即再戦
- 新規実績解除時のポップアップ表示
- 難易度オートアジャスト提案の表示

## 実績システム

| 実績 | 条件 |
|------|------|
| 初勝利 | 初めて CPU に勝利 |
| パーフェクト | 無失点で勝利 |
| 3連勝 | 3回連続で勝利 |
| 5連勝 | 5回連続で勝利 |
| ハードモード制覇 | Hard で勝利 |
| フィールドマスター | 全フィールドで勝利 |
| 大逆転 | 3点差以上から逆転勝利 |
| スピードデーモン | パック速度 15 以上を記録 |
| アイテムコレクター | 全種類のアイテムを使用 |

- タイトル画面から実績一覧を確認可能

## 難易度オートアジャスト

- 3連敗で難易度を下げる提案、3連勝で上げる提案を自動表示

## フィールド/アイテムアンロック

初期利用可能: Original, Wide フィールド + Split, Speed, Invisible アイテム

| アンロック対象 | 条件 |
|--------------|------|
| Pillars | 3勝 |
| Zigzag | 5勝 |

## デイリーチャレンジ

| パターン | 特殊ルール | 勝利スコア |
|---------|-----------|-----------|
| スピードラッシュ | パックが高速 | 3点 |
| タイニーゴール | ゴールが狭い | 3点 |

| ビッグマレット | マレットが巨大 | 3点 |

| マスターチャレンジ | ゴールが狭い | 7点 |

## チュートリアル

初回プレイ時に4ステップのインタラクティブチュートリアルを表示:
1. マレットを動かしてみよう（マウス/タッチ操作）
2. パックを打ち返そう
3. ゴールに入れよう
4. アイテムを使おう

- スキップ可能、完了フラグは localStorage に保存
- タイトル画面の `?` ボタンからいつでも再確認可能

## ゲーム中ヘルプ

- タイトル画面の「？」ボタンから手動で表示
- 全6種アイテムのアイコン・名前・説明、現在のフィールド特徴を表示
- タップで閉じてゲーム再開
- ※ ゲーム中の自動ヘルプ表示は無効化済み

## 2P 対戦モード

- タイトル画面の「2P 対戦」ボタンからローカル対戦を開始
- キャラクター選択画面で 1P / 2P が好きなキャラを選択可能（ミラーマッチ対応）
- フィールド・勝利スコアも自由に設定可能

### 操作方法

| プレイヤー | キーボード |
|-----------|-----------|
| 1P | 矢印キー（↑↓←→） |
| 2P | WASD キー |

- ポーズ: Escape / P（共通）

### 2P 対戦の特徴

- CPU AI を使わず、2 人のプレイヤーが同時にマレットを操作
- 2P 対戦では実績・アンロック・ストーリー進行は無効
- リザルト画面で「1P Win!」/「2P Win!」を表示
- 「キャラ選択に戻る」ボタンで連戦が可能

## ペアマッチ（2v2）モード

- タイトル画面の「ペアマッチ」ボタンから開始
- チーム構成: 人+CPU vs CPU+CPU（デフォルト）/ 人+人 vs CPU+CPU
- P2 は CPU/人間をセグメントコントロールで切り替え可能
- P3/P4 はキャラクター選択可能（アンロック済みキャラから選択）
- チーム1（青）/チーム2（赤）の色分けで視覚的に区別

### ペアマッチの特徴

- 味方同士は同じ半面全体を自由に移動可能（上下 2 分割ゾーン）
- ゴールサイズが拡大（通常フィールド: 240px、Wide: 280px）
- マレット間衝突判定あり（重なり防止、ゾーン制約の再適用）
- パックスタック脱出（0.5 秒間低速でフィールド中央方向に射出）
- キャラ別 AI プロファイル（下表参照）
- ally CPU には味方補正（aggressiveness 上限 0.5 で守備的に行動）

### キャラクター別 AI プロファイル（S6-3 で深化済み）

| キャラ | バッジ | aggressiveness | 守備スタイル | 打ち返し | 反応速度 | 役割 | 特徴 |
|--------|-------|---------------|------------|---------|---------|------|------|
| ヒロ | ⚔️ | 0.7（攻撃的） | aggressive | ストレート(-0.3) | 50ms | attacker | 攻撃型エース、直線で撃ち抜く |
| ミサキ | ⚖️ | 0.5（バランス） | wide | バウンス(0.5) | 80ms | balanced | 壁バウンスで翻弄する |
| タクマ | 🛡️ | 0.2（守備的） | center | ストレート(-0.5) | 30ms | defender | 鉄壁の守護神、ゴール前で素早く反応 |
| ユウ | ⚖️ | 0.4（やや守備的） | wide | ややバウンス(0.2) | 40ms | balanced | adaptability で試合展開に適応 |
| ルーキー | ⚖️ | 0.3（消極的） | center | なし(0) | 200ms | balanced | 反応が遅く癖がない素直な動き |
| レギュラー | ⚖️ | 0.5（バランス） | wide | わずか(0.1) | 100ms | balanced | バランスの良い中堅 |
| エース | ⚔️ | 0.6（やや攻撃的） | aggressive | バウンス(0.3) | 50ms | attacker | 積極的にバウンスショットを狙う |

#### S6-3 で追加された AI パラメータ

- **defenseStyle**: パック相手陣地時のポジション（center/wide/aggressive）。パック自陣時は aggressiveness が優先
- **deflectionBias**: マレット・パック衝突時の打ち返し角度バイアス（-1.0 ストレート 〜 1.0 バウンス、最大 ±30°）
- **reactionDelay**: パック方向転換後のターゲット再計算遅延（ms）。通常時は reactionDelay*3 で定期再計算
- **teamRole**: 2v2 でのチーム内役割（attacker: +0.3, defender: -0.3, balanced: ±0）
- **sidePreference**: 横方向のホームポジションオフセット（-1.0 左 〜 1.0 右）
- **動的 teamRole 切り替え**: スコア差 2 点以上で adaptability に応じて aggressiveness を動的調整
- **ally reactionDelay キャップ**: ally CPU は reactionDelay 120ms 以下に制限（味方性能保証）
- VsScreen で 4 キャラ表示 + チームラベル + P2 操作タイプラベル
- ResultScreen で 4 体立ち絵 + チーム間⚡区切り
- 「同じ設定でリプレイ」「チーム設定に戻る」の導線
- `prefers-reduced-motion` 対応（アニメーションスキップ）

## 画面トランジション

- menu → game → result の各遷移にフェードトランジション（フリー対戦）
- menu → characterSelect → vsScreen → game → result（2P 対戦）
- menu → teamSetup → vsScreen → game → result（ペアマッチ）
- menu → stageSelect → preDialogue → vsScreen → game → postDialogue → result（ストーリーモード）

## 中断確認ダイアログ（S8-2）

- ゲーム中のメニューボタンで全モード共通の確認ダイアログを表示
- CSS transition アニメーション: opening 200ms ease-out / closing 150ms ease-in
- 状態遷移: `closed → opening → open → closing → closed`
- `prefers-reduced-motion` 対応（transition-duration: 0ms）
- closing フェーズ中は `pointer-events: none` で操作ブロック
- フォーカストラップ: Tab / Shift+Tab でダイアログ内ボタン間をループ
- `onTransitionEnd` + フォールバック `setTimeout` の二重保証

## ストーリーモード

### 画面構成

- **ステージ選択画面**: チャプターセクション分けで表示。解放済みステージにカード、クリア済みにチェックマーク
- **チャプタータイトルカード**: 新チャプター開始時に表示（例: 第2章「はじめての大舞台」）
- **ダイアログオーバーレイ**: キャラアイコン＋名前＋セリフ（1文字ずつ表示、スキップ可能）
- **VS 画面**: 対決演出（2秒自動遷移、フェードイン/アウト）
- **勝利カットイン**: チャプターフィナーレ勝利時に表示

### チャプター構成（S8-3）

| 章 | ステージ数 | 新キャラ | テーマ |
|----|----------|---------|--------|
| 第1章 | 3（1-1〜1-3） | ヒロ、ミサキ、タクマ、ユウ | 部内での成長 |
| 第2章 | 4（2-1〜2-4） | カナタ、リク、シオン | 地区大会 |

詳細は [ストーリーモード](./story-mode.md) を参照。

### キャラクターリアクション

- ゴール時に得点側・失点側キャラの吹き出しを表示
- 勝利/敗北時に専用リアクション

### バランス調整

各ステージに個別の AI 設定が適用される。詳細は [ストーリーモード](./story-mode.md) を参照。

### ヒットストップ・スローモーション

- 強打（パック速度 > 8）時に3フレームのヒットストップ＋衝撃波演出
- ゴール時に 400ms のスローモーション＋ビネット効果

## S9 で追加された機能（2026-04-19）

### デザイントークン基盤（S9-D）

- `src/features/air-hockey/core/design-tokens.ts` で air-hockey 固有の `AH_TOKENS` を定義
- 既存グローバルトークン（`src/styles/tokens/`）を参照する方針で重複定義を防止
- アニメーション原則: `enter` (200ms) / `exit` (150ms) / `emphasis` (300ms) の 3 種のみ
- `hooks/useReducedMotion.ts` で `prefers-reduced-motion` 対応を一元化

### アクセシビリティ強化（S9-V-2）

- `components/CanvasLiveRegion.tsx` を追加
- Canvas で描画されるゲーム状態（スコア・ゲームパッドトースト・勝敗）を `aria-live` で DOM に露出
- スクリーンリーダーでゲーム進行を把握可能
- スコア変化は `polite`、ゲーム終了は `assertive` で通知

### i18n 準備（S9-V-3）

- `core/i18n-strings.ts` で Canvas 描画される日本語・英語文字列を一元管理
- `AH_STRINGS.common / player / playerAria / game` の 4 カテゴリ
- 実翻訳は未実施（将来の Jotai atom 注入に備えた構造のみ）

### Canvas フォント統一（S9-V-4）

- `core/canvas-fonts.ts` で `FONT_STACK_BODY` / `FONT_STACK_HEADING` / `CANVAS_FONTS` を提供
- DOM と Canvas の書体を統一（Inter + Noto Sans JP + 絵文字フォールバック）
- `renderer.ts` / `ui-renderer.ts` の Arial ハードコードを全面排除

### VsScreen 2v2 モバイルレスポンシブ（S9-A1）

- styled-components ベースで 600px 以下は縦積みレイアウト
- 立ち絵サイズは `max-width: min(45vw, 240px)` で動的
- VS テキストは `clamp()` で流動タイポ
- `useReducedMotion` フックでアニメーション制御を一元化

### ゲームパッドトースト通知の強化（S9-A2）

- ゲーム画面 Canvas 上にトースト描画（既存実装を `CanvasLiveRegion` と連携）
- 接続/切断が支援技術にも読み上げられる

### 操作タイプ表示統一（S9-A3）

- 内部語彙: `p2` / `p3` / `p4`（スロット識別）
- 表示文字列: `2P` / `3P` / `4P`（日本ゲーム慣習）
- `aria-label`: 「プレイヤー2（WASD/タッチ）」など詳細説明
- TeamSetupScreen の人間ボタンは `aria-pressed` / `aria-disabled` 付与

### VRT（Visual Regression Testing）基盤（S9-V-1）

- `e2e/air-hockey-visual.spec.ts` で 4 viewport × 主要画面のスクショ比較
- `reducedMotion: 'reduce'` + アニメーション強制停止で flaky 対策
- `375x667 / 393x852 / 768x1024 / 1280x720` の 4 解像度
- `assertNoHorizontalOverflow` ヘルパで `scrollWidth <= innerWidth` 検証

### パフォーマンス計測基盤（S9-C1）

- `core/perf-probe.ts` で `physics` / `ai` / `render` / `total` セクション別計測
- p50 / p95 / p99 + TBT（Total Blocking Time）+ Heap + DPR 集計
- `?perf=1` URL パラメータで有効化
- `window.__ahPerfSnapshot` に snapshot を expose（E2E 用）
- `e2e/air-hockey-perf.spec.ts` + `npm run test:e2e:perf`

### パフォーマンス最適化（S9-C2 一部）

- パーティクル上限を 2v2 時に半減、フィーバー時は 1.5 倍許容
- マレット-マレット衝突判定に AABB 事前除外（`Math.sqrt` 呼び出し削減）

### シナリオ補強（S9-S）

第 2 章ストーリーの整合性向上:
- アキラの準々決勝勝利を Stage 2-3 preDialogue に追加
- リクの台詞を Stage 2-4 preDialogue に追加（詳細は `story-mode.md`）
- ユウの勝利祝辞を Stage 2-4 postWin に追加
- カナタ戦の会話フロー再構成
- 冗長な `expression: 'normal'` 指定の削除
