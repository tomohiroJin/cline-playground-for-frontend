# Phase 2: キャラクター図鑑・ゲーム内紹介 — 実装計画

## 概要

Phase 1 で整備した立ち絵・表情差分・キャラクターデータを活用し、プレイヤーがキャラクターの魅力をより深く知れる「キャラクター図鑑」機能を実装する。
ストーリー進行に伴うアンロック体験により、ゲームの収集要素と没入感を強化する。

## 目標

- キャラクターの世界観を楽しめる図鑑画面の提供
- ストーリー進行に連動した段階的なキャラクター解放体験
- リザルト画面でのキャラ表情差分による感情表現の強化
- 図鑑コンプリートへのモチベーション創出

## 前提条件

- Phase 1 成果物がすべて完了済み（ビジュアル強化・画像アセット・データ層拡張）
- 参照ドキュメント:
  - `src/features/air-hockey/doc/world/character-profiles.md`（D-02: キャラ設定）
  - `src/features/air-hockey/doc/world/world-setting.md`（D-01: 世界観）
  - `.docs/ah-20260311-02/spec.md`（Phase 1 仕様書 — データ層・アセット仕様）

## 設計制約

- 既存のゲームロジック（物理エンジン・AI・アイテム等）には変更を加えない
- 新コンポーネントは既存のスタイルパターン（インラインスタイル + CSS-in-JS）に準拠
- 図鑑データは `character-profiles.md` の情報をコード化したもの（第1章分 + フリー対戦キャラ）
- 第2章キャラ（リク・カナタ・シオン）の図鑑エントリは Phase 2 のスコープ外（データ構造のみ対応可能にする）
- アンロック状態は既存の `localStorage` 永続化パターンに準拠
- パフォーマンス: 図鑑画面の画像は遅延読み込み（既存の `useImagePreloader` を活用）

## タスク一覧

### P2-00: 画像アセット準備

**目的**: Phase 2 で使用する画像アセットの確認と、未作成の `yuu-vs.png` を作成する

> **重要**: Phase 2 は Phase 1 の既存アセットをほぼすべて再利用します。
> AI 画像生成は不要で、既存立ち絵からのトリミングのみが必要です。
> 詳細なプロンプト・手順は `image-generation-guide.md` を参照。

**作業内容**:
1. 既存アセットの存在確認（アイコン 8枚、立ち絵 16枚、VS 7枚、背景 3枚）
2. 既存アセットの品質確認（透過状態、表情差分の明確さ）
3. `yuu-vs.png` の作成
   - 入力: `portraits/yuu-normal.png`（512x1024、透過済み）
   - 出力: `vs/yuu-vs.png`（256x512、透過）
   - 方法: 他の VS 画像と同じトリミング + リサイズ方法で作成
4. `characters.ts` のユウに `vsImage` パスを追加

**成果物**: `public/assets/vs/yuu-vs.png`、更新された `characters.ts`
**完了条件**: 全 8 キャラの VS 画像が揃っている、既存アセットの品質確認完了

### P2-01: データ層整備

**目的**: 図鑑機能に必要な型定義・プロフィールデータ・アンロック条件をコードレベルで整備する

**作業内容**:
1. `types.ts` に図鑑関連の型を追加
   ```typescript
   // キャラクターのプロフィール情報（図鑑表示用）
   type CharacterProfile = {
     characterId: string;         // Character.id と紐づく
     fullName: string;            // フルネーム（例: '蒼葉 アキラ'）
     reading: string;             // 読み（例: 'あおば あきら'）
     grade: string;               // 学年（例: '1年生'）
     age: number;                 // 年齢
     birthday: string;            // 誕生日（例: '4月8日'）
     height: string;              // 身長（例: '165cm'）
     school: string;              // 所属校名（例: '蒼風館高校'）
     club: string;                // 部活名（例: 'エアホッケー部'）
     personality: string[];       // 性格キーワード（例: ['素直', '負けず嫌い', '行動派']）
     quote: string;               // 代表セリフ
     playStyle: string;           // プレイスタイル名（例: 'オールラウンダー'）
     specialMove: string;         // 得意技名（例: 'ライジングショット'）
     specialMoveDesc: string;     // 得意技説明
     description: string;         // キャラクター紹介文（1-2文の短い説明）
   };

   // アンロック条件の定義
   type UnlockCondition = {
     type: 'story-clear';         // 現在はストーリークリアのみ
     stageId: string;             // クリアが必要なステージID（例: '1-1'）
   } | {
     type: 'default';             // 初期状態で解放済み
   };

   // 図鑑エントリ（キャラクター + アンロック条件）
   type DexEntry = {
     profile: CharacterProfile;
     unlockCondition: UnlockCondition;
   };

   // アンロック状態（永続化用）
   type DexProgress = {
     unlockedCharacterIds: string[];
     newlyUnlockedIds: string[];   // 未確認の新規アンロック
   };
   ```
2. `ScreenType` に `'characterDex'` を追加（spec.md S-01 参照）
3. `character-profiles.md` の情報を基に全キャラクターの `DexEntry` データを作成
   - 蒼風館メンバー 5名: アキラ/`player`（default）、ヒロ/`hiro`（stageId: `'1-1'`）、ミサキ/`misaki`（stageId: `'1-2'`）、タクマ/`takuma`（stageId: `'1-3'`）、ユウ/`yuu`（default）
   - フリー対戦キャラ 3名: ソウタ/`rookie`（default）、ケンジ/`regular`（default）、レン/`ace`（default）

**成果物**: 更新された `types.ts`, 新規 `core/dex-data.ts`
**完了条件**: 型エラーなし（`tsc --noEmit`）、既存テストがパス

### P2-02: アンロックシステム

**目的**: ストーリー進行に連動したキャラクター解放ロジックと永続化を実装する

**作業内容**:
1. `core/dex.ts` を新規作成
   - `loadDexProgress()`: localStorage からアンロック状態を読み込み
   - `saveDexProgress()`: localStorage にアンロック状態を保存
   - `resetDexProgress()`: 図鑑進行のリセット
   - `checkNewUnlocks()`: ストーリー進行状態から新規アンロック対象を判定
   - `markAsViewed()`: 新規アンロック通知の既読処理
2. `useCharacterDex` カスタムフックの作成（`hooks/useCharacterDex.ts`）
   - 図鑑の状態管理（ロード・アンロック・通知）
   - ストーリークリア時の自動アンロックチェック
   - 新規アンロック通知の管理

**成果物**: `core/dex.ts`, `hooks/useCharacterDex.ts`
**完了条件**: ユニットテストがパス、ストーリークリア時にアンロックが正しく動作

## リスク管理

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 図鑑画面の画像読み込みによるパフォーマンス低下 | 中 | 遅延読み込み + サムネイルサイズのキャッシュ活用 |
| プロフィールデータと character-profiles.md の乖離 | 中 | データファイルに参照元コメントを記載、テストで整合性チェック |
| AirHockeyGame.tsx のさらなる肥大化 | 中 | useCharacterDex フックで図鑑ロジックを分離、本体への追加は最小限に |
| アンロック状態の localStorage 破損 | 低 | 既存パターンに準じた try-catch + フォールバック処理 |
| タイトル画面のボタン増加によるレイアウト崩れ | 低 | 450x900 内部解像度でのレイアウト検証 |
