## S-02: 入力システム仕様

### キーボード入力（2P 対応）

| プレイヤー | 上 | 下 | 左 | 右 |
|-----------|----|----|----|----|
| 1P | ↑ (ArrowUp) | ↓ (ArrowDown) | ← (ArrowLeft) | → (ArrowRight) |
| 2P | W | S | A | D |

**仕様**:
- 各プレイヤーのキー状態は独立して管理
- 1P と 2P のキー同時押しが可能
- ポーズキー（Escape, P）は共通（どちらのプレイヤーからも利用可能）
- 2P モード以外では WASD は無視される
- キーボード移動速度は既存の `KEYBOARD_SPEED`（6.0）を使用

### マルチタッチ入力

**タッチゾーン定義**:
```
┌─────────────────────┐
│                     │
│   2P タッチゾーン    │ ← Canvas 上半分（y < height/2）
│                     │
├─────────────────────┤ ← 中央ライン
│                     │
│   1P タッチゾーン    │ ← Canvas 下半分（y >= height/2）
│                     │
└─────────────────────┘
```

**仕様**:
- `Touch.identifier` で各タッチポイントを追跡
- 各ゾーンは最初のタッチのみ追跡（2 本目以降は無視）
- タッチ座標は Canvas 座標系に変換
- `touch-action: none` で ブラウザのデフォルトジェスチャーを無効化
- `touchstart`, `touchmove`, `touchend`, `touchcancel` を処理
- 1P モードでは従来のタッチ入力（Canvas 全体が 1P ゾーン）を維持

### 入力モード切替

| gameMode | 1P 入力 | 2P 入力 |
|----------|---------|---------|
| `'free'` | マウス / タッチ / 矢印キー | CPU AI |
| `'story'` | マウス / タッチ / 矢印キー | CPU AI |
| `'daily'` | マウス / タッチ / 矢印キー | CPU AI |
| `'2p-local'` | マウス(下半分) / タッチ(下半分) / 矢印キー | マウスなし / タッチ(上半分) / WASD |

## S-03: ゲームループ 2P 対応仕様

### 変更箇所

ゲームループの 1 フレーム処理は以下の順序:

```
1. 入力取得（1P）
2. 入力取得（2P / AI）  ← ここが変わる
3. 物理演算（変更なし）
4. 衝突判定（変更なし）
5. アイテム処理（変更なし）
6. スコア判定（変更なし）
7. 描画（ラベル変更のみ）
```

**2P モード時の変更点**:
- ステップ 2 で AI の `updateAI()` を呼ばず、2P の InputSource から位置を取得
- 2P マレットの移動制限: Canvas 上半分（y < height/2）に制限（既存の CPU 制限と同じ）
- 2P マレットの速度計算: 前フレーム位置との差分（既存の CPU と同じ方式）

### アイテムシステム

- 2P 対戦でもアイテムは通常通り出現する
- アイテム効果は取得したプレイヤー側に適用される
- アイテムの出現・取得ロジックは変更なし

### 実績・アンロック

- `gameMode === '2p-local'` の場合、以下を無効化:
  - 実績の判定・解除
  - フィールド/アイテムのアンロック
  - ストーリー進行
  - デイリーチャレンジ記録
- 理由: 2P 対戦は対人であり、CPU 相手の実績とは性質が異なる

## S-04: キャラクター選択画面仕様

### 画面レイアウト

```
┌─────────────────────────────────────────┐
│  ← 戻る          2P 対戦                │
├─────────────────────────────────────────┤
│                                         │
│  ┌─────────┐   VS   ┌─────────┐        │
│  │ 1P キャラ│        │ 2P キャラ│        │
│  │  アイコン │        │  アイコン │        │
│  │  名前    │        │  名前    │        │
│  └─────────┘        └─────────┘        │
│                                         │
│  ─── キャラクター選択 ──────────────      │
│                                         │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐                   │
│  │ア │ │ヒ │ │ミ │ │タ │  ← 選択中: 1P   │
│  │キラ│ │ロ │ │サキ│ │クマ│               │
│  └──┘ └──┘ └──┘ └──┘                   │
│                                         │
│  ─── 設定 ──────────────────────         │
│                                         │
│  フィールド: [Original ▼]                │
│  勝利スコア: [5 ▼]                       │
│                                         │
│  ┌─────────────────────────┐            │
│  │     対戦開始！            │            │
│  └─────────────────────────┘            │
│                                         │
└─────────────────────────────────────────┘
```

### 操作仕様

| 操作 | 動作 |
|------|------|
| キャラアイコンタップ（1P 選択中） | 1P のキャラクターを変更 |
| キャラアイコンタップ（2P 選択中） | 2P のキャラクターを変更 |
| 1P/2P パネルタップ | 選択中プレイヤーを切替 |
| フィールド変更 | フィールドを変更（アンロック済みのみ） |
| 勝利スコア変更 | 勝利スコアを変更（1-10） |
| 対戦開始ボタン | VS 画面へ遷移 |
| 戻るボタン | タイトル画面に戻る |

### 表示するキャラクター

- 図鑑でアンロック済みのキャラクターのみ表示
- `unlockCondition.type === 'hidden'` のキャラは非表示（Phase 2 仕様と同様）
- 初期値: アキラ（player）が 1P、未選択キャラなし → 2P はグリッドの 2 番目のキャラを初期選択
- 同キャラ選択可能（ミラーマッチ）

### アニメーション

- キャラ選択時: アイコンのバウンスアニメーション（100ms）
- 選択パネル切替時: ハイライト境界のスライドアニメーション（200ms）
- 対戦開始ボタン: パルスアニメーション（1.5秒周期）

## S-09: 画面遷移フロー

### 2P 対戦の画面遷移

```
menu
  → characterSelect（キャラ選択 + 設定）
    → vsScreen（2P 対決演出）
      → game（2P 対戦）
        → result（2P リザルト）
          → characterSelect（キャラ選択に戻る）
          → game（同設定で再戦）
          → menu（タイトルに戻る）
```

### 既存フローへの影響

- `menu → game → result` のフリー対戦フロー: **影響なし**
- `stageSelect → ... → result` のストーリーフロー: **影響なし**
- `menu → characterDex` の図鑑フロー: **影響なし**

## S-10: Phase 2 残課題仕様

### プロフィールカード閉じるフェードアウト

- 閉じるボタン/背景タップ時に `isClosing` 状態を設定
- `isClosing === true` 時: オーバーレイをフェードアウト（200ms）、カードをスライドダウン + フェードアウト（200ms）
- アニメーション完了後に `onClose` コールバックを実行
- `animationend` イベントまたは `setTimeout` で完了検知

### club フィールドの表示

- プロフィールカードの基本情報セクションに `club`（部活名）を追加
- 「所属」の下に表示: `{school} {club}`
- 既に `CharacterProfile` 型に `club` フィールドは存在するため、表示追加のみ
