# Agile Quiz Sugoroku ブラッシュアップ仕様書

## フェーズ1: 演出強化

### 1.1 すごろくボードビジュアル

**現状**: `TimelineContainer` 内に小さなドットが並ぶだけ
**目標**: すごろくボード風の経路表示でコマが進む感覚を演出

#### 仕様
- 現在のタイムラインを横長のすごろく経路UIに置き換え
- 各マスにイベントアイコンと名前を表示
- 現在のマスにキャラクターアイコン（コマ）を配置
- コマ移動時にスライドアニメーション（300ms ease-out）
- 完了マスは色が変わり、チェックマーク表示
- 緊急対応マスは赤く点滅するアニメーション
- モバイル対応: 横スクロール可能、現在マスが中央に自動スクロール

#### 影響ファイル
- `components/styles/quiz.ts`: `TimelineContainer`, `TimelineItem` のスタイル刷新
- `components/QuizScreen.tsx`: タイムライン部分の JSX 変更

### 1.2 正解/不正解アニメーション強化

**現状**: `ResultBanner` がフェードインするのみ
**目標**: 正解/不正解の瞬間に強いフィードバック

#### 仕様

**正解時:**
- 画面全体に緑の短いフラッシュ（100ms、opacity 0→0.15→0）
- `ResultBanner` がスケール 0.8→1.0 + バウンスアニメーション
- 正解アイコン（チェックマーク）が回転しながら出現
- スコア加算テキストが上方向にフロートして消える（+10pt 的な表現）
- コンボ中は正解アイコンの色がコンボ色に変化

**不正解時:**
- 画面全体に赤の短いフラッシュ（100ms）
- パネルが左右に軽く振動（shake、200ms）
- 不正解アイコンがフェードインしつつ軽くバウンド

**タイムアップ時:**
- 画面にグレーアウトエフェクト（200ms）
- 時計アイコンが縮小しながらフェードアウト

#### 影響ファイル
- `components/styles/animations.ts`: 新規アニメーション定義追加
- `components/styles/quiz.ts`: `ResultBanner` スタイル拡張
- `components/QuizScreen.tsx`: アニメーション制御ロジック追加

### 1.3 コンボ演出強化

**現状**: テキストで「🔥 N COMBO」表示のみ
**目標**: コンボ数に応じた段階的な演出

#### 仕様

| コンボ数 | 演出 |
|----------|------|
| 2-3 | 炎エフェクト、テキストがオレンジに光る |
| 4-5 | 稲妻エフェクト、テキストが紫に光る、パーティクル追加 |
| 6-7 | 虹色エフェクト、テキストが虹色グラデーション、画面端からパーティクル |
| 8+ | 全画面金色エフェクト、「LEGENDARY COMBO」テキスト |

- コンボ継続中はボード上のコマにオーラエフェクト
- コンボ切れ時に「Combo Break...」テキスト + ガラスが割れるような効果音

#### 影響ファイル
- `components/styles/animations.ts`: コンボ段階別アニメーション
- `components/QuizScreen.tsx`: コンボ演出制御
- `audio/sound.ts`: コンボ切れ効果音追加
- `combo-color.ts`: 既存のコンボ色ロジック拡張

### 1.4 タイマー緊迫演出

**現状**: タイマーの色が変わるのみ（赤/黄/青）
**目標**: 残り時間に応じた段階的な緊迫感

#### 仕様

| 残り時間 | 演出 |
|----------|------|
| 8秒以上 | 通常表示 |
| 5-7秒 | タイマー数字が脈動（pulse）、背景がわずかに暗く |
| 3-4秒 | タイマーが大きく脈動、パネル枠が赤く光る、ティック音のピッチが上がる |
| 1-2秒 | 画面全体が赤みがかる、タイマーが激しく振動、心拍のようなティック音 |

- 既存の `TimerValue` の `$pulse` / `$shake` プロパティを活用・拡張
- 背景色変化は `PageWrapper` に CSS 変数でオーバーレイ

#### 影響ファイル
- `components/styles/quiz.ts`: `TimerValue`, `TimerContainer` スタイル拡張
- `components/QuizScreen.tsx`: 残り時間に応じたクラス付与
- `audio/sound.ts`: ティック音のピッチ変化対応

### 1.5 グレード発表演出

**現状**: 結果画面でグレードが静的に表示
**目標**: ドラマチックなグレード発表シーケンス

#### 仕様

1. 結果画面遷移時、最初は暗い画面
2. 「BUILD SUCCESS」テキストがタイプライター風に表示（1秒）
3. ドラムロール効果音（1.5秒）
4. グレードサークルがスケール0→1.2→1.0でバウンス表示
5. グレードに応じた色のパーティクルが放射状に飛散
6. S ランクの場合は金色の特別エフェクト + ファンファーレ音
7. エンジニアタイプカードがスライドインで表示

- 各ステップ間に適切なディレイ（setTimeout/CSS animation-delay）
- ユーザーがクリック/Enter で演出スキップ可能

#### 影響ファイル
- `components/ResultScreen.tsx`: 演出シーケンス制御
- `components/styles/result.ts`: アニメーション追加
- `audio/sound.ts`: ドラムロール・ファンファーレ追加

### 1.6 キャラクターリアクション

**現状**: キャラクターはガイド画面と結果画面のサマリーにのみ登場
**目標**: ゲーム中にキャラが状況に応じたリアクションで盛り上げる

#### 仕様

- 3体（ネコ・イヌ・ウサギ）が常にパネル内フローで表示（コンテンツと被らない）
- アバターには `AQS_IMAGES.characters` のキャラ画像を使用
- 各キャラは均等幅の固定レイアウト（吹き出しの長さでガタつかない）
- 残り時間に応じて段階的にセリフが変化する
- 得意分野（character-profiles.ts の skills）に応じて、残り時間が少ない時にヒントを出す

| 状況 | タイミング | コメント例 |
|------|-----------|-----------|
| 待機中 | 回答前・余裕あり | 「考えるにゃ...」「どれかなワン？」「うーんぴょん」 |
| 残り7秒 | timer <= 7 | 「そろそろ決めるにゃ」「時間あるワン」「考えてぴょん」 |
| 残り5秒(ヒント) | timer <= 5 | 得意分野キャラが問題のタグに関連するヒントを出す |
| 残り3秒 | timer <= 3 | 「急ぐにゃ！」「時間がないワン！」「早くぴょん！」 |
| 正解 | 回答後 | 「にゃいす！」「その通りワン！」「正解ぴょん！」 |
| 不正解 | 回答後 | 「ドンマイにゃ」「次がんばるワン」「惜しいぴょん」 |
| コンボ3+ | 回答後 | 「ノッてるにゃ！」「すごいワン！」「絶好調ぴょん！」 |
| 緊急対応 | イベント | 「障害発生にゃ！」「落ち着くワン！」「冷静にぴょん！」 |

**得意分野とタグのマッピング:**
- ネコ（エンジニア）: design-principles, design-patterns, programming, code-quality, refactoring
- イヌ（PO）: scrum, agile, estimation, backlog, team
- ウサギ（QA）: testing, ci-cd, sre, incident, release

ヒントは正解を直接教えず、ジャンルに関連する一般的なアドバイスを出す。
該当キャラがいない場合（data-structures等）は全員が一般的なコメントを出す。

#### 影響ファイル
- `components/CharacterReaction.tsx`: フロー配置、固定幅、timer/tags対応
- `character-reactions.ts`: ヒントコメント、タグマッピング追加
- `components/QuizScreen.tsx`: フロー配置、timer/tags を CharacterReaction に渡す

## 非機能要件

### パフォーマンス
- アニメーションは CSS transform / opacity のみ使用（レイアウトスラッシュ回避）
- パーティクル数は最大50個に制限
- requestAnimationFrame ベースのアニメーション（JavaScript 制御の場合）

### アクセシビリティ
- アニメーションは `prefers-reduced-motion` メディアクエリで無効化可能
- キーボード操作は既存のものを維持
- 色覚多様性への配慮（色だけでなくアイコン/テキストでも状態を示す）

### テスト
- 新規ロジック（実績判定、難易度計算、履歴管理）は単体テスト必須
- コンポーネントテストは既存パターンに準拠
- アニメーション自体のテストは対象外（視覚確認）
