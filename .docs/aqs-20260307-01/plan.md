# Agile Quiz Sugoroku ブラッシュアップ計画

## 目標

**ユーザーに「面白い」と思ってもらえるゲーム体験を実現する。**

現状のゲームは「教育クイズツール」としての完成度は高いが、「ゲームとして面白い」要素が不足している。
本計画では、ゲーミフィケーション・演出・リプレイ価値の3軸でブラッシュアップを行う。

## 現状分析

### 強み（維持すべき点）
- 306問＋解説付きの豊富なコンテンツ
- 16ジャンルタグによる体系的な分類
- エンジニアタイプ診断のユニーク性
- 技術的負債 → 緊急対応という現実的なメカニクス
- 勉強会モードの学習特化体験
- BGM・効果音の基盤実装済み

### 課題（改善すべき点）

| カテゴリ | 課題 | 影響度 |
|----------|------|--------|
| ビジュアル | 「すごろく」なのにボード表現がない（タイムラインドットのみ） | 高 |
| 演出 | 正解/不正解・コンボ・グレード発表時のアニメーションが弱い | 高 |
| ゲーミフィケーション | 実績・ランキング・アンロック要素がない | 高 |
| リプレイ価値 | 履歴が最新1件のみ、成長の可視化がない | 中 |
| キャラクター活用 | キャラがガイド画面にしか登場しない | 中 |
| タイマー演出 | 残り時間の緊迫感が弱い（色変化のみ） | 中 |
| 難易度 | スプリント数以外の難易度調整がない | 中 |
| ソーシャル | シェアがテキストコピーのみ | 低 |

## フェーズ計画

### フェーズ1: 演出強化（体感の改善）
**目的**: 既存のゲームフローを変えずに、視覚的・聴覚的フィードバックを強化する

1. **すごろくボードビジュアル**: クイズ画面のタイムラインをすごろくボード風UIに刷新
2. **正解/不正解アニメーション強化**: 画面全体のフラッシュ、アイコンバウンス、スコア加算アニメーション
3. **コンボ演出強化**: コンボ数に応じたエフェクト段階（炎→稲妻→虹）
4. **タイマー緊迫演出**: 残り5秒で画面振動、残り3秒で心拍音、背景色変化
5. **グレード発表演出**: ドラムロール → グレード表示のシーケンスアニメーション
6. **キャラクターリアクション**: 正解/不正解時にキャラが一言コメント（吹き出し表示）

### フェーズ2: ゲーミフィケーション（やり込み要素）
**目的**: 繰り返し遊ぶ動機を作る

1. **実績システム**: 条件クリアでバッジ獲得（初クリア、全問正解、10コンボ達成 等）
2. **履歴・成長グラフ**: 過去のプレイ結果を最大10件保存、正答率推移グラフ表示
3. **難易度選択**: Easy（20秒/ヒント付き）/ Normal（15秒）/ Hard（10秒/負債2倍）
4. **チャレンジモード**: 全問正解を目指すサバイバルモード（1ミスで終了）

### フェーズ3: コンテンツ・ソーシャル拡充
**目的**: コンテンツの深みとユーザー間の接点を作る

1. **キャラクターナラティブ**: スプリント開始/終了時にキャラが状況に応じた会話
2. **デイリークイズ**: 毎日5問の日替わりクイズ（日付ベースのシード）
3. **SNSシェア強化**: OGP画像生成、X(Twitter)シェアボタン
4. **ジャンル別ランキング**: ジャンルごとの正答率ランキング表示

## 優先順位

フェーズ1を最優先とする。理由:
- 既存コードへの影響が最小限
- ユーザーの体感改善に直結
- 追加コンテンツ不要で実装可能
- フェーズ2・3の基盤になる

## 技術的制約

- フレームワーク: React + styled-components
- アニメーション: CSS Animation / CSS Transition を基本とする（ライブラリ追加は最小限）
- 音声: 既存の Tone.js ベースを継続
- ストレージ: localStorage（容量制限に注意、履歴保存時）
- バンドルサイズ: 既存比 +50KB 以内を目標
