# Step 2: CPU AI のキャラ個性化 — 実装計画

## 概要

各キャラクターに固有の AI プレイスタイルを定義し、対戦相手としての個性を際立たせる。
現在の AI は難易度パラメータのみで動作が決まるが、キャラクターごとに「戦い方の癖」を持たせることで、
「ヒロと戦うのとミサキと戦うのは全然違う」という体験を実現する。

## 目標

1. **プレイスタイルパラメータの追加**: `AiBehaviorConfig` にスタイル制御パラメータを追加
2. **キャラクター別 AI プロファイル定義**: 各キャラに固有のプレイスタイルを設定
3. **ストーリーモードへの反映**: ステージの AI にキャラ個性を組み込む
4. **フリー対戦キャラの個性付与**: ルーキー/レギュラー/エースにも個性を追加
5. **既存モードの非破壊**: 全テストパス、ゲームバランスの維持

## 設計原則

### キャラクター別プレイスタイル

各キャラクターに以下の「戦い方の癖」を持たせる:

| キャラ | プレイスタイル | 特徴 |
|--------|-------------|------|
| ヒロ | ストレートシューター | 直線的でシンプルな動き。予測に頼らず正面からぶつかる |
| ミサキ | テクニシャン | サイドへの揺さぶりが多い。斜めの角度から打ち込む |
| タクマ | パワーバウンサー | 壁反射を多用。ゴール前に構えて跳ね返す重厚な動き |
| ユウ | アナライザー | プレイヤーの打ち方に適応。序盤は様子見、中盤から本気 |
| ルーキー | ビギナー | 動きが遅く反応が鈍い。中央に留まりがち |
| レギュラー | オールラウンダー | 平均的な動き。癖が少なくバランス型 |
| エース | エリート | 高精度な予測と速い反応。隙のない動き |

### 新規パラメータの設計

`AiBehaviorConfig` に以下を追加:

```typescript
type AiPlayStyle = {
  /** 横方向のターゲットオフセット傾向（-1.0 左寄せ 〜 0 中央 〜 1.0 右寄せ） */
  sidePreference: number;
  /** ターゲット位置の横ブレの振幅（px）— ミサキの揺さぶり用 */
  lateralOscillation: number;
  /** ターゲット位置の横ブレの周期（ms）— 揺さぶりの速さ */
  lateralPeriod: number;
  /** 前後のポジショニング — 0: ゴール前に構える、1: 前に出る */
  aggressiveness: number;
  /** スコア差に応じた適応度 — 高いほど負けているとき強くなる */
  adaptability: number;
};
```

### 既存パラメータとの関係

```
AiBehaviorConfig（既存: 難易度ベース）
  ├── maxSpeed           — 移動速度
  ├── predictionFactor   — 先読み深度
  ├── wobble             — 狙いのブレ
  ├── skipRate           — ミス率
  ├── centerWeight       — 中央吸引
  ├── wallBounce         — 壁反射予測
  └── playStyle（新規: キャラ個性ベース）
        ├── sidePreference      — 横方向の癖
        ├── lateralOscillation  — 揺さぶり幅
        ├── lateralPeriod       — 揺さぶり周期
        ├── aggressiveness      — 前後のポジショニング
        └── adaptability        — スコア差適応度
```

**難易度パラメータ × キャラ個性**:
- 難易度は「どれだけ正確か」を制御
- キャラ個性は「どう動くか」を制御
- 両方を組み合わせることで「easy なミサキは揺さぶるが精度が低い」等を表現

### キャラクター AI プロファイルの管理

キャラクター ID に紐づく `AiPlayStyle` を新規ファイルで定義し、
ゲームループがキャラ ID から AI プロファイルを取得する仕組みを作る。

### Phase S2-1: 型定義・データ構造

**目的**: キャラ個性を表現する型とデータを定義する

#### S2-1-1: AiPlayStyle 型の定義

- `core/story-balance.ts` に `AiPlayStyle` 型を追加
- デフォルト値を持つ `DEFAULT_PLAY_STYLE` 定数を定義（オールラウンダー = 無個性）

#### S2-1-2: AiBehaviorConfig への playStyle 追加

- `AiBehaviorConfig` に `playStyle?: AiPlayStyle` をオプショナルフィールドとして追加
- 既存の `AI_BEHAVIOR_PRESETS` は `playStyle` 未設定（後方互換）
- AI ロジック側で `playStyle ?? DEFAULT_PLAY_STYLE` にフォールバック

#### S2-1-3: キャラクター AI プロファイルの定義

- `core/character-ai-profiles.ts` を新規作成
- キャラクター ID → `AiPlayStyle` のマッピング
- 各キャラの具体的なパラメータ値:

| キャラ | sidePreference | lateralOscillation | lateralPeriod | aggressiveness | adaptability |
|--------|---------------|-------------------|--------------|---------------|-------------|
| ヒロ (hiro) | 0 | 0 | 0 | 0.7 | 0.2 |
| ミサキ (misaki) | 0 | 40 | 2000 | 0.5 | 0.3 |
| タクマ (takuma) | 0 | 0 | 0 | 0.2 | 0.1 |
| ユウ (yuu) | 0 | 20 | 3000 | 0.4 | 0.8 |
| ルーキー (rookie) | 0 | 0 | 0 | 0.3 | 0 |
| レギュラー (regular) | 0 | 10 | 4000 | 0.5 | 0.2 |

### Phase S2-2: AI ロジックの拡張

**目的**: AI の動きにプレイスタイルを反映する

#### S2-2-1: calculateTargetWithBehavior にプレイスタイル反映

- `core/ai.ts` の `calculateTargetWithBehavior` を改修
- `config.playStyle` からプレイスタイルパラメータを取得
- 各パラメータの適用順序:
  1. 基本ターゲット計算（既存ロジック）
  2. `sidePreference` によるX方向オフセット
  3. `lateralOscillation` による横揺さぶり
  4. `aggressiveness` による前後ポジション調整
  5. `adaptability` によるスコア差対応

#### S2-2-2: 揺さぶり（lateralOscillation）ロジック

- ターゲットの X 座標に `sin(now / lateralPeriod) * lateralOscillation` を加算
- 周期的にターゲットが左右に揺れることで、パックの打ち返し角度が変わる
- `lateralOscillation = 0` の場合は無効（ヒロ、タクマ）

#### S2-2-3: アグレッシブネス実装

- ターゲットの Y 座標を `aggressiveness` で制御
  - `0`: ゴール近く（Y = 60〜80）に留まる — 守備的
  - `1`: 中央ライン近く（Y = H/2 - 80）まで前に出る — 攻撃的
- パックが CPU 側に向かっている時のみ適用
- パックが来ていない時はデフォルトのポジションに戻る

#### S2-2-4: 適応度（adaptability）実装

- `GameState` にスコア情報を渡す or 別引数で受け取る
- スコア差（CPU が負けている時）に応じて以下を強化:
  - `maxSpeed` を最大 +20% 加速
  - `predictionFactor` を最大 +30% 強化
  - `wobble` を最大 -50% 減少（精度向上）
- 計算式: `boost = adaptability * clamp(scoreDiff, 0, 3) / 3`
- `adaptability = 0` の場合は無効（ルーキー）

**成果物**: 改修された ai.ts
**完了条件**: AI がプレイスタイルに応じた動きをする

## リスク管理

| リスク | 影響度 | 対策 |
|--------|--------|------|
| プレイスタイルパラメータが多すぎて調整が難しい | 中 | 5 パラメータに絞り、各キャラの特徴的なパラメータのみ非ゼロにする |
| 適応度が強すぎてゲームバランスが崩れる | 中 | 最大 boost を控えめに（+20%/+30%）し、プレイテストで微調整 |
| 揺さぶりロジックが CPU の動きを不自然にする | 中 | 揺さぶりはターゲット計算にのみ影響させ、移動のスムーズさは維持 |
| AiBehaviorConfig の playStyle 追加で既存テストが壊れる | 低 | オプショナルフィールドとして追加し、後方互換を維持 |
