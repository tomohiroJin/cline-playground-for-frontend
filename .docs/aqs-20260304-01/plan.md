# Agile Quiz Sugoroku ブラッシュアップ計画 (Phase 3)

## 日付: 2026-03-04

## Context

Phase 2b（ペンギン追加・全キャラクター画像統一リニューアル）が完了済み。
今回は Phase 3 として、ゲーム体験全体のブラッシュアップを行う。

## 要望一覧

| # | 要望 | カテゴリ |
|---|------|----------|
| 1 | キャラクター以外の各種画像の一新 | 画像 |
| 2 | スプリント間の成長物語イラスト追加 | 画像・ストーリー |
| 3 | 透過キャラクター用の背景画像作成と切り替え | 画像・UI |
| 4 | GOAL結果をエンジニアタイプ→TEAMタイプに変更 | ロジック・UI |
| 5 | スプリント数に応じたスキップ可能な物語 | ストーリー・UI |
| 6 | 勉強会モードのキャラクター別ジャンル絞り込み | 機能 |
| 7 | スプリント切れ目のSAVE/LOAD機能 | 機能 |

## 実装フェーズ

### Phase 3a: TEAM化＆SAVE機能（コアロジック変更）

**優先度: 最高** — ゲームの根幹に関わる変更

1. **GOAL結果のTEAM化**
   - `engineer-classifier.ts` → `team-classifier.ts` へリネーム・改修
   - エンジニアタイプ6種 → チームタイプ6種に変更
   - `ResultScreen.tsx` のUI・テキスト・画像をTEAM観点に変更
   - `constants.ts` の `getSummaryText` をチーム視点のフィードバックに変更
   - グレード判定ロジックはチーム成熟度としてリフレーム

2. **スプリント切れ目のSAVE/LOAD機能**
   - `save-manager.ts` 新規作成（ゲーム途中状態の保存・復元）
   - `RetrospectiveScreen.tsx` にSAVEボタン追加
   - `TitleScreen.tsx` に「続きから」ボタン追加
   - localStorage のキー設計（既存の `result-storage.ts` との住み分け）

### Phase 3b: ストーリー機能（新規UI）

**優先度: 高** — ゲーム体験の大きな向上

3. **スプリント間の成長物語**
   - `StoryScreen.tsx` 新規コンポーネント作成
   - スプリント1〜8の成長物語データ定義
   - テーマ: 5人の出会い → 衝突 → 協力 → 成長 → 真のTeam完成
   - スキップ機能付き（クリック or キー操作）
   - フェード演出でストーリー画面を表示

4. **スプリント数に応じた物語表示**
   - 選択したスプリント数に応じてストーリーをフィルタリング
   - 例: 3スプリント選択時はスプリント1, 4, 8のストーリーを表示

### Phase 3c: 画像リニューアル（ビジュアル強化）

**優先度: 中** — コード変更後に画像を差し替え

5. **キャラクター以外の各種画像一新**
   - イベント画像8種（planning, impl1, test1, refinement, impl2, test2, review, emergency）
   - UI画像（correct, incorrect, timeup, build_success, grade_celebration, retro）
   - 統一デザインガイドラインに沿って生成

6. **成長物語イラスト**
   - スプリント1〜8の8枚のストーリーイラスト
   - 5キャラの成長過程を描いた一連のシーン

7. **背景画像の作成と切り替え**
   - 3〜5種類のシーン別背景画像を作成
   - スプリント進行に応じた背景切り替えロジック
   - キャラクターの透過画像と背景の合成表示

### Phase 3d: 勉強会モード拡張（機能追加）

**優先度: 中** — 独立した機能拡張

8. **キャラクター別ジャンル絞り込みモード**
   - 各キャラクターの役割・スキルとジャンルのマッピング定義
   - `StudySelectScreen.tsx` にキャラクター選択UIを追加
   - キャラクター選択時に関連ジャンルを自動選択

## 依存関係

```
Phase 3a (TEAM化 + SAVE)
   ├── TEAM化 ─────────────────── → Phase 3c (チームタイプ画像生成)
   └── SAVE機能 ────────────────── → (独立)
Phase 3b (ストーリー機能)
   └── ストーリーコンポーネント ── → Phase 3c (ストーリーイラスト生成)
Phase 3c (画像リニューアル)
   └── 画像生成はユーザー作業、コード変更と並行可能
Phase 3d (勉強会拡張)
   └── (独立、他フェーズと並行可能)
```

## リスク・注意点

1. **既存テストへの影響**: TEAM化により `engineer-classifier` 関連のテストが大幅に変更される
2. **セーブデータの互換性**: スキーマ変更時のマイグレーション考慮が必要
3. **画像数の増加**: 合計30枚以上の画像生成が必要（ユーザー作業の負荷）
4. **ストーリーの文量**: 8スプリント分のストーリーテキスト作成が必要
5. **フェーズ分割**: 各フェーズは独立してリリース可能にする（段階的デプロイ）
