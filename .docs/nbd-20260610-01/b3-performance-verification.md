# Non-Brake Descent B3: グラフィック性能検証

- 実施日: 2026-06-10
- 対象: P5（PR #101）で追加した SVG グローフィルタ（`feGaussianBlur`）と速度線等の演出
- 目的: 演出強化が 60fps（フレーム時間 16.7ms）を阻害していないかを実測で確認する
- 結論: **対応不要**（性能問題は実在せず、軽量化の必要なし）

## 背景

P5 のレビューで「`feGaussianBlur` ベースのグローは CPU 描画環境では重くなる可能性」が観察事項として挙がっていた（ぼかし半径は 3〜4px と最小限）。実測で確認するのが本検証（B3）。

## 検証方法

- 開発サーバー（`npm start`, http://localhost:3000/non-brake-descent）上で実プレイ。
- **ゴッドモード（チート `jinjinjin`）**で死亡を防ぎ、加速を持続させて**高速状態（速度線・グロー・障害物グローが最も多い負荷）**を維持。
- ブラウザの `requestAnimationFrame` で 5 秒間（約 300 フレーム）のフレーム時間を採取し、平均 FPS・中央値/p95/最大フレーム時間・コマ落ち数を算出。
- 実行環境は **ヘッドレス Chromium（ソフトレンダリング）**＝実機 GPU より重い**最悪ケース**の目安。

## 測定結果（PLAY 中・フル負荷）

| 指標 | 値 |
|------|-----|
| 計測フレーム数 | 298（5秒） |
| 同時フィルタ定義数 | 5（playerGlow / dangerGlow / itemGlow / enemyGlow ほか） |
| フィルタ適用要素数 | 15〜20 |
| 速度線本数 | 19 |
| 平均 FPS | **59.6** |
| 中央値フレーム時間 | 16.7ms（= 60fps） |
| p95 フレーム時間 | 16.8ms |
| 最大フレーム時間 | 33.4ms |
| 20ms 超フレーム | 2 / 298 |
| 33ms 超フレーム（コマ落ち） | 2 / 298（0.7%） |

## 評価

- 中央値・p95 ともに 16.7〜16.8ms で**安定して 60fps**を維持。
- 33ms 超は 298 フレーム中 2 回のみ（孤立したスパイク）。フレーム時間が一貫して上昇していないため、SVG フィルタが恒常的ボトルネックである兆候はなく、GC やリコンサイル由来の単発ヒッチと判断。
- これが**ソフトレンダリングの最悪ケース**での結果であり、実機の GPU アクセラレーション環境ではさらに余裕がある。

## 結論

P5 のグローフィルタ・速度線等の演出は **60fps を阻害しておらず、軽量化（フィルタ削減・ぼかし半径縮小等）は不要**。本件はコード変更なしでクローズする。

将来、低スペック端末で問題が観測された場合の軽量化候補（参考・今回は実施せず）:
- `feGaussianBlur` グローを `box-shadow`/`drop-shadow` ベースへ置換
- 速度線・トレイル等の上限本数を端末性能に応じて可変化
- `prefers-reduced-motion` 有効時にグローを簡素化（既に時間操作・速度線等は抑制済み）
