# Falldown Shooter ブラッシュアップ仕様書

## 1. レスポンシブ対応

### 1.1 対応デバイス・画面サイズ

| デバイス | 画面幅 | セルサイズ | 備考 |
|---------|--------|----------|------|
| スマートフォン縦 | ~480px | 動的計算 | メインターゲット |
| スマートフォン横 | ~896px | 動的計算 | 横レイアウト対応 |
| タブレット | ~1024px | 最大30px | iPad等 |
| デスクトップ | 1024px~ | 30px固定 | 既存動作維持 |

### 1.2 レイアウト仕様

#### スマートフォン縦向き（portrait）
```
┌──────────────────┐
│ Header（コンパクト）│
│ SkillGauge        │
│ StatusBar         │
├──────────────────┤
│                  │
│   GameBoard      │
│  （画面幅に合わせ） │
│                  │
├──────────────────┤
│ Controller       │
│ （画面下部固定）   │
└──────────────────┘
```

- セルサイズ: `(画面幅 - 32px) / 12`
- コントローラーは画面下部に固定（`position: sticky; bottom: 0`）
- ゲーム画面が画面内に収まるよう `max-height` を設定

#### スマートフォン横向き（landscape）
```
┌────────────────────────────────┐
│ Header（1行コンパクト）          │
├──────────┬─────────────────────┤
│ GameBoard│ HUD + Controller    │
│          │ ┌───────────────┐  │
│          │ │ SkillGauge    │  │
│          │ │ StatusBar     │  │
│          │ │ PowerUp       │  │
│          │ ├───────────────┤  │
│          │ │ ← 🔫 →       │  │
│          │ └───────────────┘  │
└──────────┴─────────────────────┘
```

- セルサイズ: `(画面高さ - 80px) / 18`
- ゲーム画面とHUD/コントローラーを横並び（`flex-direction: row`）
- `@media (orientation: landscape)` で切り替え

#### タブレット・デスクトップ
- 既存レイアウトを維持（縦積み中央寄せ）
- セルサイズは最大30px

### 1.3 ブレイクポイント

```typescript
const BREAKPOINTS = {
  mobile: 480,      // スマートフォン
  tablet: 768,      // タブレット
  desktop: 1024,    // デスクトップ
} as const;
```

### 1.4 useResponsiveSize フック改修

現在の実装を拡張し、以下を返す:

```typescript
interface ResponsiveLayout {
  cellSize: number;          // セルのピクセルサイズ
  isLandscape: boolean;      // 横向きかどうか
  isMobile: boolean;         // モバイルかどうか
  controllerPosition: 'bottom' | 'side'; // コントローラー位置
}
```

## 2. コントローラーUI刷新

### 2.1 デザインコンセプト

**テーマ**: SF/サイバー風ゲームコントローラー

現在の問題:
- `←` `→` の矢印記号がプレーンテキストで安っぽい
- `🎯` の絵文字がゲームの世界観と合わない

### 2.2 ボタンデザイン仕様

#### 左右移動ボタン
- **アイコン**: SVGシェブロン（`<` `>` 形状）
- **形状**: 角丸正方形（56×56px）
- **色**: グラデーション（#374151 → #4b5563）
- **枠**: 1px solid rgba(99, 102, 241, 0.5)（インジゴの半透明）
- **押下時**: 背景色変化 + scale(0.92) + インジゴのグロー

```svg
<!-- 左ボタンアイコン例 -->
<svg viewBox="0 0 24 24" fill="none" stroke="white" strokeWidth="3" strokeLinecap="round">
  <polyline points="15,18 9,12 15,6" />
</svg>
```

#### 射撃ボタン（中央）
- **アイコン**: カスタムSVG十字照準（クロスヘア）
- **形状**: 円形（64×64px）
- **色**: グラデーション（#dc2626 → #991b1b）
- **枠**: 2px solid rgba(239, 68, 68, 0.7)
- **グロー**: 常時 `box-shadow: 0 0 12px rgba(239, 68, 68, 0.4)`
- **押下時**: scale(0.92) + グロー増大 + 短い脈動

```svg
<!-- 射撃ボタンアイコン例 -->
<svg viewBox="0 0 24 24" fill="none" stroke="white" strokeWidth="2">
  <circle cx="12" cy="12" r="6" />
  <line x1="12" y1="2" x2="12" y2="8" />
  <line x1="12" y1="16" x2="12" y2="22" />
  <line x1="2" y1="12" x2="8" y2="12" />
  <line x1="16" y1="12" x2="22" y2="12" />
</svg>
```

### 2.3 インタラクション仕様

| アクション | 視覚フィードバック | 時間 |
|-----------|-----------------|------|
| ホバー | 枠線の色がインジゴに変化 | 0.2s |
| 押下 | scale(0.92) + 背景色変化 | 0.1s |
| リリース | 元に戻る | 0.15s |
| 長押し（移動） | 連続移動（既存動作維持） | - |

### 2.4 レスポンシブ対応

| デバイス | ボタンサイズ | 射撃ボタンサイズ | 間隔 |
|---------|-----------|---------------|------|
| モバイル | 52×52px | 60×60px | 12px |
| タブレット | 56×56px | 64×64px | 16px |
| デスクトップ | 56×56px | 64×64px | 16px |

## 3. コンボシステム

### 3.1 基本仕様

| パラメータ | 値 | 説明 |
|----------|---|------|
| コンボウィンドウ | 2秒 | この時間内の連続破壊でコンボ成立 |
| 最大コンボ倍率 | 5.0x | コンボ倍率の上限 |
| 倍率増加量 | +0.5x / コンボ | 1コンボごとの倍率増加 |
| スキルゲージボーナス | +10% / 5コンボ | 5コンボごとにゲージボーナス |

### 3.2 コンボ倍率テーブル

| コンボ数 | 倍率 | 表示テキスト |
|---------|------|------------|
| 0 | 1.0x | （非表示） |
| 1 | 1.0x | （非表示） |
| 2 | 1.5x | "2 COMBO!" |
| 3 | 2.0x | "3 COMBO!!" |
| 5 | 3.0x | "5 COMBO!!!" |
| 8 | 4.5x | "8 COMBO!!!!" |
| 10+ | 5.0x | "MAX COMBO!!!!!" |

### 3.3 UI仕様

- **コンボ表示**: ゲーム画面中央上部にフローティング表示
- **フォント**: 太字、サイズはコンボ数に応じて拡大（1.5rem → 2.5rem）
- **色**: コンボ数に応じてグラデーション（白 → 黄 → 赤 → 紫）
- **アニメーション**: ポップイン（scale 0→1.2→1.0）+ フェードアウト
- **表示時間**: 2秒（コンボ継続中はリセット）

### 3.4 実装方針

新規フック `use-combo-system.ts` を作成:

```typescript
interface ComboState {
  count: number;          // 現在のコンボ数
  multiplier: number;     // 現在の倍率
  isActive: boolean;      // コンボ継続中かどうか
  lastHitTime: number;    // 最後にブロックを破壊した時刻
}
```

## 4. スコア演出

### 4.1 フローティングスコア

- ブロック破壊位置に `+10` 等のスコア数字をポップアップ
- 上方向に浮き上がりながらフェードアウト（0.8秒）
- コンボ倍率適用時は色が変化

### 4.2 ラインクリア演出

- クリアされたラインが白く光ってから消える（0.3秒）
- 消去後に上のブロックが落下するアニメーション
- 複数ライン同時クリア時は連続で演出

### 4.3 ハイスコア更新演出

- "NEW HIGH SCORE!" テキストが金色で表示
- 星のパーティクルエフェクト
- ゲームオーバー画面でハイライト表示

## 5. 画面シェイクエフェクト

### 5.1 仕様

| トリガー | 強度 | 持続時間 | 方向 |
|---------|------|---------|------|
| 爆弾使用 | 4px | 300ms | XY |
| ブラストスキル | 6px | 400ms | XY |
| ラインクリア | 2px | 200ms | Y |
| ゲームオーバー | 8px | 500ms | XY |

### 5.2 実装

GameArea に CSS transform を適用:
```css
@keyframes shake {
  0%, 100% { transform: translate(0, 0); }
  25% { transform: translate(-Npx, Npx); }
  50% { transform: translate(Npx, -Npx); }
  75% { transform: translate(-Npx, -Npx); }
}
```

### 5.3 アクセシビリティ

- `prefers-reduced-motion` メディアクエリに対応
- ユーザー設定でON/OFF切り替え可能
