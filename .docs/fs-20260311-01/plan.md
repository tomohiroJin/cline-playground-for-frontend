# Falldown Shooter ブラッシュアップ計画

## 概要

落ち物シューティング（Falldown Shooter）をユーザーに「面白い」と思ってもらえるゲームに仕上げるためのブラッシュアップ計画。
UIの改善、ゲーム性の向上、演出の強化を軸に進める。

## 現状分析

### 強み
- 7つのカスタムフックによる設計分離が良好
- 4ステージ × 3難易度 × 5パワーアップ × 3スキルの組み合わせ
- ランキング・ハイスコア・ヘルプ画面など基本UIは揃っている
- テストカバレッジが高い（158テスト）

### 課題

| カテゴリ | 課題 | 優先度 |
|---------|------|--------|
| **UI/レスポンシブ** | ゲーム画面がモバイル横向き・タブレットに未対応 | 高 |
| **UI/コントローラー** | 左右の矢印と🎯がダサい、ゲームらしくない | 高 |
| **演出** | クリア・ゲームオーバー時の演出が地味 | 中 |
| **フィードバック** | 操作時の視覚・触覚フィードバックが弱い | 中 |
| **UI/情報** | ゲーム中のHUD（スキル・パワーアップ）が分かりづらい | 中 |
| **リプレイ性** | コンボシステムがなく、連続プレイのモチベーションが薄い | 中 |

## ブラッシュアップ方針

### Phase 1: UI改善（ユーザー要望対応）
**目標**: ユーザーから指摘された課題を解消する

1. **レスポンシブ対応の強化**
   - モバイル縦向き・横向き両対応
   - タブレット対応（iPad等）
   - デスクトップでの最適表示（動的計算、下限30px・上限50px。※ 当初30px固定としたが、フィードバックにより動的計算に変更）
   - 画面サイズに合わせたセルサイズの拡大・縮小（幅と高さ両方を考慮し、画面に収まる最大サイズを算出）

2. **コントローラーUIの刷新**
   - 左右ボタン: 矢印記号 → SVGアイコン（シェブロン）にリデザイン
   - 中央射撃ボタン: 🎯 → カスタムSVGアイコン（照準/十字線デザイン）
   - ボタン全体: ゲーミングコントローラー風のデザインに統一
   - ボタン押下時のリップルエフェクト追加

### Phase 2: ゲーム体験の向上
**目標**: 「もう1回遊びたい」と思わせる仕掛けを追加

3. **コンボシステムの導入**
   - 短時間での連続ライン消しでコンボ倍率が上昇（※ 当初はブロック破壊でコンボとしていたが、フィードバックによりライン消しに変更）
   - コンボ表示UI（画面中央にポップアップ）
   - コンボ維持でスキルゲージボーナス

4. **スコア演出の強化**
   - ブロック破壊時のスコアポップアップ（フローティング数字）
   - ラインクリア時の派手なエフェクト
   - ハイスコア更新時の特別演出

5. **画面シェイクエフェクト**
   - 爆弾使用時の画面揺れ
   - ゲームオーバー時の画面揺れ
   - ラインクリア時の軽い振動

### Phase 2.5: テストモード
**目標**: 開発・動作確認を効率化する裏技モードを導入する

9. **隠しコマンドによるテストモード起動**
   - タイトル画面（idle状態）で「jin」を3回タイプするとテストモード有効化
   - セッション中は維持（リロードでリセット）
   - 画面隅にテストモードインジケーター表示

10. **デバッグパネル**
    - グリッド操作（指定行数分「あと1セルで揃う行」を生成 → 弾を撃つだけでライン消しテスト可能）
    - エフェクト手動発火（各種シェイク、ハイスコア演出）
    - スコア・ステージ操作
    - 内部状態のリアルタイム表示（コンボ数・倍率・スキルゲージ）

### Phase 3: スコアバランス調整
**目標**: ゲームルールとして自然なスコア設計にする

6. **コンボトリガーの修正**
   - 弾でブロックを壊した回数 → ライン消し回数に変更（済）
   - コンボは「短時間に連続でラインを消した回数」で成立

7. **同時消しボーナスの導入**
   - 複数ライン同時消しに倍率ボーナスを追加
   - 同時消しの方が連続消しより高得点になるようバランス調整
   - 1ライン: 1.0倍、2ライン: 1.5倍、3ライン: 2.0倍、4ライン: 3.0倍

8. **ライン消しスコアへのコンボ倍率適用**
   - 現状コンボ倍率は弾ヒットスコアにのみ適用されている
   - ライン消しスコアにもコンボ倍率を適用する

## 技術方針

- 既存のカスタムフック構成を活かし、新機能はフック単位で追加
- styled-components のテーマ機能は使わず、既存パターンを踏襲
- 新規コンポーネントは `src/features/falldown-shooter/components/` に配置
- SVGアイコンはインラインSVGで実装（外部ファイル不要）
- CSS アニメーションを活用（JavaScript アニメーションは最小限）
- 各Phase完了ごとにテストを追加・更新

## スケジュール目安

| Phase | 内容 | 見積り |
|-------|------|--------|
| Phase 1 | UI改善（レスポンシブ + コントローラー） | 作業項目 1-2 |
| Phase 2 | ゲーム体験の向上（コンボ + 演出） | 作業項目 3-5 |
| Phase 2.5 | テストモード | 作業項目 9-10 |
| Phase 3 | スコアバランス調整 | 作業項目 6-8 |

## リスクと注意点

- レスポンシブ対応はブラウザ間の差異に注意（特にiOS Safari）
- コンボシステムはゲームバランスへの影響が大きいため、数値は調整可能にする
- 画面シェイクは酔いやすいユーザーへの配慮が必要（ON/OFF設定）
- 既存テスト158件が壊れないよう段階的に進める
