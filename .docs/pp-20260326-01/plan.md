# 原始進化録（PRIMAL PATH）大規模改良計画

## 概要

原始進化録の画面サイズ・フォント・スプライトを大幅に拡大し、PC・モバイル両対応のレスポンシブレイアウトを実現する。

## 現状分析

| 項目 | 現在値 | 課題 |
|------|--------|------|
| ゲーム画面 | 480×720px 固定 | PC では小さすぎる、モバイルではアスペクト比崩れ |
| フォント | 8px〜24px 絶対値 | 小さい文字が読みづらい（特に 8〜10px） |
| スプライト | ベース16px × スケール2 = 32px | ドット絵が小さく、ディテール表現の余地が少ない |
| レスポンシブ | 500px 以下で全画面のみ | PC 大画面での活用不足、タブレット未対応 |

## 改良方針

### Phase 1: レイアウト基盤の刷新

ゲーム画面の基準サイズを拡大し、CSS ベースのレスポンシブスケーリングを導入する。

**目標画面サイズ**:
- 基準サイズ: **800×1200px**（アスペクト比 2:3 維持）
- PC: ビューポートに収まる最大サイズで中央配置（`max-height: 95vh` でフィット）
- タブレット: ビューポート幅に合わせてスケーリング
- モバイル: 全画面表示（現行と同様だが比率維持）

**アプローチ**:
- `GameShell` の基準サイズを 800×1200 に変更
- CSS `scale()` + `transform-origin` による動的スケーリング
- `ResizeObserver` でビューポート変化を検知し、スケール係数を再計算
- アスペクト比 2:3 を常に維持

### Phase 2: フォントサイズの拡大

全フォントサイズを底上げし、最小サイズの可読性を確保する。

**サイズ変更方針**:

| 用途 | 現在 | 変更後 | 備考 |
|------|------|--------|------|
| タイトル | 22px | 32px | 画面拡大に比例 |
| サブタイトル | 14px | 20px | |
| オーバーレイテキスト | 16px | 22px | |
| スキルボタン | 14px | 20px | |
| ゲームボタン | 12px | 16px | |
| パネルテキスト | 10px | 14px | 最小可読サイズ以上 |
| ログ・バッジ | 9px | 13px | |
| スピードボタン | 9px | 13px | |
| 降伏ボタン | 8px | 12px | 旧最小→新最小 |
| ポーズ画面 | 24px | 36px | |
| オーバーレイアイコン | 48px | 64px | |

**原則**: 最小フォントサイズは **12px** とする（モバイルでの可読性確保）

### Phase 3: スプライトの拡大とクォリティアップ

ドット絵のベースサイズとスケール係数を上げ、描画ディテールを追加する。

**サイズ変更**:

| キャラクター | 現在ベース | 現在表示 | 新ベース | 新表示 | 倍率 |
|------------|----------|---------|---------|--------|------|
| プレイヤー | 16×22 | 32×44 | 24×32 | 72×96 (s=3) | ×2.2 |
| 味方 | 12×16 | 24×32 | 18×24 | 54×72 (s=3) | ×2.3 |
| 敵（通常） | 16×16 | 32×32 | 24×24 | 72×72 (s=3) | ×2.3 |
| 敵（ボス） | 24×24 | 48×48 | 32×32 | 96×96 (s=3) | ×2.0 |
| タイトルロゴ | 240×130 | 240×130 | 400×200 | 400×200 | ×1.7 |

**クォリティアップ内容**:
- ベースグリッドの拡大（16→24px）による描画ピクセル増加でディテール向上
- プレイヤー: 手足のアウトライン追加、武器のディテール強化
- 敵: ボディパターンの追加、目の表現改善
- ボス: 形態変化時のディテール追加（オーラ、模様など）
- 味方: 種族ごとの識別性向上（シルエットの差別化）
- HP バー: 3px → 5px に拡大

### Phase 4: UI コンポーネントの調整

画面拡大に伴う各 UI パーツのサイズ・間隔調整。

- バトル画面のレイアウト再配置
- パネル（敵情報、ステータス）のサイズ調整
- ボタン類のタップ領域拡大（モバイル UX 向上）
- ログ領域の行数・表示幅調整
- タブ UI のサイズ調整

### Phase 5: テスト・動作確認

- 全画面遷移の動作確認
- PC（1920×1080、1366×768）での表示確認
- タブレット（768×1024）での表示確認
- モバイル（375×667、390×844）での表示確認
- 既存テストの修正と実行
- バトルロジックの動作確認（スプライト変更の影響なし確認）

## スケジュール目安

| Phase | 内容 | 規模感 |
|-------|------|--------|
| 1 | レイアウト基盤 | 中（styles.ts + GameShell リサイズ + レスポンシブ） |
| 2 | フォント拡大 | 小（styles.ts の数値変更中心） |
| 3 | スプライト改良 | 大（sprites.ts の描画ロジック全面改修） |
| 4 | UI 調整 | 中（各コンポーネントの微調整） |
| 5 | テスト | 小〜中（既存テスト修正 + 手動確認） |

## リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| スプライト描画の崩れ | キャラクター表示の破綻 | Phase ごとにビルド確認、段階的変更 |
| パフォーマンス低下 | Canvas 再描画コスト増 | 描画最適化（キャッシュ Canvas の活用） |
| レイアウト崩れ | UI 要素の重なり・はみ出し | 各ブレークポイントで手動確認 |
| 既存テストの大量失敗 | CI 通過不可 | Phase 1 完了時にテスト一括修正 |

## 技術方針

- アスペクト比 2:3 は維持（縦向きゲームのため）
- CSS `scale()` によるスケーリングを採用（Canvas 内部座標に影響しない）
- スプライトのスケール係数 `s` をデフォルト 2→3 に変更
- ベースピクセルグリッドを 16→24 に拡大してディテール追加
- Retina 対応は今回スコープ外（将来の改善事項）
