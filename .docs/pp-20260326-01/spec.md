# 原始進化録（PRIMAL PATH）大規模改良 — 仕様書

## 1. 画面サイズ仕様

### 1.1 基準サイズ

| 項目 | 値 |
|------|-----|
| 基準幅 | 800px |
| 基準高さ | 1200px |
| アスペクト比 | 2:3（変更なし） |
| 最小幅 | 320px |
| 最大幅 | 800px（基準サイズ上限） |

### 1.2 レスポンシブブレークポイント

| 名称 | 条件 | 動作 |
|------|------|------|
| デスクトップ（大） | `min-width: 801px` | 基準サイズ（800×1200）で中央配置、余白あり |
| デスクトップ（小） | `601px ≤ width ≤ 800px` | ビューポート幅に合わせて縮小 |
| タブレット | `481px ≤ width ≤ 600px` | ビューポート幅に合わせて縮小、UI 配置維持 |
| モバイル | `width ≤ 480px` | 全画面表示、アスペクト比維持で上下余白 |

### 1.3 スケーリング方式

```
スケール係数 = min(viewportWidth / 800, viewportHeight / 1200)
```

- `GameShell` に `transform: scale(係数)` と `transform-origin: top center` を適用
- `ResizeObserver` でビューポート変化を監視
- 最小スケール: 0.4（320px / 800px）
- 最大スケール: 1.0（基準サイズ以上では拡大しない）

### 1.4 GameShell スタイル変更

**変更前**:
```css
width: 480px;
height: 720px;
```

**変更後**:
```css
width: 800px;
height: 1200px;
```

**メディアクエリ変更**:
```css
/* 旧: @media (max-width: 500px) → 削除 */
/* 新: JavaScript による動的スケーリングに統一 */
```

## 5. レスポンシブ動作仕様

### 5.1 デスクトップ（801px 以上）

- ゲーム画面: 800×1200px、中央配置
- 背景に暗色余白
- 枠線表示

### 5.2 小デスクトップ・タブレット（481px〜800px）

- ゲーム画面: ビューポートに収まるようスケーリング
- 枠線表示
- アスペクト比 2:3 維持

### 5.3 モバイル（480px 以下）

- 全画面表示
- 枠線なし
- アスペクト比 2:3 維持（上下に黒帯の可能性）
- タッチ操作のタップ領域を最低 44px 確保

### 5.4 スケーリング実装

```typescript
// useGameScale カスタムフック
const useGameScale = () => {
  const [scale, setScale] = useState(1);

  useEffect(() => {
    const observer = new ResizeObserver((entries) => {
      const { width, height } = entries[0].contentRect;
      const scaleX = width / 800;
      const scaleY = height / 1200;
      setScale(Math.min(scaleX, scaleY, 1));
    });
    observer.observe(document.documentElement);
    return () => observer.disconnect();
  }, []);

  return scale;
};
```

## 7. 非機能要件

- Canvas 描画のフレームレートは既存と同等を維持
- ゲームロジック（バトル計算、進化、育成）には一切影響しない
- ローカルストレージのセーブデータ互換性を維持
- ビルドサイズの大幅な増加は避ける
