# SEO プロジェクト管理

SEO 改善を 6 フェーズで体系的に管理・実行します。

## エージェント参照

詳細なフェーズ管理知識は `agents/seo-manager.yaml` を参照してください。
エージェントファイルが見つからない場合は、以下の手順で完結できます。

## 管理フェーズ

### フェーズ 1: 現状分析

- `seo-review` スキルによるフルスキャンを実施
- 検出された問題を重要度別に分類
- 現在の SEO 状態のベースラインを記録

### フェーズ 2: 緊急是正（スパム・ポリシー違反排除）

- Critical レベルの問題を最優先で対処
- Google スパムポリシー違反の排除
- ペナルティリスクの高い要素を即時修正

### フェーズ 3: インデックス制御修正

- robots.txt の最適化
- canonical タグの整合性確保
- noindex / nofollow の適切な設定
- サイトマップの更新・提出
- クロールバジェットの最適化

### フェーズ 4: レンダリング・モバイル対応

- レンダリング方式に起因する問題の修正
- モバイルフレンドリーの確保
- viewport・タップターゲットの最適化
- レスポンシブデザインの検証

### フェーズ 5: パフォーマンス・UX 最適化

- Core Web Vitals（LCP, INP, CLS）の改善
- 画像最適化（WebP/AVIF、遅延読み込み）
- JavaScript の最適化（コード分割、defer/async）
- クリティカル CSS の抽出

### フェーズ 6: 長期戦略

- コンテンツ SEO 戦略の策定
- 構造化データの拡充
- AI 検索対応（GEO / AIO）の検討
- 継続的モニタリング体制の構築

## 進捗管理

各フェーズの開始時に:

1. 対象タスクの一覧と優先度を提示
2. 完了条件を明確化
3. フェーズ完了後に成果を報告

## 連携スキル

- フェーズ 1 では `seo-review` を活用します
- フェーズ 2-4 の実装には `seo-implement` を使用します
- フェーズ 5 には `seo-performance` が有効です
- フェーズ 6 の AI 検索対応は `seo-ai-search` を参照してください
