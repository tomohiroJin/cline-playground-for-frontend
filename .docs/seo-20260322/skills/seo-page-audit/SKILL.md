# ページ SEO 監査

単一ページの SEO を包括的に深堀り分析します。

**注意: このスキルは分析のみを行います。コードの変更は行いません。**

## 監査手順

### 1. title タグ

- 文字数の確認（推奨: 30〜60 文字）
- 主要キーワードが前方に配置されているか
- ページ固有のタイトルか（サイト全体で一意）
- ブランド名の適切な配置

### 2. meta description

- 文字数の確認（推奨: 70〜160 文字）
- 検索意図に対応した内容か
- CTA（行動喚起）が含まれているか
- ページ固有の description か

### 3. 見出し構造

- h1 タグがページに 1 つだけ存在するか
- h2〜h6 の論理的な階層構造が保たれているか
- 見出しレベルのスキップがないか（h1 → h3 等）
- 見出しが内容を適切に要約しているか

### 4. コンテンツ品質

- **E-E-A-T**: 経験・専門性・権威性・信頼性のシグナル
- **検索意図適合性**: ユーザーの検索意図に応えるコンテンツか
- **独自性**: オリジナルの価値を提供しているか
- **網羅性**: トピックを十分にカバーしているか

### 5. 画像

- alt 属性が適切に設定されているか
- width / height 属性が指定されているか（CLS 防止）
- 遅延読み込み（loading="lazy"）が適用されているか
- ファーストビュー画像に fetchpriority="high" が設定されているか

### 6. リンク・URL

- canonical タグが正しく設定されているか
- 内部リンクが適切に配置されているか
- 壊れたリンクがないか
- URL 構造が SEO フレンドリーか（短く、意味のある文字列）

### 7. OGP / SNS

- og:title, og:description, og:image が設定されているか
- Twitter Card（twitter:card, twitter:title 等）が設定されているか
- og:image のサイズが推奨仕様を満たしているか

### 8. 構造化データ

- JSON-LD が存在するか
- ページ種別に適したスキーマが使用されているか
- 必須プロパティが揃っているか

## 連携スキル

- 検出された問題の修正には `seo-implement` を使用してください
- 構造化データの詳細設計には `seo-structured-data` が有効です
- サイト全体のレビューには `seo-review` を活用できます
