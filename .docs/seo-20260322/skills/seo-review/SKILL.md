# SEO レビュー

SEO アンチパターンとポリシー違反を 3 層構造で体系的に検出します。

**注意: このスキルは読み取り専用です。コードの変更は行いません。**

## エージェント参照

詳細な検査項目は `agents/seo-reviewer.yaml` を参照してください。
エージェントファイルが見つからない場合は、以下の手順で完結できます。

## レビュー手順

### 1. 対象プロジェクトの構造把握

- プロジェクトのディレクトリ構成を確認する
- ページ・テンプレートの一覧を把握する
- SEO 関連の設定ファイルを特定する

### 2. A 層検査: スパム・ポリシー違反

最も重大な問題を優先的に検出:

- **クローキング**: ユーザーと検索エンジンへの異なるコンテンツ提供
- **隠しテキスト・リンク**: CSS/JS による不可視コンテンツ
- **キーワードスタッフィング**: 不自然なキーワード詰め込み
- **不正なリダイレクト**: スニーキーリダイレクト
- **自動生成コンテンツ**: 低品質な自動生成テキスト
- **リンクスキーム**: 不自然なリンク操作
- **構造化データの悪用**: ページ内容と不一致のマークアップ

### 3. B 層検査: 技術・UX 系

- **インデックス制御**: robots.txt, noindex, canonical の矛盾
- **レンダリング問題**: CSR でのインデックス不能コンテンツ
- **モバイル対応**: viewport 設定、タップターゲットサイズ
- **HTTPS**: Mixed Content、リダイレクトチェーン
- **ページ速度**: 明らかなパフォーマンスボトルネック
- **クロールバジェット**: 無限パラメータ URL、重複ページ
- **国際化**: hreflang の不整合
- **サイトマップ**: 不整合、欠落ページ

### 4. C 層検査: AI コンテンツ品質

- **E-E-A-T シグナル**: 著者情報、引用元の有無
- **付加価値**: 既存コンテンツの単なるリライトでないか
- **検索意図の適合**: コンテンツと想定クエリの整合性

### 5. レポート生成

以下の形式で構造化レポートを出力:

```
## SEO レビューレポート

### Critical（即時対応必須）
- [問題と影響の説明]

### High（早期対応推奨）
- [問題と影響の説明]

### Medium（改善推奨）
- [問題と影響の説明]

### Low（余裕があれば対応）
- [問題と影響の説明]

### 改善ロードマップ
1. [優先度順の対応計画]
```

## 連携スキル

- 検出された問題の修正には `seo-implement` を使用してください
- 包括的な改善プロジェクトには `seo-manage` を活用できます
