# 原始進化録（primal-path）大規模 UI リサイズ計画

## 概要

ゲーム画面・フォント・ドット絵すべてを大きくし、PC 大画面でも快適にプレイできるよう改修する。
モバイル対応も段階的に行い、レスポンシブ対応を強化する。

## 背景・問題

| 項目 | 現状 | 問題 |
|------|------|------|
| GameShell | 480×720px 固定 | PC 画面で非常に小さく見える |
| フォント | 8px〜22px（大半が10px以下） | 視認性が低い |
| プレイヤースプライト | 16×22px → 2× = 32×44px | 小さすぎてドット絵の魅力が伝わらない |
| 敵スプライト（通常） | 16×16px → 2× = 34×34px | 同上 |
| 敵スプライト（ボス） | 24×24px → 2× = 52×52px | 同上 |
| レスポンシブ | @media (max-width: 500px) のみ | タブレット・大画面に未対応 |

## 目標

1. **PC 大画面対応**: 一般的なモニタ（1920×1080）で画面中央に適度なサイズで表示
2. **フォント拡大**: 最小フォントを 11px 以上に（モバイル含む）、タイトルは 28px+ に
3. **スプライト拡大・品質向上**: スケール 3×〜4× に拡大、ドット絵の描き込み追加
4. **レスポンシブ対応**: モバイル・タブレット・PC の3段階ブレイクポイント
5. **アクセシビリティ強化**: `prefers-reduced-motion` 対応、コントラスト検証

## アプローチ

### Phase 0: デザイントークン定義

CSS 変数の定義のみを先行して実施。後続 Phase でトークンを参照する基盤を作る。

- フォントサイズトークン（`--fs-title` 〜 `--fs-tiny`）
- スペーシングトークン（`--sp-screen-pad` 等）
- カラーパレットトークン（`--c-accent`, `--c-text`, `--c-bg` 等）
- ゲームサイズトークン（`--game-width`, `--game-height`）
- ブレイクポイント別の値設定（メディアクエリ）
- `clamp()` による流動的サイズ（タイトル等の大きなフォント）

### Phase 1: GameShell リサイズ + レスポンシブ基盤

GameShell を固定サイズから可変サイズに変更する。

| ブレイクポイント | 幅 | 高さ | 備考 |
|---------------|-----|------|------|
| モバイル (≤600px) | 100vw | 100dvh | 全画面表示（`dvh` 非対応ブラウザ向けに `100vh` フォールバック） |
| タブレット (601px〜1024px) | 90vw (max 700px) | 90vh (max 960px) | 中間サイズ |
| タブレット横向き | 90vw (max 960px) | 90vh (max 700px) | `@media (orientation: landscape)` で幅と高さの比率を調整 |
| PC (≥1025px) | 720px | 960px | 固定サイズ、画面中央配置 |

- Phase 0 で定義したトークンを使用
- `clamp()` を活用して滑らかなスケーリング

### Phase 2: フォントサイズ拡大

全フォントサイズを段階的に引き上げる。最小フォントは全デバイスで 11px 以上を保証。

| 用途 | 現状 | PC | モバイル | 流動的サイズ |
|------|------|-----|---------|------------|
| タイトル (Title) | 22px | 28px | 22px | `clamp(22px, 4vw, 28px)` |
| サブタイトル (SubTitle) | 14px | 18px | 15px | `clamp(15px, 2.5vw, 18px)` |
| ボタン (GameButton) | 12px | 15px | 13px | — |
| パネルテキスト | 10px | 13px | 12px | — |
| 小テキスト | 9px | 12px | 11px | — |
| 極小テキスト | 8px | 11px | 11px | — |

- CSS 変数で管理し、ブレイクポイントで段階調整
- `clamp()` をタイトル・サブタイトル等の大きなフォントに適用
- モバイルでも最小 11px を下回らない

### Phase 3: スプライト拡大・品質向上

#### サイズ変更

| 対象 | 現状 Canvas/Scale | 変更後 Canvas/Scale | 実表示 |
|------|-----------------|-------------------|--------|
| プレイヤー | 16×22 / 2× | 16×22 / 3× | 48×66px |
| 敵（通常） | 16×16 / 2× | 16×16 / 3× | 48×48px |
| 敵（ボス） | 24×24 / 2× | 24×24 / 3× | 72×72px |
| 仲間 | 12×16 / 2× | 12×16 / 3× | 36×48px |
| タイトルロゴ | 240×130 / 1.5× | 240×130 / 2× | 480×260px |

#### 品質向上（※サイズ変更 PR とは別 PR で実施を推奨）

- プレイヤースプライト: 文明別の装飾パーツ追加（テクノロジーは歯車、生活は葉、儀式は紋章）
- ボススプライト: 追加ディテール（影、グロー効果）
- タイトルロゴ: グラデーション改善、星エフェクト密度アップ

#### Canvas 描画関数の座標系確認

スケール変更に伴い、以下の描画関数の座標計算が `scale` パラメータに正しく追従するか確認:

- `drawDmgPopup`: `popup.fs`（フォントサイズ）、ポップアップ座標
- `drawEnemyHpBar`: HP バーの Y 座標・幅
- `drawBurnFx`: 軌道半径・パーティクル位置
- `drawTitle`: Canvas 内部の描画座標全般

### Phase 4: 個別画面レイアウト調整

各画面のパディング・マージン・間隔をリサイズに合わせて調整。

- BattleScreen: パネル間隔拡大、ログ表示エリア拡大（max-height を GameShell 高さに連動）
- EvolutionScreen: カード表示エリア拡大
- EventScreen: イベントカードの余白調整
- TitleScreen: ロゴとボタンの配置最適化
- SkillPanel: ボタンサイズ拡大（min-width: 120px, min-height: 52px）
- 天候パーティクル: アニメーション距離を `--game-height` に連動（固定値 720px/740px を廃止）
- `image-rendering: pixelated` のブラウザ互換性対応（Safari 向け `-webkit-optimize-contrast` フォールバック）

### Phase 5: テスト・微調整

- 全画面のスクリーンショット取得・目視確認
- モバイル/タブレット（横向き含む）/PC でのレイアウト崩れチェック
- Playwright によるスクリーンショット比較（Visual Regression Testing）の導入検討
- 既存テストの修正（Canvas サイズ変更に伴う）
- `prefers-reduced-motion: reduce` でのアニメーション抑制確認
- E2E テスト実行

## リスク

| リスク | 対策 |
|--------|------|
| Canvas サイズ変更でレイアウト崩れ | Phase ごとに動作確認 + 描画関数の座標系棚卸し |
| Canvas 描画関数の座標系不整合 | `drawDmgPopup`, `drawEnemyHpBar`, `drawBurnFx` 等を個別検証 |
| 大画面でスクロール不要になりすぎる | max-height で制御 |
| モバイルでボタンが押しづらくなる | タッチターゲット 44px 以上を保証 |
| 既存テスト破損 | Canvas モック更新 |
| 天候パーティクルの座標ハードコード | `--game-height` に連動する動的計算に変更 |
| `dvh` 非対応ブラウザ | `100vh` → `100dvh` の段階的フォールバック |
| `image-rendering: pixelated` のブラウザ差異 | Safari 向け `-webkit-optimize-contrast` フォールバック |
| 横向きタブレットでのレイアウト不整合 | `@media (orientation: landscape)` で専用レイアウト |

## スケジュール

Phase 1〜5 を順番に実施。Phase ごとにビルド・テスト確認を行う。
