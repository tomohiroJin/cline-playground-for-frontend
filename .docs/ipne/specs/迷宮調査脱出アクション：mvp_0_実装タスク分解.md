# MVP0 実装タスク分解（E2E最小構成）

本ドキュメントは、
E2E MVP計画における **MVP0「歩けて脱出できる」** を
実際に手を動かせるレベルまで分解した実装タスクリストである。

目的は「最短で“遊べるもの”を作る」こと。
完成度・拡張性・最適化は一切考慮しない。

---

## MVP0のゴール（再確認）

- ゲームを起動できる
- プロローグが表示される
- プレイヤーが操作できる
- ゴールに到達するとクリアになる
- 1プレイが2〜3分で完結する

※ これがE2Eで動けば成功

---

## 0. 前提・制約

- Webフロントエンドのみ
- フレームワーク不問（Canvas / DOM / WebGL どれでも可）
- スマホ・PC両対応を意識（最低限）

---

## 1. 画面構成（最小）

### 必要な画面は3つだけ

1. **タイトル／開始画面**
2. **ゲーム画面**
3. **クリア画面**

※ 設定画面・リトライUIは後回し

---

## 2. タスク一覧（実装順）

### Task 1：プロジェクト雛形

- HTML / CSS / JS の最小構成
- 画面全体を占有するゲーム領域
- リサイズ対応（最低限）

**完了条件**
- ブラウザで空の画面が表示される

---

### Task 2：画面遷移の骨組み

- タイトル画面表示
- 「開始」ボタン
- ゲーム画面へ遷移

**完了条件**
- ボタンを押すと画面が切り替わる

---

### Task 3：プロローグ表示

- 固定テキスト2〜3行
- 任意キー／タップでスキップ

**完了条件**
- 開始前に必ず一度表示される

---

### Task 4：ゲーム画面ベース

- 背景（単色でOK）
- マップ表示エリア
- プレイヤー表示（四角でOK）

**完了条件**
- プレイヤーが画面内に存在する

---

### Task 5：プレイヤー移動

- キーボード（WASD or 矢印）
- タップ／スワイプ（簡易）
- 移動速度一定

**完了条件**
- プレイヤーが自由に動ける

---

### Task 6：簡易マップ（固定）

- 固定サイズの迷路 or 部屋構成
- 壁（衝突判定あり）
- 一本道構成でOK

**完了条件**
- 壁にぶつかると止まる

---

### Task 7：ゴール配置

- ゴールオブジェクト配置
- 接触判定

**完了条件**
- ゴールに触れるとクリアになる

---

### Task 8：クリア画面

- 固定テキスト表示
- 「もう一度」ボタン

**完了条件**
- クリア後に操作不能になる
- 再スタートできる

---

## 3. あえて「やらないこと」リスト

MVP0では以下は**実装しない**。

- 敵
- HP
- レベルアップ
- 自動生成
- 自動マッピング
- 職業差
- 罠
- アイテム
- サウンド

👉 やらないことで、
「E2E完成」を最短にする。

---

## 4. MVP0のチェックリスト

MVP0完成時に、以下を確認する。

- [ ] 操作がストレスなくできる
- [ ] 画面が狭すぎない／広すぎない
- [ ] 2〜3分で終わる
- [ ] 「次に何を足すか」が明確

1つでも×なら、
次のMVPに進まず修正する。

---

## 5. MVP0 → MVP1への移行条件

以下をすべて満たしたら、MVP1へ進む。

- 操作に違和感がない
- マップサイズ感が妥当
- ゴール到達が分かりやすい

---

## 6. 開発者向けメモ（重要）

- 見た目は四角でいい
- 演出は不要
- 「完成させる」ことが唯一の目的

**MVP0は“ゲームを作る準備運動”である。**
