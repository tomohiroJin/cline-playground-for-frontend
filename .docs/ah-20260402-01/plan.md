# Air Hockey — VsScreen ラベル・トースト通知・Canvas 描画最適化 実装計画

## 背景

S6（ペアマッチ品質向上）が 2026-04-01 に完了し、全 604 スイート / 7685 テスト全パスの状態。
ただし以下の 3 領域が「次フェーズ送り」または「未実装」として残っている。

| ID | 残タスク | 理由 |
|----|---------|------|
| S6-5-12 | VsScreen P3/P4 操作タイプラベル表示 | 次フェーズ送り |
| S6-5-13〜14 | ゲームパッド接続/切断トースト通知（Canvas 描画） | Canvas 描画ロジック未実装 |
| S6-4-11〜12 | Canvas 描画最適化 | パフォーマンス計測後に対応 |

## 目標

> 「2v2 モードの UI 情報伝達と描画パフォーマンスを完成させる」
> — P3/P4 の操作タイプが一目で分かり、ゲームパッド操作のフィードバックが即座に伝わり、
>   4 マレット同時描画でも安定した FPS を維持する状態にする

### スコープ

**含む:**
- VsScreen での P3/P4 操作タイプラベル（CPU / 3P / 4P）表示
- Canvas 上でのゲームパッド接続/切断トースト通知（フェードアウト付き）
- Canvas 描画パイプラインの最適化（quickReject 本格適用、不要描画スキップ）

**含まない:**
- ストーリー第 2 章
- VsScreen 2v2 レスポンシブ対応（モバイル）
- 画像アセットの品質改善
- ゲームパッド振動フィードバック

## 実装順序と依存関係

```
Phase 1: VsScreen P3/P4 ラベル（S7-1）
  ├── P3/P4 の controlType を VsScreen に伝搬
  └── CharacterPanel にラベル表示
          ↓
Phase 2: Canvas トースト通知（S7-2）
  ├── ui-renderer にトースト描画関数を追加
  ├── useGameLoop から toast 状態を渡して描画
  └── フェードアウトアニメーション
          ↓
Phase 3: Canvas 描画最適化（S7-3）
  ├── パフォーマンス計測（ベースライン取得）
  ├── quickReject を processCollisions に本格統合
  ├── 描画スキップ最適化（destroyed 障害物等）
  └── パフォーマンス計測（改善効果確認）
```

### 依存関係の理由

- Phase 1 は独立。UI コンポーネントのみの変更
- Phase 2 は Canvas 描画基盤の変更を伴うが、Phase 3 のボトルネックにはならない軽量な追加
- Phase 3 は Phase 2 完了後に実施することで、トースト描画も含めた総合的な最適化が可能

## リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| VsScreen Props 追加で既存テストが壊れる | 中 | Props はすべて optional。既存テストに影響なし |
| Canvas トースト描画がゲームループの FPS に影響 | 低 | テキスト 1 行の描画のみ。Phase 3 で計測確認 |
| Canvas 絵文字レンダリングが環境依存（MF-1） | 中 | フォントフォールバック `Segoe UI Emoji`, `Apple Color Emoji` を追加 |
| トースト表示が HUD と重複（MF-2） | 高 | Y 座標を `H-100` に引き上げ、HUD 領域と分離 |
| quickReject 統合で衝突判定の挙動が変わる | 高 | 既存の物理テスト全パスを維持。`QUICK_REJECT_MARGIN = 2` で定数化（R-3） |
| 最適化の効果が計測上微小 | 中 | 2v2 + エフェクト多数のワーストケースで計測。効果が薄い場合はスキップ |

## 成功基準

1. VsScreen で P3/P4 の操作タイプが正しくラベル表示される
2. ゲームパッド接続/切断時に Canvas 上でトースト通知が表示・フェードアウトする
3. 2v2 モードで安定 60 FPS（または最適化前比で FPS 改善が計測可能）
4. 既存テスト全パス + 新規テスト追加
5. ESLint エラー 0 / TypeScript 型エラー 0
