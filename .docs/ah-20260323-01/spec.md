## S-04: 物理・速度パラメータ仕様

### 定数変更

| 定数 | 場所 | 変更前 | 変更後 | 理由 |
|------|------|--------|--------|------|
| `MAX_POWER` | constants.ts | 12 | 16 | パック最高速度をフィールド比率に合わせる |
| `MIN_SPEED` | constants.ts | 1.5 | 2.0 | 停止判定の閾値をフィールドに比例 |
| `CPU.easy` | constants.ts | 1.5 | 2.0 | AI 移動速度をフィールドに比例 |
| `CPU.normal` | constants.ts | 3.5 | 4.7 | 同上 |
| `CPU.hard` | constants.ts | 6.0 | 8.0 | 同上 |
| `KEYBOARD_MOVE_SPEED` | keyboard.ts | 6 | 8 | キーボード操作の体感速度を維持 |
| `SPEED_NORMAL` | renderer-utils.ts | 6 | 8 | パック速度の演出閾値 |
| `SPEED_FAST` | renderer-utils.ts | 10 | 13 | 同上 |
| `FRICTION` | constants.ts | 0.998 | 0.998 | **据え置き**（フレーム単位の減衰率） |

### AI 速度パラメータの考え方

フィールドが 1.33x に広がると、同じ速度では端から端への移動に 1.33 倍の時間がかかる。
ゲームバランスを維持するため、AI の最大速度も 1.33x に調整する。

```
新速度 = 旧速度 × (新幅 / 旧幅) = 旧速度 × 1.333
```

### FRICTION について

`FRICTION = 0.998` はフレーム単位の減衰率（毎フレーム速度が 0.2% 減衰）。
フィールドが大きくなっても「パックの減速感」はフレーム数に依存するため変更不要。
ただし `MIN_SPEED`（停止判定）は、フィールドが広い分、低速でも長く移動するため引き上げる。

## S-06: レンダラー修正仕様

### ハードコード座標の洗い出し

`renderer.ts` および `infrastructure/renderer/` 内で、以下のパターンのハードコード座標を特定し修正する:

| パターン | 例 | 対応方針 |
|---------|-----|---------|
| 外枠オフセット | `6`, `12` | フィールド幅比率で計算 or 据え置き（装飾なので影響小） |
| 中央座標 | `225`, `450` | `W/2`, `H/2` で計算（多くは対応済みの見込み） |
| 中央サークル半径 | `60` | `W * 0.1` として計算 |
| 中央ドット半径 | `8` | 据え置き or `W * 0.013` |
| ゴール LED 高さ | `6` | 据え置き（描画ピクセルの精度） |
| LED ドット間隔 | `12` | 据え置き |
| テキストサイズ | 各種 | フォントサイズをフィールド幅比率で計算 |

### 修正方針

1. `consts.CANVAS.WIDTH` / `consts.CANVAS.HEIGHT` を直接参照する既存のコードは変更不要
2. 数値リテラルで書かれた座標は、`W` / `H` ベースの計算式に置換
3. 装飾用の小さなオフセット（2〜12px）は据え置きでも視覚的に問題ない場合はそのまま

## S-07: CSS・UI 仕様

### styles.ts の変更

```typescript
// GameCanvas
max-width: 450px  →  max-width: 600px

// ScoreBoardContainer
max-width: 450px  →  max-width: 600px
```

### MenuCard の判断

- `max-width: 450px` は据え置き（メニューがゲームフィールド幅に合わせて広がる必要はない）
- ゲーム画面以外の UI は 450px 幅で統一感を維持

### レスポンシブ動作

- アスペクト比 1:2 は維持（`aspect-ratio: 1 / 2`）
- スマートフォン（幅 < 600px）: 幅いっぱいに縮小表示（既存動作と同じ）
- タブレット・PC: 最大 600px 幅で表示
- `max-height: calc(100vh - 100px)` は維持
