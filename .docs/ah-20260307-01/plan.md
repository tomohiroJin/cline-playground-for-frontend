# Air Hockey ブラッシュアップ計画

## 目標

ユーザーに「面白い」と思ってもらえるエアホッケーゲームに仕上げる。
「もう1回やりたい」と思わせるゲーム体験を実現する。

## 現状分析

### 既存機能（実装済み）

| カテゴリ | 内容 |
|---------|------|
| コアメカニクス | パック・マレットの物理演算、壁反射、ゴール判定 |
| AI | 3段階難易度（easy/normal/hard）、予測・スタック検出 |
| アイテム | Split（3分裂）、Speed（速度変化ゾーン）、Invisible（パック透明化） |
| フィールド | 6種（Original, Wide, Pillars, Zigzag, Fortress, Bastion） |
| 演出 | ネオングロー、パックトレイル、パーティクル、ゴールエフェクト、フィーバーモード |
| サウンド | Web Audio API による合成音（hit, wall, item, goal, lose, start） |
| UI | タイトル画面、スコアボード、リザルト画面、シェアボタン |
| 画面サイズ | 2段階の固定サイズ（300x600 / 450x900）を手動選択 |
| その他 | ハイスコア保存、ヘルプ画面 |

### 課題と不足点

#### 1. 画面サイズ・レスポンシブ対応の不足
- Canvas サイズが**固定値（300x600 / 450x900）の手動選択**で、画面に合わない
- スマートフォンでは小さすぎ、大画面では余白が多い
- Size 選択 UI が**直感的でなく使いづらい**
- スコアボードの幅が `max-width: 300px` 固定で Canvas と連動しない

#### 2. ゲームフィール（手触り）の不足
- ゴールや強打時の**画面シェイク**がない
- サウンドが単調な合成音のみで**BGM がない**
- ゲーム開始時の**カウントダウン演出**がない
- パックの速度に応じた**ビジュアルフィードバック**が弱い

#### 3. ゲームプレイの深みが不足
- アイテムが3種類のみで**バリエーションが少ない**
- **ポーズ機能**がない（中断不可）
- **コンボ/連続得点ボーナス**がフィーバーのみで単調
- 劣勢時の**カムバック要素**がない

#### 4. リプレイ性が低い
- **実績/アチーブメント**がない
- **戦績統計**（ヒット数、セーブ数等）がない
- リザルト画面の情報量が少ない
- **アンロック要素**がない

#### 5. UX の粗さ
- ヘルプが5秒無操作で表示される**静的テキスト**のみ
- **キーボード操作**に非対応
- 画面遷移に**トランジション**がない
- **音量調整**ができない

## 施策優先度

面白さへのインパクトと実装コストで優先度を決定する。

### Phase 1: 基盤改善＋ゲームフィール強化（高インパクト・低〜中コスト）
ユーザーが最初に感じる「遊びやすさ」と「手触り」を改善。これだけで体験が大きく変わる。

1. **レスポンシブデザイン対応**（画面サイズに自動フィット、Size 選択廃止）
2. **カウントダウン演出**（3, 2, 1, GO!）
3. **画面シェイク**（ゴール時・強打時）
4. **パック速度ビジュアル**（高速時の色変化・トレイル強化）
5. **BGM 追加**（Web Audio API でループ生成）
6. **サウンド改善**（ヒット音のバリエーション、音量に応じた変化）

### Phase 2: ゲームプレイ深化（高インパクト・中〜高コスト）
繰り返し遊ぶ動機を作る。

7. **ポーズ機能**
8. **新アイテム追加**（Shield: ゴール前バリア、Magnet: パック吸引、Big/Small: マレットサイズ変更）
9. **コンボシステム強化**（連続得点でスコアボーナス＋演出強化）
10. **カムバックメカニクス**（大差時に劣勢側のマレットが少し大きくなる等）
11. **試合統計**（ヒット数、最高速パック、セーブ数をリザルトに表示）

### Phase 3: リプレイ性・UX 向上（中インパクト・中コスト）
長期的な遊びの動機を追加。

12. **実績システム**（初勝利、連勝、ノーミス勝利等）
13. **リザルト画面強化**（統計表示、MVP 演出、リプレイボタン）
14. **画面トランジション**（フェードイン・アウト）
15. **音量設定**（BGM / SE 個別調整）
16. **チュートリアル改善**（インタラクティブなガイド）

### Phase 4: 発展的機能（低〜中インパクト・高コスト）
余裕があれば対応。

17. **キーボード操作対応**
18. **難易度オートアジャスト**（連敗で自動調整）
19. **フィールド/アイテムアンロック**
20. **デイリーチャレンジ**（特殊ルールでの対戦）

## 技術方針

- 既存のアーキテクチャ（core/ hooks/ components/ renderer）を維持
- 新機能は既存パターンに従って追加（型定義 → ロジック → 描画 → フック統合）
- テストは各 Phase 完了時に追加（既存テストの網羅率を維持）
- パフォーマンスに注意（requestAnimationFrame ループ内の処理を軽量に保つ）
- Canvas は内部解像度を固定し、CSS でビューポートに応答的にスケーリングする（物理演算の安定性とレスポンシブ性を両立）
- `CanvasSize` 型と Size 選択 UI は廃止し、コードを簡素化する
