# Deep Sea Interceptor 大規模改良計画

## 概要

Deep Sea Interceptor の9項目の改良要望に対する実装計画。
ゲームプレイの質向上、バランス調整、画面対応、テスト支援機能の追加を行う。

## 現状サマリ

| 項目 | 現在の値 |
|------|---------|
| Canvas サイズ | 400×560px 固定 |
| ステージ数 | 5 |
| ボス HP | 全て 40 |
| ボス出現スコア | Stage1: 3000, Stage2: 7000, ... |
| 武器種類 | トーピード / ソナーウェーブ / バイオミサイル |
| パワーアップアイテム | P, S, B, W, ★, ♥ の6種 |
| プレイヤー速度 | 4 px/frame（最大 7） |
| ホーミング連射 | 制限なし |
| テストモード | なし |

## 改良項目と実装方針

### Phase 1: 画面・速度の基盤調整

#### 1-1. 画面サイズの拡大とレスポンシブ対応（要望①）

**方針:** Canvas の論理サイズを拡大し、CSS でビューポートに合わせてスケーリング

- 論理サイズ: 800×1000px（現在の 2 倍相当）
- CSS で `max-width: 100vw; max-height: 100vh` + アスペクト比維持
- モバイル: 縦画面時はフル幅、横画面時はフル高さ
- 全エンティティの座標・サイズを論理サイズに合わせてスケーリング

**影響範囲:**
- `constants.ts` — Canvas サイズ、全座標系定数
- `DeepSeaInterceptorGame.tsx` — レスポンシブ CSS
- `components/TouchControls.tsx` — タッチ操作の座標変換
- `game-logic.ts` — スポーン座標、境界判定
- `enemy-ai.ts` — ボス移動範囲、弾パターン座標

#### 1-2. 基本速度の低下（要望⑥）

**方針:** 全エンティティの速度を 0.65 倍に統一調整

| エンティティ | 現在 | 変更後 |
|-------------|------|--------|
| プレイヤー基本速度 | 4.0 | 2.6 → 3.0（丸め） |
| プレイヤー最大速度 | 7.0 | 5.0 |
| basic 敵 | 1.8 | 1.2 |
| fast 敵 | 3.2 | 2.0 |
| shooter 敵 | 1.2 | 0.8 |
| 通常弾速 | 11.0 | 7.0 |
| 敵弾速 | 3.0-4.5 | 2.0-3.0 |
| アイテム落下速度 | （要確認） | 低速化 + 吸引範囲拡大 |

**追加対応:** アイテムの吸引範囲を拡大（取りやすくするため）

### Phase 2: ステージバランス調整

#### 2-1. ボス出現スコアの引き上げ（要望②）

**方針:** 各ステージのボス出現スコアを 2.5〜3 倍に引き上げ

| Stage | 現在 | 変更後 |
|-------|------|--------|
| 1 | 3,000 | 8,000 |
| 2 | 7,000 | 20,000 |
| 3 | 12,000 | 35,000 |
| 4 | 18,000 | 50,000 |
| 5 | 25,000 | 70,000 |

#### 2-2. 雑魚戦フェーズの充実

- スポーン間隔の調整（ステージ序盤は緩やかに、中盤から激化）
- ウェーブ制の導入検討（一定数の敵を倒すと次のウェーブ）
- アイテムドロップ頻度の微増（楽しさ向上）

### Phase 3: 武器バランス調整

#### 3-1. パワーアップアイテム全3武器対応確認（要望③）

**確認結果:** 現在の実装では P（パワー）アイテムは全武器共通で `power` を +1 する仕組み。
各武器は power レベルに応じて弾数・ダメージ・スプレッド角度を変化させている。

- **トーピード:** Power 1→5 で弾数 1→4、中央弾ダメージ 1.5
- **ソナーウェーブ:** Power 1→5 でスプレッド角度拡大、ダメージ 1.5→2.5、射程延長
- **バイオミサイル:** Power 1→5 で弾数 1→3

→ 3武器とも Power レベルの効果は存在する。ただしスプレッド(W)アイテムの効果が武器ごとに異なるか要検証。

#### 3-2. ホーミング連射制限（要望④）

**方針:** バイオミサイルに発射クールダウンを導入

- 通常射撃: 200ms クールダウン（現在は毎フレーム発射可能）
- チャージショット: 変更なし
- 他の武器にも適度なクールダウンを設定（トーピード: 120ms、ソナーウェーブ: 150ms）
- ボタン押しっぱなし時は自動連射だがクールダウンで制御

#### 3-3. ソナーウェーブの強化（要望⑤）

**方針:** 射程・溜め撃ち・威力を全面的に強化

| 項目 | 現在 | 変更後 |
|------|------|--------|
| 通常弾速 | 8 | 10 |
| 射程（Power1-2） | 28f（≈0.45s） | 45f（≈0.72s） |
| 射程（Power3-4） | 33f（≈0.53s） | 55f（≈0.88s） |
| 射程（Power5+） | 40f（≈0.64s） | 70f（≈1.12s） |
| ダメージ（Power1-2） | 1.5 | 2.0 |
| ダメージ（Power3-4） | 2.0 | 3.0 |
| ダメージ（Power5+） | 2.5 | 4.0 |
| チャージ弾数 | 8発 | 12発 |
| チャージダメージ | 4.0 | 6.0 |
| チャージ射程 | 32f | 50f |
| **新規: 貫通** | なし | チャージショットに貫通追加 |

### Phase 4: 演出強化

#### 4-1. ボス前警告演出（要望⑦）

**方針:** 既存の WARNING 表示を大幅強化

- 警告音（サイレン風 SE）を追加
- 画面端の赤い点滅エフェクト
- 「WARNING」テキストの拡大アニメーション
- ステージごとのボス名表示（例: 「WARNING - アンコウ・ガーディアン接近中」）
- 警告演出時間を 2000ms → 3000ms に延長
- BGM の一時停止 → 警告音 → ボス戦 BGM 切り替え

### Phase 5: ボス拡張

#### 5-1. ボス種類の追加（要望⑧）

**方針:** 既存 5 ボスに加え、各ステージにサブパターンを追加

- Stage 1-4: 既存ボスの攻撃パターンを 3 フェーズ化（現在は 2 フェーズ）
- ボス HP の引き上げ: 40 → Stage に応じて 60〜120

| Stage | ボス | HP | フェーズ数 |
|-------|------|-----|----------|
| 1 | アンコウ・ガーディアン | 60 | 3 |
| 2 | マインレイヤー | 70 | 3 |
| 3 | サーマルドラゴン | 80 | 3 |
| 4 | ルミナス・リヴァイアサン | 100 | 3 |
| 5 | アビサル・コア | 150 | **3段階変形** |

#### 5-2. 最終ボス 3段階変形（要望⑧）

**Stage 5 ボス「アビサル・コア」の 3 段階変形:**

**第1形態: 「外殻」（HP 150→100）**
- 外見: 巨大な球体、硬い外殻
- 攻撃: 他ボスパターンのランダム切替（現行と同じ）
- 弱点: 外殻が開く瞬間のみダメージ有効（3秒おきに2秒間）

**第2形態: 「内核露出」（HP 100→50）**
- 外見: 外殻が割れ、内部の有機的構造が露出
- 攻撃: 16方向回転弾幕 + 追尾レーザー（新規）
- 特殊: 小型雑魚を召喚（4体ずつ、10秒おき）

**第3形態: 「暴走コア」（HP 50→0）**
- 外見: コアがむき出し、赤く発光・振動
- 攻撃: 全方向高速弾幕（24方向）+ ホーミング弾 + 画面端からのレーザー壁
- 特殊: 移動速度上昇、攻撃間隔短縮（HP 減少に比例）
- 演出: 画面振動・赤フラッシュ

### Phase 6: テストモード

#### 6-1. 隠しコマンドによるテストモード（要望⑨）

**方針:** タイトル画面で「jin」を 3 回入力でテストモード起動

- 入力検知: タイトル画面でのキーボード入力を監視
- 「j」「i」「n」の順序で入力を検知、3 回繰り返しで発動
- テストモード効果:
  - 全パワーアップアイテムがフル（Power: 5, Speed: 3, Shield: 有効, Spread: 有効）
  - ライフ: 99
  - 画面に「TEST MODE」表示
- テストモードは localStorage に保存しない（セッション限り）

## 実装順序

```
Phase 1 (基盤) → Phase 2 (バランス) → Phase 3 (武器) → Phase 4 (演出) → Phase 5 (ボス) → Phase 6 (テスト)
```

各 Phase 完了後にテストを実行し、既存テストが通ることを確認。

## リスク・注意点

1. **画面サイズ変更**: 全座標系に影響するため、段階的に実施
2. **速度変更**: ゲームフィール全体に影響、テストプレイで要調整
3. **ボス HP 増加 + 速度低下**: ボス戦が冗長にならないようダメージバランスも調整
4. **テスト**: 既存テストで座標ハードコードされている箇所の更新が必要
