# Deep Sea Interceptor 大規模改良 — 仕様書

## 1. 画面サイズとレスポンシブ対応

### 1.1 論理サイズ

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| Canvas 幅 | 400px | 800px |
| Canvas 高さ | 560px | 1000px |
| スケール比 | - | 2.0x |

### 1.2 レスポンシブ挙動

| 画面 | 挙動 |
|------|------|
| PC（横長） | 中央配置、高さ基準でスケーリング、最大 100vh |
| タブレット（縦長） | 幅基準でスケーリング、最大 100vw |
| モバイル（縦） | 幅 100vw、アスペクト比維持 |
| モバイル（横） | 高さ 100vh、アスペクト比維持 |

### 1.3 スケーリング方式

```
CSS: object-fit: contain 相当のスケーリング
aspect-ratio: 4 / 5 (800:1000)
```

- タッチ座標は CSS スケール比で逆変換
- HUD テキストサイズも論理サイズ基準

### 1.4 座標系のスケーリング

全エンティティの座標・サイズに SCALE_FACTOR = 2.0 を適用：

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| プレイヤーサイズ | 20 | 40 |
| 敵基本サイズ | 28 | 56 |
| 弾サイズ | 6 | 12 |
| チャージ弾サイズ | 22 | 44 |
| スポーン Y 座標 | -40 | -80 |

| ボス振幅 | 80 | 160 |

## 2. 速度バランス調整

| shooter 敵 | 1.2 | 1.6 | 遅い |
| tank 敵 | 0.8 | 1.0 | 遅い |
| プレイヤー弾（トーピード） | 11.0 | 14.0 | 遅い |

## 5. ボス前警告演出

### 5.1 演出タイムライン

```
T+0ms    : 雑魚敵スポーン停止
T+0ms    : 警告 SE 開始（サイレン音、ピッチ上昇）
T+0ms    : 画面端に赤い点滅エフェクト開始
T+500ms  : 「⚠ WARNING ⚠」テキスト表示（拡大→収縮アニメ）
T+1500ms : ボス名テキスト表示（フェードイン）
T+3000ms : 演出終了、ボス出現
```

### 5.2 警告 SE 仕様

- Web Audio API で生成
- 低音のパルス音（200Hz）を 0.5 秒間隔で 3 回
- 最後にピッチ上昇のスイープ音（200Hz → 800Hz, 0.5s）

## 7. テストモード

### 7.1 起動条件

タイトル画面で以下のキー入力を検知:

```
j → i → n → j → i → n → j → i → n
（「jin」を3回連続入力）
```

- 入力バッファ: 最新 9 文字を保持
- 判定: バッファが "jinjinjin" と一致
- 途中で間違えた場合: バッファリセットなし（スライディングウィンドウ方式）
- 入力受付: タイトル画面表示中のみ

### 7.2 テストモード効果

| 項目 | 通常 | テストモード |
|------|------|------------|
| ライフ | 3（STANDARD） | 99 |
| パワー | 1 | 5（最大） |
| スピード | 0 | 3（最大） |
| シールド | なし | 有効（無期限） |
| スプレッド | なし | 有効（無期限） |

### 7.3 UI 表示

- テストモード発動時: SE（ファンファーレ）+ 「TEST MODE ACTIVATED」メッセージ（2秒）
- プレイ中: 画面右上に「TEST」バッジ表示（半透明）
- スコアはハイスコアに記録しない

### 7.4 永続性

- テストモードのフラグはセッション限り（localStorage に保存しない）
- タイトルに戻ると解除
- 毎回コマンド入力が必要

## 8. プレイテストフィードバック修正（2026-03-24）

### 8.1 Boss5 外殻が開かないバグ修正

**原因:** `updateSingleEnemy` 内でフェーズ遷移チェックが Boss5 特殊処理より先に実行され、bossPhase=1 の外殻ロジックが到達不能だった。
**修正:** Boss5 特殊処理をフェーズ遷移チェックの前に移動。

### 8.2 テストモードでハート取得時のライフクランプ修正

**原因:** `processItemCollection` で `Math.min(maxLives, lives+1)` により 99→5 にクランプ。
**修正:** `UiState` に `testMode: boolean` を追加。テストモード時はクランプをスキップ。

### 8.3 ソナーウェーブのナーフ

| 項目 | 修正前 | 修正後 |
|------|--------|--------|
| Power1-2 damage | 2.0 | 1.5 |
| Power3-4 damage | 3.0 | 2.0 |
| Power5+ damage | 4.0 | 2.5 |
| Power5+ count | 7 | 5 |
| チャージ弾数 | 12 | 8 |
| チャージダメージ | 6.0 | 4.0 |
| クールダウン | 150ms | 200ms |

修正後 DPS: 5発×2.5=12.5/shot、200ms クールダウンで秒間 62.5DPS。

### 8.4 トーピード スプレッド(W) 5WAY 化

**修正前:** 3WAY `[-0.25, 0, 0.25]` — ソナーウェーブと見た目が同じ
**修正後:** 5WAY `[-0.3, -0.15, 0, 0.15, 0.3]`、Power5+W 時は中央弾追加で 6 発
