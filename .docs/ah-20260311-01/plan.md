## P0-01: 世界観設定の策定

### 概要
ゲームの舞台となる学校・エアホッケー部・大会制度を設計する。
Phase 1 の背景画像制作、および第2章以降のストーリー展開の土台になる。

### 作業内容

1. **学校の基本設定を決める**
   - 学校名（正式名称 + 略称）
   - 種別（公立/私立）、規模（生徒数の目安）
   - 校風のイメージ（「自由で活気がある」等）
   - 所在地のイメージ（背景に都市が見えるか、田舎か等）

2. **エアホッケー部の設定を決める**
   - 部の歴史（新設なのか伝統があるのか）
   - 部員数と構成（5名の少数精鋭: 主人公 + 第1章の3名 + 新入生の解説役）
   - 活動場所の具体的イメージ（部室の広さ、設備）
   - 顧問の有無（名前までは不要だが、存在するかどうか）

3. **大会制度を設計する**
   - 地区大会 → 県大会 → 全国大会の構造
   - 個人戦 + ペア戦（協力プレイ）の2種目制
   - 大会の規模感（参加校数の目安）
   - 第1章の時点で大会はまだか、すでに始まっているか

4. **時系列を整理する**
   - 第1章 = 入部（春・4月）
   - 第2章 = 想定時期
   - 年間スケジュールの概要

### 成果物
- `world-setting.md`（`src/features/air-hockey/doc/world/` に配置）

### 完了条件
- [ ] 学校・部活・大会制度の設定が文書化されている
- [ ] 背景画像のイメージにつながる場所の描写がある
- [ ] 第2章以降の展開を自然に受け入れられる構造になっている

## P0-02: 既存キャラクター設定の深掘り

### 概要
既存の7キャラクター（主人公 + ストーリー3名 + フリー対戦3名）の詳細プロフィールを作成する。
既存コード（`characters.ts`, `dialogue-data.ts`, `image-prompts.md`）と矛盾しないよう注意する。

### 作業内容

1. **主人公アキラの詳細設定**
   - フルネーム（姓を追加）
   - 学年: 1年生（入部したての設定と整合）
   - 基本情報（誕生日、身長、年齢）
   - 性格の深掘り（3つのキーワード）
   - エアホッケーを始めた動機
   - 既存リアクション（`よし！`, `くっ…！` 等）と一致する口調の定義
   - 外見: 既存プロンプト「黒髪ショート、茶色い目、白+青ウェア」を踏襲

2. **ヒロの詳細設定**
   - フルネーム
   - 学年: 2年生（「先輩」とダイアログで呼ばれている）
   - 基本情報
   - 性格: 「明るい」「面倒見がいい」を軸に深掘り
   - 動機と目標
   - 弱点・コンプレックス
   - プレイスタイル: 「ストレートショット」→ AI の `wobble: 40px` との対応
   - 他キャラとの関係（ミサキ・タクマとの関わり）
   - 外見: 既存プロンプト「茶髪ハネ髪、緑目、オレンジウェア」を踏襲

3. **ミサキの詳細設定**
   - フルネーム
   - 学年: 2年生（ヒロが「ミサキ先輩」と呼んでいる）
   - 基本情報
   - 性格: 「テクニカル」「知的」を軸に深掘り
   - 口調: 既存ダイアログ「〜わよ」「〜ね♪」との整合
   - プレイスタイル: 「テクニカル」→ AI の高予測 + アイテム活用との対応
   - 外見: 既存プロンプト「紫黒ポニテ、紫目、紫ウェア」を踏襲

4. **タクマの詳細設定**
   - フルネーム
   - 学年: 3年生（「部長」の設定と整合）
   - 基本情報
   - 性格: 「威厳」「厳しい」を軸に深掘り
   - 口調: 既存ダイアログ「〜ぞ」「来い」との整合
   - プレイスタイル: 「パワー + 壁反射」→ AI の `wallBounce: true` との対応
   - 外見: 既存プロンプト「黒髪オールバック、赤茶目、赤ウェア」を踏襲

5. **5人目の部員（新入生・解説役）の設定**
   - アキラと同じ新入生で、解説役としてストーリーに関わる
   - フルネーム・基本情報・性格キーワード（3つ）
   - エアホッケー部での立ち位置（プレイヤーではなく分析・サポート寄り）
   - 将来的にフリー対戦キャラとして戦える隠しモードの構想メモ

6. **フリー対戦キャラ3名の設定**
   - ルーキー・レギュラー・エースは部外のキャラとして簡易プロフィールを作成
   - フルネーム・性格のキーワード（2つ）
   - 既存リアクションとの口調整合
   - ※ 隠しモードとして将来実装予定

6. **キャラクター関係図の作成**
   - 全キャラの関係性をテキストベースで整理
   - 先輩後輩、友人、ライバル等の関係線

### 参照すべき既存コード

| ファイル | 確認ポイント |
|---------|-------------|
| `core/characters.ts` | ID, 名前, カラー, リアクション |
| `core/dialogue-data.ts` | ダイアログのセリフ内容・口調 |
| `.docs/ah-20260308-01/image-prompts.md` | 外見（髪色・目色・服装）の記述 |
| `core/story-balance.ts` | AI パラメータ（プレイスタイルとの対応） |

### 成果物
- `character-profiles.md`（前半: 既存7キャラ）

- [ ] 既存コードの設定と矛盾がない（カラー、口調、外見）

## P0-03: 第2章プロット・新キャラ設計

### 概要
第2章のストーリー展開案と、登場する新キャラクターの概要を設計する。
完成版のセリフや AI パラメータは不要。方向性とキャラの骨格を固めることが目的。

### 作業内容

1. **第2章のテーマと構造を決める**
   - 章のタイトル案
   - テーマ（例: 「初めての公式戦」「ライバル校との遭遇」「部の存続危機」等）
   - 第1章からの接続: タクマの「次は地区大会だ」を受けた展開
   - 全体のトーン（友情・努力をベースに維持しつつ、緊張感を追加）

2. **ステージ構成を設計する**
   - ステージ数: 3〜4
   - 各ステージの概要:
     - ステージID と仮タイトル
     - 対戦相手（新キャラ or 既存キャラの再戦）
     - 使用フィールド（既存6フィールドから or 新フィールドの必要性）
     - 難易度レンジ（Normal〜Hard+）
     - ストーリー上の位置づけ（何が起きるか）
   - ダイアログの方向性（要点のみ、セリフ本文は不要）

3. **新キャラクター3名の概要設計**
   - 各キャラの概要:
     - 名前（仮）
     - 所属（他校 or 同校 or その他）
     - 性格のキーワード（3つ）
     - プレイスタイルの方向性
     - 主人公やヒロたちとの関係性
     - テーマカラー案
   - ※ D-02 と同等レベルの詳細は第2章実装時に確定する

4. **第1章キャラの第2章での役割**
   - ヒロ: 主人公の応援役？ 自身も大会に出場？
   - ミサキ: 大会でのアドバイザー？ 別ブロックで戦っている？
   - タクマ: 部長としてチームを率いる？ 引退が近い？

5. **第3章へのフックを仕込む**
   - 第2章の終わりに「続きが気になる」要素を入れる
   - 例: 全国大会への切符、謎のライバルの登場、部長の引退宣言 等

### 成果物
- `chapter2-plot.md`
- `character-profiles.md`（後半: 新キャラ3名の概要追記）

### 完了条件
- [ ] 第2章のテーマ・構造・ステージ構成が文書化されている
- [ ] 新キャラ3名の概要プロフィールがある
- [ ] 既存キャラの第2章での役割が定義されている
- [ ] 第3章へのフックが1つ以上ある
- [ ] 既存フィールド6種から使用するものが選定されている（新フィールドが必要な場合はその要件も記載）

## P0-04: 画像生成スタイルガイドの策定

### 概要
Phase 1 で実際に画像を生成する際の指針を策定する。
既存のアイコン（128x128 チビキャラ）とのスタイル接続、立ち絵・背景・VS用イラストの統一ルールを定める。

### 作業内容

1. **既存アイコンのスタイル分析**
   - 既存 `image-prompts.md` のプロンプト構造を確認
   - アイコンの特徴（チビ、1.5頭身、セルシェーディング）を整理
   - 「アイコンから立ち絵に拡張する際に何を変え、何を維持するか」を定義

2. **立ち絵プロンプトの設計**
   - 共通プロンプト要素（アートスタイル、品質指定、ネガティブプロンプト）
   - スタイル変更点: チビ（1.5頭身）→ 5〜6頭身への変更
   - 各キャラの立ち絵プロンプト案（D-02 の外見設定をベースに）
   - 表情差分の生成方法（同一ベースで表情のみ変更）
   - ファイル命名規則: `{キャラID}-{表情ID}.png`（例: `hiro-normal.png`, `hiro-happy.png`）

3. **背景画像プロンプトの設計**
   - 3枚分のプロンプト案:
     - `bg-clubroom`: 部室（D-01 の部室設定に基づく）
     - `bg-gym`: 体育館（大会会場のイメージ）
     - `bg-school-gate`: 校門と桜並木
   - 共通スタイル要素（アニメ背景風、パステルトーン等）
   - ゲーム内での表示方法（全画面表示、暗転オーバーレイとの組み合わせ）

4. **VS画面用イラストの設計**
   - 立ち絵との差分（ポーズ、表情、構図）
   - Phase 1 では立ち絵トリミングで代用する方針を明記
   - 将来的な専用イラスト制作時の仕様

5. **勝利カットインの設計**
   - 第1章クリア時の1枚絵の構図案
   - プロンプト案

6. **ファイル配置とサイズの仕様表**
   - 全画像アセットのファイル名、サイズ、フォーマットの一覧
   - 配置ディレクトリ: `public/assets/` 以下の構造

### 参照すべき既存データ

| 参照元 | 用途 |
|--------|------|
| `.docs/ah-20260308-01/image-prompts.md` | 既存アイコンプロンプト |
| `public/assets/characters/*.png` | 既存アイコン画像の実物 |
| D-01（`world-setting.md`） | 背景のイメージソース |
| D-02（`character-profiles.md`） | 各キャラの外見設定 |

### 成果物
- `image-style-guide.md`

### 完了条件
- [ ] 既存アイコンスタイルとの接続方針が明記されている
- [ ] 全キャラ（7名 + 第2章新キャラ概要）の立ち絵プロンプト案がある
- [ ] 背景3枚のプロンプト案がある
