# Phase 0: 世界観・キャラクター設定整備 — 仕様書

## 目的

Phase 1（ストーリービジュアル強化）で立ち絵・背景画像・演出を制作する際の**設計基盤**を整える。
設定が曖昧なままビジュアルを作ると、キャラの印象がブレる・場面に合わない絵になる等の手戻りが発生する。
Phase 0 のアウトプットは**ドキュメント（`.docs/` 配下）のみ**で、コード変更は行わない。

## D-01: 世界観設定書

### 必須項目

#### 1. 舞台設定
- **学校名**: 正式名称と略称
- **学校の特徴**: 公立/私立、規模感、校風（進学校 or スポーツ強豪 or 自由な校風 等）
- **所在地**: 都市部/郊外（背景画像のイメージに影響）
- **時代**: 現代（西暦何年頃のイメージか）

#### 2. エアホッケー部の設定
- **部の規模**: 部員数（5名の少数精鋭）、活動頻度
- **部の位置づけ**: 学校内での認知度（マイナー部活 or 有名 or 新設等）
- **部室の雰囲気**: 内装のイメージ（Phase 1 の背景画像に直結）
- **大会制度**: 地区大会 → 県大会 → 全国大会。個人戦 + ペア戦（協力プレイ）の2種目制
- **練習場所**: 部室なのか体育館なのか専用ルームなのか

#### 3. ストーリーの時系列
- 第1章の時期（入部直後 = 何月頃か）
- 各章がどの季節にあたるか（背景画像の季節感に影響）

### 設計方針
- **リアルすぎない**: エアホッケーの競技としてのリアリティは追求しない。「楽しさ優先」の世界観
- **少年漫画的なトーン**: 友情・努力・成長がテーマ。暗い展開は避ける
- **拡張可能性**: 他校の存在・大会制度を設計しておくことで、第2章以降の舞台を自然に広げられるようにする

## D-02: キャラクター設定書

### 対象キャラクター

#### 部員（5名）

| カテゴリ | キャラ名 | 既存情報の有無 |
|---------|---------|-------------|
| 主人公 | アキラ | 名前・色・リアクションのみ |
| ストーリー | ヒロ | 名前・色・リアクション・ダイアログ |
| ストーリー | ミサキ | 同上 |
| ストーリー | タクマ | 同上 |
| 新規部員 | 5人目（新入生・解説役） | なし（P0-02 で新規設定） |

#### フリー対戦キャラ（3名・部外）

| カテゴリ | キャラ名 | 既存情報の有無 | 備考 |
|---------|---------|-------------|------|
| フリー対戦 | ルーキー | 名前・リアクションのみ | 隠しモードとして将来実装予定 |
| フリー対戦 | レギュラー | 同上 | 同上 |
| フリー対戦 | エース | 同上 | 同上 |

#### 第2章用（新規3名程度）

### 各キャラクターの必須項目

#### 基本情報
- **フルネーム**: 姓名
- **学年**: 1年 / 2年 / 3年
- **年齢**: 15-18歳
- **誕生日**: 月日（キャラ図鑑の情報として）
- **身長**: cm 目安（立ち絵のサイズ感に影響）

#### 外見
- **髪型・髪色**: 既存アイコンプロンプトとの整合性を保つ
- **目の色**: 同上
- **体格**: 細身 / 普通 / がっしり
- **服装**: ユニフォームの色 + 私服のイメージ
- **特徴的なアクセサリ**: バンダナ、リストバンド等（キャラの識別に寄与）

#### 性格・内面
- **性格のキーワード**: 3つ程度（例: 「明るい」「負けず嫌い」「面倒見がいい」）
- **口調**: 既存リアクションとの整合性（例: ヒロ = カジュアルな男言葉）
- **動機**: なぜエアホッケーをしているのか
- **目標**: エアホッケーにおける目標（全国大会、楽しむ、部長を倒す等）
- **弱点・コンプレックス**: キャラに深みを出す要素

#### プレイスタイル
- **得意技の名前**: 1つ（VS画面・図鑑で表示する用途）
- **戦い方の特徴**: AI の振る舞いと整合性を取る
  - ヒロ: ストレート主体 → AI の `wobble: 40px`（狙いが大きくブレる＝ストレートで直線的だが荒い）
  - ミサキ: テクニカル → AI の `predictionFactor: 4` + アイテム活用
  - タクマ: パワー＋壁反射 → AI の `wallBounce: true` + 高速
- **弱点（ゲーム的）**: 「ミサキはパワーが弱い」等、プレイヤーの攻略ヒントにもなる設定

#### 人間関係
- **他キャラとの関係**: ヒロとミサキは幼馴染、タクマはヒロの憧れ、等
- **主人公との関係の変化**: 対戦前 → 対戦後でどう変わるか

### 既存コードとの整合性チェックリスト

以下の既存データと矛盾しないこと:

| 項目 | ソース | 確認事項 |
|------|--------|---------|
| キャラID | `characters.ts` | `player`, `hiro`, `misaki`, `takuma`, `rookie`, `regular`, `ace` |
| テーマカラー | `characters.ts` | アキラ=#3498db, ヒロ=#e67e22, ミサキ=#9b59b6, タクマ=#c0392b |
| リアクション | `characters.ts` | 口調・語尾の一貫性 |
| ダイアログ | `dialogue-data.ts` | 既存セリフの内容と矛盾しないこと |
| アイコンプロンプト | `image-prompts.md` | 外見描写の整合性（髪色、目の色、服装） |
| AI 設定 | `story-balance.ts` | プレイスタイルと AI パラメータの対応 |

## D-03: 第2章プロット概要

### 必須項目

#### ストーリー構造
- **章のタイトル**: 仮タイトル
- **テーマ**: 第1章「入部・仲間」→ 第2章は何か（例: 「初めての大会」「ライバルとの出会い」）
- **ステージ数**: 3〜4ステージ
- **対戦相手**: 新キャラ3名程度のプロフィール概要

#### 各ステージの概要
- **ステージID**: `2-1`, `2-2`, `2-3` (+ `2-4` の可能性)
- **ステージ名**: 仮タイトル
- **対戦相手**: 誰と戦うか
- **フィールド**: どのフィールドを使うか（既存6フィールドから選択 or 新フィールド）
- **難易度**: Normal〜Hard の範囲
- **ストーリー上の位置づけ**: この試合が物語にどう影響するか
- **ダイアログの方向性**: 試合前後の会話の要点（セリフの完成版は不要）

#### 新キャラクターの概要
- D-02 と同等の項目を概要レベルで記載（詳細は第2章実装時に確定）
- **他校のキャラ or 同校の隠れた実力者 or 卒業生** 等の立ち位置

### 設計方針
- 第1章の「部内での成長」から舞台を広げる
- 第1章のキャラ（ヒロ・ミサキ・タクマ）が第2章でも何らかの形で関わる
- 第2章クリア後に「第3章が楽しみ」と思わせるフックを残す

## D-04: 画像生成スタイルガイド

### 必須項目

#### 1. 既存アイコンとの統一方針
- 既存の 128x128 チビキャラアイコンのスタイルを分析
- 立ち絵への拡張時にどの要素を維持し、どの要素を変更するか明記
- **原則**: アイコンを見た人が「同じキャラだ」と認識できること

#### 2. 立ち絵の仕様

| 項目 | 仕様 |
|------|------|
| サイズ | 512×1024px（表示時は縮小） |
| フォーマット | PNG（透過背景） |
| 構図 | 膝上〜全身（キャラによって調整） |
| 頭身 | 5〜6頭身（アイコンの1.5頭身から変更） |
| 表情差分 | 最低2パターン（通常 / 感情変化） |

#### 3. 表情差分の定義

各キャラクター共通で以下の表情パターンを用意:

| ID | 表情名 | 用途 |
|----|--------|------|
| `normal` | 通常 | ダイアログ（デフォルト）、VS画面、図鑑 |
| `happy` | 喜び | 勝利時ダイアログ、得点リアクション |
| `angry` | 怒り/闘志 | 挑発時、負けそうな時 |
| `surprised` | 驚き | 失点時、予想外の展開 |

- Phase 1 では最低限 `normal` + `happy` の2表情からスタート
- 将来的に `angry`, `surprised` を追加可能な構造にする

#### 4. 背景画像の仕様

| 項目 | 仕様 |
|------|------|
| サイズ | 450×900px（ゲームの内部解像度に合わせる） |
| フォーマット | WebP（ファイルサイズ軽減） |
| 枚数（Phase 1） | 3枚（部室 / 体育館 / 校門・桜） |

#### 5. 場面別背景の定義

| ID | 場面 | 使用箇所 | イメージ |
|----|------|---------|---------|
| `bg-clubroom` | 部室 | ステージ1-1, 1-2 の試合前後ダイアログ | エアホッケー台が2台ある部室。窓から光が差す |
| `bg-gym` | 体育館 | ステージ1-3 の試合前後ダイアログ | 広い体育館に本格的なエアホッケー台。観客席あり |
| `bg-school-gate` | 校門（桜） | チャプタータイトル、TO BE CONTINUED | 桜並木の校門。春の入部シーズンのイメージ |

- 第2章用の背景（対戦会場等）は D-03 の内容に基づいて後から追加

#### 6. 立ち絵プロンプトの構造テンプレート

```
[アートスタイル], [キャラの外見], [表情], [服装], [ポーズ],
[構図], [背景指定], [品質指定]
```

- 各キャラクターのプロンプトは D-02 の外見設定をベースに生成
- **セッション統一**: 全キャラの立ち絵は同一の生成セッション（または同一モデル＋同一設定）で作成し、スタイルを統一

#### 7. VS画面用イラストの仕様

| 項目 | 仕様 |
|------|------|
| サイズ | 256×512px |
| フォーマット | PNG（透過背景） |
| 構図 | 半身（腰上）、やや正面向き |
| 表情 | 闘志のある表情（VS用特別表情） |
| ポーズ | 戦闘前の構え（腕組み、拳を前に等） |

- 立ち絵と同一キャラだが、VS画面用はポーズと表情が異なる
- Phase 1 では立ち絵のトリミングで代用し、余裕があれば専用イラストを作成

#### 8. 勝利カットインの仕様

| 項目 | 仕様 |
|------|------|
| サイズ | 450×400px |
| フォーマット | PNG |
| 枚数（Phase 1） | 1枚（第1章クリア時用） |
| 内容 | アキラが拳を突き上げるポーズ + 他キャラが背景にいる構図 |

## 設計制約

### やらないこと
- **コード変更**: Phase 0 はドキュメントのみ。`characters.ts` や `dialogue-data.ts` の変更は Phase 1 以降
- **画像生成の実行**: Phase 0 ではプロンプト設計まで。実際の生成は Phase 1
- **既存設定の破壊的変更**: 既存のキャラID・カラー・リアクションは変更しない。設定は「拡張」のみ
- **第2章の完成版データ**: D-03 はあくまで概要レベル。セリフや AI パラメータの詳細は Phase 1 以降

### 整合性の担保
- D-02 の外見設定は既存の `image-prompts.md` と矛盾しないこと
- D-02 の口調設定は既存の `characters.ts` リアクションおよび `dialogue-data.ts` セリフと矛盾しないこと
- D-04 のスタイルガイドは既存アイコン画像のスタイルを踏襲すること
