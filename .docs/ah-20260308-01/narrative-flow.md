# Air Hockey ブラッシュアップ ナラティブフロー v2

## MVP でのユーザー体験

MVP で実現する体験は以下の2つ:
1. **見た目・手触りの改善** → 「触っていて気持ちいい」
2. **ストーリーモード** → 「次のステージに進みたい」

---

## フロー A: 初回ユーザーの体験（見た目が効く）

```
[起動]
  ↓
  タイトル画面が表示される
  ┌──────────────────────────────┐
  │   AIR HOCKEY                  │
  │                                │
  │   📖 ストーリー               │  ← 新規追加
  │   🎮 フリー対戦               │  ← 既存 START の名称変更
  │                                │
  │   Difficulty: [Easy|Normal|Hard]│
  │   Field: [Original ...]        │
  │   Win Score: [3|7|15]          │
  │                                │
  │   🏆 実績  ? ヘルプ  ⚙ 設定   │
  └──────────────────────────────┘

  → ユーザーの判断分岐:
     「ストーリーって何だろう？」→ フロー B へ
     「とりあえず遊んでみよう」→ フロー C へ
```

### B-4: 試合（見た目改善 + キャラリアクション）

```
[VS 画面フェードアウト]
  ↓
  カウントダウン: 3 → 2 → 1 → GO!
  ↓
  試合開始

  【見た目改善が効く瞬間】

  時間 0:00 ─ 台が綺麗
  │  ・外枠のグラデーション → 「おっ、質感がある」
  │  ・フィールドの照明効果 → 「本物のテーブルみたい」
  │  ・LED風のゴールライン → 「光ってる！」
  │  ・マレットが立体的 → 「触りたい」
  │
  時間 0:05 ─ 最初のヒット
  │  ・パックの金属質感 → 「重みがある」
  │  ・ヒット音 + 速度変化 → 「いい音」
  │
  時間 0:10 ─ 強打！
  │  ・速度 > 8 → ヒットストップ発動！
  │  ・画面が一瞬止まる → 衝撃波リング
  │  → ユーザー: 「おお！ 気持ちいい！」
  │
  時間 0:15 ─ ゴール！
  │  ・パックがゴールラインへ
  │  → スローモーション（0.3倍速、400ms）
  │  → ビネット（四隅が暗くなる）
  │  → パーティクル + シェイク
  │  → スコアボード: ヒロ 0 - 1 アキラ
  │  → 吹き出し: ヒロ「うわっ！」
  │  → ユーザー: 「よっしゃ！ ヒロの反応かわいい」
  │
  時間 0:25 ─ ヒロの反撃
  │  → スローモーション → パーティクル
  │  → スコアボード: ヒロ 1 - 1 アキラ
  │  → 吹き出し: ヒロ「へへっ！」
  │  → ユーザー: 「くっ、負けない！」
  │
  時間 1:00 ─ 決着！
  │  → 最後のゴール → スローモーション → シェイク
  │  → ユーザー: 「勝った！」（or「負けた…悔しい！」）
  │
  試合終了
```

  │         ★ ヒロかわいい       ★ ゴールスローモーション！

## Phase 間の依存関係

```
Phase 1（見た目改善）
  │
  ├── P1-01〜P1-07: 描画改善（独立、並行可能）
  │
  ├── P1-08〜P1-10: ヒットストップ（順序依存）
  │    types.ts → useGameLoop.ts → renderer.ts
  │
  └── P1-11〜P1-13: スローモーション（順序依存）
       types.ts → useGameLoop.ts → renderer.ts

Phase 2（キャラクター基盤）← Phase 1 完了後
  │
  ├── P2-01〜P2-02: 型・データ定義（並行可能）
  │
  ├── P2-03〜P2-04: 画像プロンプト設計 → 画像生成
  │
  ├── P2-05〜P2-06: Scoreboard + リアクション
  │
  └── P2-07〜P2-08: ストーリー進行 + ダイアログデータ

Phase 3（ストーリー UI）← Phase 2 完了後
  │
  ├── P3-01〜P3-02: タイトル画面変更 + 状態管理
  │
  ├── P3-03: ステージ選択画面
  │
  ├── P3-04: ダイアログオーバーレイ
  │
  ├── P3-05: VS 画面
  │
  └── P3-06〜P3-07: 遷移統合 + リザルト調整

Phase 4（統合テスト）← Phase 3 完了後
```

---

## なぜこの順番なのか

1. **Phase 1 が最初**: 見た目と手触りは「全モードに効く基盤」。ストーリーモードでもフリー対戦でも、台・マレット・パックの質感とヒットストップ・スローモーションが体験の土台になる。

2. **Phase 2 が次**: キャラクターデータと画像を先に準備しないと、Phase 3 の画面が作れない。特に **画像生成は時間がかかる**（プロンプト調整 → 生成 → 選定 → リサイズ）ので早めに着手。

3. **Phase 3 が最後**: データ（キャラ、ステージ、ダイアログ）が揃った状態で UI を組み立てる。データ変更による手戻りを最小化。

4. **Phase 4 で仕上げ**: 全体を通したテストとバランス調整。
