# Air Hockey ブラッシュアップ計画 v2

## ゴール

**ユーザーが「面白い、もう1回やりたい」と感じるゲーム体験を作る。**

そのために必要な3つの柱:
1. **第一印象**: 見た目で「ちゃんとしたゲーム」と感じさせる
2. **目的**: 「次のステージに進みたい」という動機を与える
3. **共有**: 「友達と一緒に遊べる」体験を提供する

---

## MVP 定義

### MVP のスコープ

**「見た目が綺麗なエアホッケーで、キャラ付き3ステージのストーリーが遊べる」** 状態を MVP とする。

| 含む | 含まない |
|------|---------|
| 台・マレット・パックの見た目改善 | 2人対戦モード |
| ヒットストップ・ゴールスローモーション | オンライン対戦 |
| CPU キャラクター（名前・リアクション） | 漫画風コマ割りカットイン |
| ストーリーモード第1章（3ステージ） | 第2章以降のストーリー |
| ステージ選択画面 | チャレンジモード |
| 試合前後のダイアログ（テキスト+アイコン） | フルイラストの漫画画像 |
| VS 画面（対決演出） | ボス固有 AI |
| キャラクターアイコン画像（AI生成） | フル立ち絵 |

### MVP の判定基準

1. 初めてのユーザーが「ストーリー」を選んで3ステージ遊べる
2. 各ステージに対戦相手のキャラクターがいて、試合前後に会話がある
3. 台・マレット・パックの見た目がフラットな円ではなく立体的に見える
4. 強打時にヒットストップ、ゴール時にスローモーションが発動する
5. 既存のフリー対戦モードが壊れていない

## ユーザーストーリー

### Epic 1: 見た目の改善（ビジュアルブラッシュアップ）

#### US-1.1: 台の質感
> ユーザーとして、エアホッケーの台がリアルに見えてほしい。
> なぜなら、見た目がチープだと遊ぶ前に離脱してしまうから。

**受け入れ基準**:
- [ ] 台の外枠にグラデーション（木目風の暗い茶〜黒）がある
- [ ] フィールド面に放射グラデーション（中央が明るく端が暗い照明効果）がある
- [ ] 中央ラインが装飾的（二重線 + 中央円の装飾）になっている
- [ ] ゴールエリアが LED 風に発光している
- [ ] 6つの全フィールドで表示が崩れない

#### US-1.2: マレットの立体感
> ユーザーとして、マレットが平面の円ではなく立体的に見えてほしい。
> なぜなら、操作対象が魅力的だと触りたくなるから。

**受け入れ基準**:
- [ ] マレットに放射グラデーション（球体感）がある
- [ ] 左上にハイライト（光沢スポット）がある
- [ ] マレット下に楕円のドロップシャドウがある
- [ ] Big アイテム効果時にグローが強化される
- [ ] プレイヤー（青）と CPU（赤）で色味が正しく適用される

#### US-1.3: パックの金属質感
> ユーザーとして、パックが金属的な質感に見えてほしい。
> なぜなら、打つ対象に重みを感じたいから。

**受け入れ基準**:
- [ ] パックにメタリックなグラデーション（反射感）がある
- [ ] 速度に応じた色変化が維持されている（白→黄→赤）
- [ ] 超高速時に既存のグロー＋追加の光芒がある

#### US-1.4: ヒットストップ
> ユーザーとして、パックを強く打った瞬間に「ガツン」とした手応えを感じたい。
> なぜなら、打撃感が気持ちよさに直結するから。

**受け入れ基準**:
- [ ] パック速度 > 8 のヒット時に 50ms の物理更新停止が発生する
- [ ] ヒットストップ中に衝突地点から衝撃波リング（半透明の拡がる円）が描画される
- [ ] 描画は継続される（画面がフリーズしたようには見えない）
- [ ] SE（ヒット音）は通常通り再生される

#### US-1.5: ゴールスローモーション
> ユーザーとして、ゴールが決まる瞬間がドラマチックに演出されてほしい。
> なぜなら、得点の達成感を最大化したいから。

**受け入れ基準**:
- [ ] パックがゴールラインを通過した瞬間に 400ms のスローモーション（0.3倍速）が発動する
- [ ] スロー中に画面四隅にビネット（暗くなる）効果がある
- [ ] スロー終了後に既存のゴールエフェクト（パーティクル + シェイク）が発動する
- [ ] 自分の得点でも相手の得点でもスローモーションが発動する

### Epic 2: キャラクター・ストーリーモード

#### US-2.1: CPU キャラクター
> ユーザーとして、対戦相手に名前と個性があってほしい。
> なぜなら、「CPU」より「ヒロ」と戦う方が感情移入できるから。

**受け入れ基準**:
- [ ] フリー対戦で CPU に難易度別のキャラクター名が表示される
- [ ] スコアボードの「CPU」がキャラ名に変わる
- [ ] ゴール時にキャラクターのリアクションテキスト（吹き出し）が 1.5 秒表示される
- [ ] リアクションはキャラごとに異なるテキストセット（得点時/失点時/勝利時/敗北時 各2-3パターン）

#### US-2.2: ストーリーモード選択
> ユーザーとして、タイトル画面から「ストーリーモード」を選べるようにしたい。
> なぜなら、目的を持って遊びたいから。

**受け入れ基準**:
- [ ] タイトル画面に「ストーリー」ボタンがある
- [ ] 既存の「START」ボタンは「フリー対戦」に名称変更される
- [ ] ストーリーボタン押下でステージ選択画面に遷移する
- [ ] 既存のフリー対戦の動作が一切変わらない

#### US-2.3: ステージ選択画面
> ユーザーとして、クリア済みステージと未解放ステージが一覧で見たい。
> なぜなら、進捗と次の目標が分かるから。

**受け入れ基準**:
- [ ] 全ステージ（第1章: 3ステージ）がカード形式で一覧表示される
- [ ] 各カードに「ステージ名」「対戦相手名」「フィールド名」「難易度」が表示される
- [ ] クリア済みステージにチェックマークがある
- [ ] 未解放ステージはロック表示（暗転 + 🔒）
- [ ] ステージ 1-1 は初期状態で解放済み
- [ ] クリアで次のステージが解放される
- [ ] 「戻る」でタイトル画面に遷移する

#### US-2.4: 試合前ダイアログ
> ユーザーとして、試合前に対戦相手との会話を見たい。
> なぜなら、対戦に物語的な意味を感じたいから。

**受け入れ基準**:
- [ ] ステージ選択後、試合開始前にダイアログが表示される
- [ ] ダイアログは画面下部のテキストボックス形式（背景半透明暗転）
- [ ] キャラアイコン（64x64px）+ 名前 + セリフが表示される
- [ ] タップ/クリックで次のセリフへ進む
- [ ] 「スキップ」ボタンで全セリフを飛ばせる
- [ ] 最後のセリフの後、VS 画面 → カウントダウン → 試合開始

#### US-2.5: VS 画面
> ユーザーとして、試合開始前に「VS」の対決演出を見たい。
> なぜなら、少年漫画のような高揚感を味わいたいから。

**受け入れ基準**:
- [ ] 画面が左右に分割され、左にプレイヤー名、右に対戦相手名が表示される
- [ ] 中央に「VS」の大文字が表示される
- [ ] ステージ名とフィールド名が下部に表示される
- [ ] 表示時間は 2 秒、その後自動的にカウントダウンへ遷移
- [ ] フェードイン → フェードアウトのトランジション付き

#### US-2.6: 試合後ダイアログ
> ユーザーとして、試合後に対戦相手の反応を見たい。
> なぜなら、勝利の達成感や敗北の悔しさを共有したいから。

**受け入れ基準**:
- [ ] 試合終了後（リザルト画面前）にダイアログが表示される
- [ ] 勝利時と敗北時で異なるダイアログが表示される
- [ ] 章の最終ステージクリア時に「TO BE CONTINUED...」が表示される
- [ ] ダイアログ終了後にリザルト画面へ遷移

#### US-2.7: キャラクターアイコン画像
> ユーザーとして、キャラクターの見た目（アイコン）が魅力的であってほしい。
> なぜなら、文字だけだとキャラクターに愛着が湧きにくいから。

**受け入れ基準**:
- [ ] 各キャラクター（第1章: 3人 + 主人公 = 4人）に 64x64px のアイコン画像がある
- [ ] アイコンは統一されたアートスタイル（アニメ調、明るい色彩）
- [ ] ダイアログのキャラアイコン位置に表示される
- [ ] VS 画面のキャラ名の横に表示される
- [ ] ステージ選択カードに対戦相手アイコンが表示される

#### US-2.8: ストーリー進行の保存
> ユーザーとして、ブラウザを閉じてもストーリーの進捗が保持されてほしい。
> なぜなら、毎回最初からやり直したくないから。

**受け入れ基準**:
- [ ] localStorage に `ah_story_progress` キーでクリア状態が保存される
- [ ] ブラウザリロード後もクリア済みステージが保持される
- [ ] 「進行リセット」ボタンがステージ選択画面にある

### Epic 3: 2人対戦（MVP 外・将来計画）

#### US-3.1: ローカル2人対戦
> ユーザーとして、同じ画面で友達と対戦したい。
> なぜなら、対人戦はAI戦とは根本的に異なる面白さがあるから。

**受け入れ基準**:
- [ ] タイトル画面に「2P対戦」ボタンがある
- [ ] タッチ: 画面上半分(2P) / 下半分(1P) で独立操作
- [ ] キーボード: WASD(2P) / 矢印(1P) で同時操作
- [ ] スコアボードに「1P」「2P」の表記
- [ ] AI が無効化されている

## ユーザー提案の評価

### 案1: マンガストーリーモード ★★★★☆

**良い点**:
- 「次のステージへ」という最強の動機付け
- キャラへの愛着 → リプレイ性向上
- 既存の6フィールド・3難易度を自然に活用

**懸念点**:
- 漫画画像の制作が最大のボトルネック
- ストーリー量が膨らみやすい

**MVP での対応**:
- フルイラストではなく **AI 生成のキャラアイコン（64x64px）** で開始
- コマ割り漫画は MVP 外、テキストダイアログ + VS 画面で代替
- 第1章3ステージに限定

### 案2: 見た目クオリティ向上 ★★★★★

**良い点**:
- 第一印象を直接改善、全モードに効く
- Canvas 描画の変更のみ、既存ロジック影響なし
- 段階的に改善可能

**懸念点**:
- パフォーマンス影響（グラデーション描画の負荷）
- ほぼなし

**MVP での対応**: 全面的に採用

### 案3: 2人対戦 ★★★☆☆

**良い点**:
- エアホッケーは本来2人ゲーム、ジャンル相性最高
- 口コミ・共有の起点

**懸念点**:
- マルチタッチの実装複雑さ（iOS Safari の制約）
- 入力分離のテストが困難

**MVP での対応**: MVP 外。ストーリー + 見た目が安定した後に着手

## リスクと緩和策

| リスク | 影響 | 緩和策 |
|--------|------|--------|
| 画像生成の品質がばらつく | キャラの印象が統一されない | プロンプトにスタイル指定を厳密に定義、同一セッションで全キャラ生成 |
| グラデーション描画のパフォーマンス | フレームレート低下 | `OffscreenCanvas` でフィールド背景をプリレンダリング |
| ストーリーのダイアログ量 | 実装が膨らむ | 1ステージ当たり前後各3-5セリフに制限 |
| 既存フリー対戦が壊れる | 既存ユーザーに影響 | `GameMode` 型を導入し、フリー対戦パスは分岐のみで既存コードを温存 |
| localStorage のデータ構造変更 | 既存セーブデータ破損 | 新キー（`ah_story_progress`）を使い、既存キーに触れない |
